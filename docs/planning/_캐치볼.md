### C-025. 캠페인 구조 + 한 전장 목표 시간

#### C-025-A. 폐기된 안 — 체크포인트/이어하기 구조
- 원래 고려했던 안: 전장 라운드를 많이(예: 40) 두고 중간 체크포인트에서 이어하기 (예: 20라운드부터 시작하면서 그만큼의 마나·전술지원을 받고 시작).
- **폐기 이유**: [[#C-022]] 무한 모드 도입으로 V-1 (긴 카타르시스) 가 자율 충족 → 굳이 본편을 길게 + 체크포인트로 분할할 필요 없음.

#### C-025-B. 한 전장 목표 시간
- **본편 ≈ 20분 + 무한 모드 ≈ 10분** (총 ≈ 30분).
- 라운드 수는 시간 목표에 맞춰 후속 결정 — **실제 플레이해봐야 결정 가능**, 잠정만.
- 가이드 (잠정): 라운드 ≈ 2분 → 본편 10라운드 / 라운드 ≈ 1.5분 → 본편 13~14라운드.

#### C-025-C. 캠페인 구조
- **전역** → 여러 전장 → 모두 클리어 시 **다음 전역** 진입.
- 전역 안에는:
  - 일반 전장 (다수)
  - **중간 보스 전장**
  - **보스 전장** (전역 마지막, 보통)
- 중간 보스·보스 전장은 일반 전장보다 **플레이 타임 ↑, 난이도 ↑** 일 수 있음.

#### 후속 검토 ([[#Q-028]] / [[#Q-029]])
- 전역당 전장 개수
- 중간 보스/보스 전장의 시간 길이 (V-2 한계 점검)
- 보스 전장에서 무한 모드 적용 여부 (기존 기획 4.1: "지역 마지막 임무엔 엑스트라 라운드 없음")
- 일시정지/저장 기능 — 30분 한 판이 모바일에선 부담 가능, UX 디테일

### C-024. 무한 모드 보상 정책 + 패배 처리

#### C-024-A. 보상 배수
- 한 전장의 최종 보상 = **본편 클리어 보상 × 무한 모드 배수**.
- **무한 1라운드 클리어마다 +0.1 배수**.
- 예시: 본편만 = ×1.0 / 무한 5라운드 = ×1.5 / 무한 10라운드 = ×2.0.
- **난이도 의도 (★)**: 무한 모드는 **가파르게 어려워지므로 ×1.5 (5라운드) 달성도 어렵게** 설계. 평균 도달은 1~3 라운드 예상 → 평균 보상 배수 ≈ ×1.1~1.3.
- 이 난이도 의도가 [[#C-023]] 제약 1 (너무 쉬우면 의미 없음) 의 구체 가이드라인이 됨.

#### C-024-B. 무한 모드 진입 시 **패배 횟수 = 0 으로 리셋** (★ 핵심 안전장치)
- 본편: 패배 3회까지 허용 ([[07 게임 시스템 기획서]] 4.3).
- **무한 진입 시점에 잔여 패배 횟수 = 0**. 패배하면 즉시 전장 종료.
- 효과:
  - 무한 모드 = 빌드의 진가를 시험하는 **하드 모드**.
  - 자연 컷오프 — 약한 빌드는 무한 0~1라운드만에 종료 → 강제 길어짐 구조적 차단.
  - 강한 빌드는 길게 카타르시스 + 보상 가속 / 약한 빌드는 빠르게 다음 판으로 → 어느 쪽이든 자율.

#### C-024-C. V-2 자율성이 보장되는 메커니즘 (★ 정정)
- **전제 정정**: "어렵다 = 시간 더 걸린다"는 *틀린 가정*. 어려움은 **선택의 어려움** 으로 표현되며, 1라운드 소요 시간은 본편/무한 거의 동일 (정비/전투 시간 고정).
- **이중 안전장치**:
  1. **패배 0 리셋** ([[#C-024-B]]): 강제로 무한 돌고 싶어도 빌드가 약하면 1~2라운드만에 종료.
  2. **가파른 난이도** ([[#C-024-A]]): ×1.5 도달조차 어려워 평균 1~3라운드. 길게 도는 게 *구조적으로 어려움*.
- 결과: 무한 모드의 길이는 *빌드 강도에 자연 비례* → 길이 강제 없음 + 보상은 강도 차별화로만 작동.

#### 보상의 진정한 의미 재정의
- 0.1 배수 = "내 빌드가 얼마나 강한지" 의 정량 측정 + 메타 진행 가속.
- 무한 모드 동기는 **카타르시스 자체** ([[#C-023]] 제약 2) 가 우선, 배수는 **자기 강도 기록 + 부수 보너스**.

### C-023. 무한 모드 디자인 3제약

[[#C-022]] 의 자율 무한 모드가 의도대로 작동하려면 다음 세 가지를 동시에 만족해야 함.

#### 제약 1. **너무 쉬우면 의미 없음**
- 본편 빌드가 무한 모드 초반을 손쉽게 휩쓸면 → V-2 (게임 길이 제한) 무너짐.
- 무한 모드는 **본편 후반보다 명백히 어려운 난이도부터 시작**, 그 이후 가파르게 상승해야 함.

#### 제약 2. **무한 모드 진입 유인이 있어야 함**
- "본편 끝났는데 왜 무한을 해야 하지?" 에 명확한 답이 있어야.
- 유인 후보 (강도순):
  - **약 — 카타르시스 그 자체**: 내 빌드가 어디까지 버티는지 확인. M-1 정신과 정합. **가장 안전**.
  - **중 — 명예/기록 보상**: 리더보드, 최고 도달 라운드 기록, 칭호.
  - **강 — 메타 재화 보상**: 무한 N라운드 = 소비 재화. **Q-025 위반 위험 — 강제 무한 분위기 유발**.

#### 제약 3. **부조리하면 안 됨**
- 갑자기 침략자가 100배 강해져서 어떤 빌드든 즉사하는 식이면 X.
- 난이도 상승 곡선이 **예측 가능** 해야 함.
- **빌드를 강화한 만큼 더 오래 버틸 수 있는 비례 관계** = 플레이어 결정의 영향이 명확.

#### 제약 간 균형
- 1번 ↔ 3번: 너무 가파르면 부조리, 너무 완만하면 1번 위반. 곡선은 **부드럽게 가파른** 형태 (시그모이드 식 / 단계적 폭발 X) 가 안전.
- 2번: 1번·3번이 잘 균형 잡히면 "적절한 도전" 자체가 자연스러운 유인이 됨. 메타 보상 의존 최소화.

### C-022. ★ V-1 ↔ V-2 충돌 해법 — "본편 + 자율 무한 모드" 구조

#### 문제 재정의
- TFT 에서 카타르시스 부족의 이유 = "내가 너무 강해서, 적이 너무 약해서, 게임이 금방 끝나니 아쉬움".
- 즉 빌드가 강해진 만큼 적도 강해져야 카타르시스가 지속됨.

#### 해법
- 한 전장에 **(본편 라운드) + (난이도 가파른 무한 모드)** 를 함께 둠.
- 예: 본편 20라운드 + 그 이후 난이도가 급격히 상승하는 무한 라운드 진행.
- 기존 기획서의 "엑스트라 라운드" 개념 ([[07 게임 시스템 기획서]] 4.1) 의 의미가 명확해진 것.

#### 왜 이 구조가 V-1·V-2 둘 다 만족시키는가
- **V-2 (길이 제한) 충족**: 본편은 정해진 라운드 수에서 끝나는 명확한 컷 → 본편만 하고 빠지면 한 게임 ≈ 목표 시간 안에 끝남.
- **V-1 (카타르시스) 충족**: 본편 끝나고도 카타르시스가 부족한 플레이어는 무한 모드로 진입해 자기 빌드가 어디까지 버티는지 확인.
- **핵심 — 강제가 아니라 자율**: "강제로 길게" 가 아니라 "원할 때 길게" → 피로 안 쌓임. V-2 가 강제 시간 제한이 아니라 *최소 완료 시간*으로 작동.
- 결과적으로 한 게임 길이가 플레이어 의지에 따라 가변. (본편만 20분 / 무한 모드까지 30분+).

#### 주의해야 할 디자인 함정 (후속 검토)
- **무한 모드 보상 정책 ([[#Q-025]])**: 무한 모드에 메타 보상이 강하게 붙으면 "본편만 하면 손해" → 강제로 무한 도는 분위기 → V-2 위반.
- **방어전(무한 전용 모드) 과의 차별화 ([[#Q-026]])**: 캠페인 무한 부분 ↔ 별도 방어전 모드의 역할 분리.

### C-021. 전장 레벨 디자인의 핵심 가치 (★)

레벨 디자인의 모든 결정은 다음 두 가치를 가능한 한 모두 만족시키도록 한다.

#### V-1. 팀 빌딩 카타르시스를 가능한 한 많이 충족
- 빌드가 완성되어 작동하는 순간 + 그 순간이 *충분히 길게* 이어져야 함.
- M-1 ("빌딩한 팀을 오래 즐기기", [[#C-004]]) 의 직접 구체화.

#### V-2. 한 게임의 길이가 너무 길지 않을 것
- 한 전장(=한 런)이 과도하게 길면 **게임 종료 후 진이 빠짐** → 강화·다음 빌딩까지의 동선에 피로감 → "또 언제 팀 빌딩하지" 회피감.
- 본 게임은 **머리 쓰는 게임** (정비 페이즈 결정 다수) → 인지 부담 큰 만큼 V-2 가 **다른 게임보다 더 엄격하게** 작동해야 함.
- 목표 톤: **카타르시스 + 캐주얼함** 의 균형.

**참고 게임 (사용자 제시)**:
- ✅ **브로타도(Brotato)**: 좋은 예시. 카타르시스 강함 + 한 판이 짧음 (≈20분).
- ❌ **뱀파이어 서바이벌류**: 카타르시스는 좋지만 한 판이 길어 V-2 위반.

**상충 인식**:
- 카타르시스를 충족시키려면 게임 길이가 어느 정도는 필요. 두 가치는 본질적으로 상충함을 인지하고 균형점을 찾는다.

#### 두 가치의 충돌과 균형
- V-1 ↔ V-2 는 단순 라운드 수로 보면 충돌 (라운드 ↑ → 카타르시스 ↑ + 시간 ↑).
- 해결의 핵심은 **"카타르시스 구간 비율"** 을 높이는 것:
  - 빌드 완성 시점을 **앞당기고** (라운드 3~5쯤에 작동 시작, 8~12쯤에 완성)
  - 그 이후의 라운드를 카타르시스 구간으로 활용
  - → 전체 라운드 수를 늘리지 않고도 V-1 충족 가능
- **방어전(무한 모드)** 은 V-2 제약과 무관 → "카타르시스를 끝까지" 원하는 플레이어용으로 역할 분담.

### C-015. 전술 지원 새로고침 비용 = 마나 (인게임 통합 자원) — [[#Q-016]] 답
- 새로고침 = 소환·배치상한 증가·재스캔과 같은 마나 풀에서 소비.
- 효과: 자원 관리 결정 깊이 ↑. "새로고침 vs 소환" 의 트레이드오프 발생.
- 새로고침 마다 비용 2배 증가 ([[#C-013-D]]) 와 결합되어 무한 리롤 자동 방지.

### C-016. 전술 지원 ↔ 시너지 — 출력 우위 + 알고리즘 보정 — [[#Q-017]] 답

- **위계**: **전술 지원이 시너지보다 출력 우위**. 전술 지원의 지원 없이는 시너지를 맞춰도 좋은 빌드 어려움.
- **★ M-2 보호 메커니즘**: 전술 지원 등장 알고리즘이 **유저 친화적 보정** 적용 → 유저가 원하는 전술 지원이 등장하기 쉽도록.
  - 보정 인자 예: 현재 맞추고 있는 시너지 / 이전에 선택한 전술 지원 등.
- **귀결**:
  - 시너지가 빌드 **방향** 결정 → 알고리즘이 그 방향에 맞는 전술 지원 우선 노출 → 전술 지원이 그 방향의 **출력** 강화.
  - 결과적으로 시너지(가챠 캐릭터) 가 게임 매력의 핵심 자리(M-2)를 유지하면서도, 매판 결정의 무게는 전술 지원이 담당.
  - "출력 핵심 = 전술 지원" + "방향성 핵심 = 시너지" 가 알고리즘으로 자연스럽게 묶임.

### C-017. 전술 지원도 가챠/수집 대상으로 확장 — [[#Q-019]] 답

- 전술 지원을 **소환술사 모집과 동일한 재화로 가챠 수집** ([[#C-013-E]] 아이디어 확정).
- 재화 종류 늘리지 않음 — 가챠 재화로 통합.
- 의미: M-2 (수집 애정) 범위가 **캐릭터 + 시스템 강화** 둘 다로 확장.

### C-018. 출격 모듈 — 보유 모든 모듈 자동 적용 — [[#Q-020]] 답

- 슬롯 제한 없음. 해금/강화한 모든 모듈이 매 인게임에 자동 적용.
- 의미: 출격 모듈은 **점진적 게임 전반 강화** 의 축. 빌드 결정 축에는 포함되지 않음 (출전 편성 단계의 결정 부담을 덜어줌).
- 빌드 결정의 깊이는 시너지(출전 편성) / 전술 지원(인게임 중) 에 집중.

### C-020. 재화 체계 — "가챠 재화 / 강화 재화" 2종 분리 — [[#Q-022]] 답

| 재화 카테고리 | 용도 |
|---|---|
| **가챠 재화** | 소환술사 모집 · 전술 지원 모집 (수집형) |
| **강화 재화** | 출격 모듈 강화 · (소환술사 육성 등 강화형) |

- 원칙: **"뽑기"는 같은 재화끼리, "강화"는 같은 재화끼리**.
- 효과: 재화 종류 폭증을 막으면서, 수집/강화의 결정 영역을 명확히 분리.
- 후속 정리 필요: 소환술사 육성(승급/레벨업) 재화가 강화 재화로 통합되는지, 별도 심상조각 체계가 그대로 유지되는지 ([[#Q-023]]).

### C-014. 출격 모듈 — 컨셉, 작동 방식, 존재 이유

> 이름 "출격 모듈" 은 잠정. 변경 가능.

#### C-014-A. 컨셉
- **인게임 전반에 미치는 버프** (패시브)
- **또는 단발성 액티브** (인게임 중 1회 발동)
- 핵심: 한 번 강화하면 **인게임 전반에 영구 영향** (한 전장 내내 자동 유지).

#### C-014-B. 참고 모델
- TFT 에는 매칭되는 요소가 **없음**.
- 스타크래프트2 유즈맵 — 랜덤 조합 디펜스 류의 영구 강화 시스템이 모티프.

#### C-014-C. 예시
- 시작 시 마나 추가 지급
- 시작 새로고침 횟수 추가
- 적 처치 시 일정 확률로 추가 마나 획득
- (그 외 인게임 시스템 전반에 영구 영향 가능)

#### C-014-D. 존재 이유 (두 가지)
1. **인게임 재화의 소비처** — 캠페인 등으로 얻은 재화가 출격 모듈 해금·강화로 흘러감. 메타 진행감 형성.
2. **덱 강화의 한 축** — 가챠·육성·전술지원과는 다른 차원의 강화 축.

#### C-014-E. 4개 강화 축 정리

| 강화 축 | 단위 | 시점 | 휘발성 |
|---|---|---|---|
| 가챠 | 캐릭터(시너지) 풀 확장 | 메타 | 영구 |
| 소환술사 육성 | 캐릭터(시너지) 능력 강화 | 메타 | 영구 |
| **출격 모듈** | 인게임 시스템 강화 | 메타 | 영구 (적용은 인게임마다) |
| 전술 지원 | 인게임 빌드 강화 | 인게임 중 선택 | 한 전장만 |

### C-013. 전술 지원 — 위상, 등장 빈도, 효과 범위, 선택 방식

#### C-013-A. 위상 — "팀 강화의 핵심"
- 시너지 = 팀 **빌딩** (컨셉 결정)
- 전술 지원 = 그 팀의 **강화** (출력 결정)
- 본 게임은 아이템이 없으므로 ([[#C-005]]), 전술 지원이 **TFT의 증강 + 아이템 역할을 동시에 흡수**.

#### C-013-B. 등장 빈도
- TFT 증강은 3번. 본 게임은 더 자주.
- 잠정: **약 3라운드마다 1번** (밸런스 따라 조정).
- 함의:
  - 10라운드 캠페인 → 3~4번 (TFT와 비슷).
  - 30라운드 캠페인 → 약 10번 → 후반으로 갈수록 누적 강화 풍부 → **완성된 빌드 카타르시스 (M-1)** 와 잘 맞음.

#### C-013-C. 효과 범위 — 인게임 시스템 전반에 작용
- 소환수 전체 강화
- 특정 심상 효과 강화
- 특정 소환수 효과 강화
- 마나·재스캔 등 자원 강화
- 그 외 인게임 시스템 전반에 영향 가능

#### C-013-D. 선택 방식
- **3택 1** (한 번에 3개 등장, 1개 선택).
- **새로고침 기능** 존재.
  - 한 번 새로고침 할 때마다 **비용 2배 증가** (잠정).
  - 비용 자원 미정 — 별도 캐치볼 ([[#Q-016]]).

#### C-013-E. 미래 아이디어 — 전술 지원 자체를 수집 요소로 확장
- 사용자 아이디어 수준. 즉시 결정 X.
- 기존 기획서엔 "전술 지원 연구소" 에서 인게임 재화로 해금/강화 ([[00 기획서]] 3.2.3) 까지 정의됨.
- **수집 요소로 격상** 시 가능성:
  - M-2 (수집 애정) 범위가 캐릭터를 넘어 시스템으로 확장.
  - 메타 진행감 강화 — 캠페인 진행 = 새 전술지원 잠금 해제.
- 우려:
  - 수집 대상 증가 → 학습 부담 ↑ ([[#C-006]] 에서 짚었던 문제 재발 가능).
  - 가챠와의 관계 정리 필요 (전술지원 가챠? 캠페인 보상으로만? 자원 구매?).
- 별도 캐치볼 대상 ([[#Q-019]]).

### C-012. 심상 효과는 별도 분류 체계를 두지 않음

- **결정**: 심상 효과를 능력치형·운용형·메커닉형 같은 카테고리로 분류하지 않음.
- **귀결**:
  - 각 심상 효과는 자유롭게 디자인 (능력치 강화·자원·트리거·메커닉 등 어떤 형태든 OK).
  - 단, 구조는 [[#C-010-D]] 의 "메인 효과 + 글로벌 버프" 이중 컴포넌트 형태는 유지.
- **참고**: TFT 도 시너지를 카테고리화하지 않음 — 자연스러운 선택.

### C-011. 소환수 역할군 분포 — 술사 안에서 균등 분포

- **결정**: 한 소환술사의 소환수 5체는 **기본적으로 5 역할군에 균형 잡힌 분포**를 가짐. 단 술사마다 **±1 수준의 작은 바리에이션** 으로 차별점을 줌. 극단적인 공격형·방어형은 만들지 않음.
- **시뮬레이션 근거**: 모든 술사가 균등 1:1:1:1:1 일 경우, 한 게임에서 활용되는 10~12체 기준으로 **딜러 비율이 부족**. 술사마다 살짝 다른 분포로 풀어서 전체 균형을 맞춤.
- **변주 예시 (감 잡기용)**:
  - A 술사: 1탱 1원거리딜 **2**근거리딜 1마법 0힐 (힐 → 근거리딜)
  - B 술사: 1탱 1원거리딜 1근거리딜 1마법 1힐 (정균등)
  - C 술사: **2**탱 1원거리딜 1근거리딜 0마법 1힐 (마법 → 탱)
  - 4 술사 합치면 자연스럽게 딜러 비율이 살짝 높아짐.
- **변주 폭 가이드**: 한 술사 안에서 어떤 역할군이 **0~2체** 사이로 분포. 3체 이상 한 역할군에 몰리는 극단 분포는 X.
- **0체 허용 — 추가 의도**:
  1. **비율 자연 조정**: 모든 술사가 힐러 1체를 가지면 한 게임 활용 10~12체 중 힐러 비율이 과다해짐. 0체 허용으로 자연스럽게 비율 통제.
  2. **밸런스 유연성**: 분포를 1:1:1:1:1로 고정하면 향후 밸런스 조정 여지가 줄어듦.
  3. ★ **출전 편성을 "역할군 퍼즐"로 만듦**: 플레이어가 4 술사를 고를 때 "내 조합의 역할군 분포가 균형 잡히는가" 를 의식적으로 고려하게 됨. 단순히 좋아하는 캐릭터 4명 고르기가 아니라, **출전 편성 자체에 빌딩 결정의 깊이가 추가**됨.
- **귀결**: 출전 편성이 게임플레이의 **첫 빌딩 결정 단계**로 격상. M-3(다양성)와 빌딩 깊이 모두 강화.
- **근거**:
  - [[#C-010]] 에서 "2 심상 고정 사용 자체는 OK + 비메인 술사는 글로벌 버프로 가치 부여" 로 결정 → 술사별 역할군 몰기로 강제 다양화할 필요 없음.
  - [[#Q-012]] 에서 짚은 "술사별 역할군 몰기" 안의 단점 3가지 모두 자동 해소:
    1. 가챠 결과에 따른 술사 조합 제약 없음
    2. 한 술사 내 동일 역할군 5체 디자인 부담 없음, 플레이어 인지 부담도 낮음
    3. 소환수 효과에 **역할군별 보정** 적용 가능 (탱커용 효과는 탱커에게 갈 확률 ↑)
- **귀결**:
  - 한 술사 = 5 역할군 풀 셋 → 그 술사만으로도 단독 운용 가능한 자족 구조.
  - 2 술사 메인 활용 시: 같은 역할군 2체씩 자연 확보 → 시너지 임계치 도달 자연스러움.

### C-010. "2심상 고정 사용"은 문제 아님 / 단, "버림패 술사"는 방지해야 함

#### C-010-A. "한 게임 내 2 심상 사용" 자체는 OK
- 클로버핏 등의 사례 — 덱빌딩의 **결과로** 2개 문양으로 수렴하는 것은 자연스러운 게임플레이.
- 본 게임도 한 게임 안에서 2 심상으로 수렴하는 것 자체는 문제 X.

#### C-010-B. 잠재 문제: "매 게임 같은 2심상 고정으로 인한 지루함"
- 본 게임은 인게임 진입 *전에* 출전 편성 단계에서 심상이 고정됨.
- 강화 매몰비용으로 인해 플레이어가 새 심상으로 갈아타기 어려움 → 장기 지루함 가능.
- 이 문제는 "역할군 분포" 와는 다른 축의 문제이므로 [[#Q-013]] 으로 분리하여 후속 검토.

#### C-010-C. 핵심 제약: **비메인 2 술사가 "버림패"가 되어서는 안 됨**
- 4 술사 출전 구조를 유지하려면, 비메인 술사도 명확한 가치를 제공해야 함.

#### C-010-D. 방향: 심상 효과의 "글로벌 버프 컴포넌트"
- 심상 효과가 **자기 심상 소환수 강화**(메인 효과) **+** **다른 심상에게도 영향을 주는 약한 효과**(글로벌 버프) 의 이중 구조.
- 운영 양상:
  - **메인 2 심상**: 임계치 깊게 활성화 (예: 4/6/8) → 본격 시너지 효과.
  - **비메인 2 심상**: **1티어(2 임계치)만 활성화** → 글로벌 약한 버프 유지.
- 효과:
  - 4 술사 모두 출전 가치를 가짐 → "버림패" 해소.
  - 비메인 2 술사도 가챠/육성의 매력 유지 → M-2 강화.
  - 빌딩 자체에 "메인 2 + 서브 2" 라는 새 결정축이 생김 → 깊이 증가.

- → 마법사의 차별점은 능력치 분리가 아니라 **"스킬 의존도가 높은 캐릭터 컨셉"** 으로 풀릴 가능성 높음. 별도 정의 필요 ([[#Q-011]]).

### C-008. 소환수 기획 — 스킬 정책 (단순 스킬 유지)

> 사용자가 풀어준 소환수 기획의 일부. 다음 항목은 차후 캐치볼:
> - C-008-a. 술사당 소환수 수 (★ C-007-B 에서 5체로 잠정)
> - C-008-b. 역할군 분류 결론 (탱커/딜러/힐러 + 딜러 세분)
> - C-008-c. 소환수 효과 분류 체계

#### C-008. 스킬을 둔다 — 단, 아주 단순한 스킬만

**스킬을 두는 이유**:
1. **최소한의 전략적 선택 보장**: 스킬이 없으면 소환수 운용이 단조로워짐.
2. **소환수 특색 확보**: 스킬이 없으면 소환수 간 차별점이 **오로지 역할군**으로만 갈리는 빈약한 구조가 됨.

**스킬을 단순하게 유지하는 이유**:
- 술사를 바꿔 끼면 → 그 술사 소속 소환수의 모든 스킬을 다시 학습해야 함.
- 스킬이 복잡하면 술사 교체 시마다 학습 부담 폭증 → 수집·교체의 재미를 망침.

**구현 방향**:
- 스킬은 **쿨타임 기반**으로 동작.
- 예시 스킬: 범위 공격 / 쉴드 부여 등 "한 줄로 설명되는" 효과.

**정리 위치 메모**:
- 스킬 *규칙* (쿨타임 산정, 발동 조건, 데미지 계산 등) → [[07 게임 시스템 기획서]] 1.8 스킬 규칙
- 각 소환수의 *고유 스킬* → 소환수 챕터 또는 별도 카탈로그
- 사용자 의견: "전투 시스템 쪽에 정리하는 게 나을 수도 있겠다" — 검토 필요.

### C-007. M-3 풀이 방식 / 지루함 극복 장치 (개요 + 소환수 효과 랜덤성 상세)

**전제**: 본 게임은 싱글게임 — 시즌제 시너지 갱신이 없음 → 지루함에 **더 취약** → 시스템 내에서 다양성을 적극적으로 만들어야 함.

**4가지 지루함 극복 장치**:
1. **소환술사 수집**: 새 시너지·새 캐릭터 = 새 빌드 가능 (M-2 ↔ M-3 연결)
2. **소환수 효과 랜덤 배치**: 매 전장마다 다른 소환수 효과 분배 → 매번 다른 전략 강제
3. **전술 지원**: TFT의 증강체 역할. 인게임 내 빌드 분기.
4. **출전 모듈**: 영구 강화로 메타 진행감.

---

#### C-007-A. 시너지의 2차원 구조 (TFT 차용)
- 택티카는 TFT처럼 시너지가 **2차원**으로 구성됨.
  - **심상 효과**: 출전 편성한 소환술사 선택에 의해 결정. (고정/선택 가능)
  - **소환수 효과** (이름 변경 가능, 기존 "소환수 특성"): 전장마다 랜덤으로 소환수에 부여. (랜덤)
- 각 소환수는 전장 진입 시 **심상효과 1개 + 5개 중 랜덤한 소환수 효과 1개** 를 보유한 상태로 시작.

#### C-007-B. 소환수 풀 / 소환수 효과 수치 변경 (★ 기존 기획서 수치 갱신)

| 항목 | 기존 기획서 | 변경 후 |
|---|---|---|
| 술사당 소환수 수 | 8 | **5** |
| 총 소환수 풀 (4술사 기준) | 4×8 = 32 | **4×5 = 20** |
| 소환수 효과 풀 (한 전장 등장 수) | 8 | **5** |
| 소환수 효과 임계치 | 2/3/4 | 2/3/4 (유지) |

#### C-007-C. 소환수 효과를 랜덤 배치하는 이유
1. **지루함 감소(M-3)**: 같은 소환수도 매 전장마다 다른 효과 보유 → 매번 다른 전략 강제.
2. **유저·기획 인지 부담 감소**: "소환술사마다 소환수 구성이 제각각" 인 상황에서, 추가로 "어떤 소환수가 어떤 효과를 가지는지" 까지 고정해놓으면 학습 부담이 폭증. 차라리 랜덤이 낫다고 판단.

**용어 검토**:
- 기존 "소환수 특성" → "소환수 효과" 로 변경 검토 중 (심상 효과와 대칭 유지).

### C-006. M-2 풀이 방식 / 수집의 대상을 "기물"에서 "시너지(=소환술사)"로 시점 전환

**캐릭터 수집을 도입한 이유**:
1. **재미·중독성 증폭**: TFT가 자신만의 팀을 꾸리는 게임이라면, 거기에 자신만의 캐릭터를 육성하는 수집 요소가 더해지면 팀 빌딩의 재미·중독성이 크게 증폭됨.
2. **반복 지루함 완화 (M-3 연결)**: 캐릭터 수집을 통해 **팀 풀 자체를 스스로 구성하는 덱빌딩**이 가능해짐 → TFT처럼 매 시즌 고정된 팀 풀이 아니라, 플레이어의 수집 상태에 따라 빌딩 경로가 달라짐.

**핵심 문제 — 기물(소환수)을 수집 대상으로 두는 것의 부적합성**:
1. **콘텐츠 제작 부담**: 기물은 20~50개 규모. 캐릭터 수집 대상이 되려면 모두에 충분한 캐릭터 디자인이 필요 → 제작량 폭증.
2. **유저 인지 부담**: "수집할 만한 매력"은 캐릭터 정보량과 비례. 매력 = 정보. 정보가 30개+ 캐릭터에 분산되면 유저가 집중 불가. 빌딩 풀 기준 1.5배 = 30개+ 이상 디자인 필요. TFT 기물 외우기도 러닝 커브가 가파른데, 거기에 캐릭터 수집까지 얹는 것은 불가능.

**해결 — 시점 전환**:
- 수집 대상을 기물이 아니라 **시너지** 로 바꿈.
- **시너지 ↔ 학교 / 심상 / 소환술사** 라는 개념으로 캐릭터(소환술사)와 묶음.
- 결과: **소환술사 = 시너지의 캐릭터화** 가 수집 단위가 됨. 인게임에서 유효한 영향(시너지 효과)을 미치는 캐릭터를 수집하는 구조.
- 이로 인해 수집 대상 수를 합리적으로 줄이면서도 "매력 + 인게임 영향" 두 조건을 동시에 만족.

**소환수 처리 — 기획 진행 중**:
- 소환수를 수집의 무대에서 어떻게 다룰지는 현재 기획 변경 진행 중. 다음 캐치볼에서 별도 설명 예정.
- 임시 메모: [[#Q-006]] (소환수 가챠 대상 여부) 는 이 새 기획에서 함께 답이 나올 것으로 예상.

**참고 — M-2 ↔ M-3 연결고리**:
- 캐릭터 수집은 M-2(애정·중독성) 의 핵심 장치이지만, **동시에 M-3(지루함 극복) 의 일부분**도 담당.
- "수집 상태에 따라 빌딩 가능 풀이 달라짐" 자체가 반복 플레이의 다양성을 만든다.

### C-005. M-1 풀이 방식 / 빌딩 축 구성 철학 (개요)

> 세부 내용은 차후 캐치볼에서 보강.

**전제**:
- 싱글 게임이므로 한 선택을 충분히 고민할 시간이 있음 → **선택지는 많이 줘도 됨**.
- 단, **선택의 축이 너무 많으면 피로감**. "3개 중 하나 고르기"는 쉽지만, 여러 축에서 동시에 최적 조합을 찾는 것은 재미와 피로의 경계.

**TFT의 빌딩 축 분석**: 시너지 / 아이템 / 증강체 / 기물(스킬·역할군·성급·레어도) / 레벨

**본 게임의 결정 — 축을 줄이고, 깊이를 늘림**:

| 축 | 처리 | 비고 |
|---|---|---|
| 레벨 (코스트별 등장 확률 변화) | **제거** | 대신 배치 상한 증가로 대체 |
| 레어도 (코스트별 분리) | **제거** | 모든 소환수가 동일 비용으로 동일 풀에서 등장 |
| 아이템 | **제거** | |
| 시너지 | 유지 | 심상 효과 + 소환수 특성 2종 |
| 증강체 (= 전술지원) | **유지 + 강화** | 한 번이 아니라 **여러 번 제공** |
| 기물 (스킬·역할군·성급) | 유지 | 합성으로 성급 상승 |
| **출격 모듈 (신규)** | **추가** | 인게임에 **영구적으로 영향** 주는 메타 업그레이드. 게임 시작 시 마나 추가, 상위 성급 등장 확률 증가 등 |

**출격 모듈 도입 이유**:
- 가챠 외에는 인게임이 **휘발적 팀 빌딩**만 존재함 (런 끝나면 사라짐).
- 메타 차원에서 **누적되는 진행감**이 필요 → 출격 모듈이 그 역할.

### C-004. 게임 제작 목적 / 핵심 가치 (★ 모든 시스템 결정의 판단 기준)

이 게임은 **TFT에서 출발한 "나의 아쉬움"을 해결하기 위해** 만든다. 시스템 결정은 항상 아래 3가지 동기에 부합하는지로 판단한다.

#### M-1. 빌딩한 팀을 오래 즐기고 싶다 (시간 압박 해소)
- **문제 정의**: TFT는 PvP 타임어택 — 증강체·아이템·시너지를 잘 맞춰 카타르시스를 느끼려 해도 3~4 전투 만에 게임이 끝남.
- **해결 방향**: PvE 싱글로 옮겨, **원하는 덱 빌딩을 천천히·충분히 하면서**, 그 덱이 동작하는 카타르시스를 **오래** 느끼도록 한다.
- **관련 시스템(현재 기획 반영)**: PvE 캠페인 10~30라운드, 엑스트라 라운드, 방어전 무한 모드, 라운드 간 HP/쿨다운 리셋(긴 전투 지속 가능).

#### M-2. 가챠 + 서브컬쳐 매력 → 캐릭터 애정·중독성
- **문제 정의**: TFT에 가챠 시스템이 있으면 재미있겠다는 생각. 덱에 등장하는 캐릭터에 **애정**이 붙으면 빌딩 자체의 재미가 훨씬 커진다.
- **해결 방향**: 매력적인 서브컬쳐 일러스트의 캐릭터를 **가챠로 수집**하고, 그 캐릭터들이 자기가 빌딩한 덱에 등장하도록.
- **관련 시스템(현재 기획 반영)**: 소환술사 가챠, 소환술사 육성, 중복 시 심상조각, 시너지가 소환술사에 1:1 귀속(가챠 캐릭터 → 빌딩 직결).

#### M-3. 지루함 극복 (반복 플레이의 다양성)
- **문제 정의**: TFT는 시즌 안에서 **빌딩 가능한 팀이 한정** 되어 시즌이 조금만 지속돼도 지루해짐. **이 문제는 싱글 게임에서도 유효** 하므로 반드시 해결해야 함.
- **해결 방향**: (현재 기획서 장치) 소환수 특성 매 전장 랜덤 부여, 전술지원 선택, 출격모듈, 시너지 사전 덱빌딩으로 매 런마다 다른 빌드.
- **현재 약점**: 위 장치들이 "시즌 한정 → 지루함" 문제를 정말 충분히 해소하는지 미검증. 가장 약한 고리로 보임 — 별도 캐치볼 필요(Q-004).

### C-003. 출전 편성 시 "같은 심상(품종) 중복 금지" 규칙 유지
- **결정**: 4명 술사는 **서로 다른 4개 품종(심상)** 으로 편성해야 함. 같은 품종 술사 중복 출전 불가.
- **유지 이유**: 동일 심상 소환수가 터미널에 중복 등장하는 상황 방지 — 기존 의도 그대로.
- **귀결**: 소환수 풀은 4품종 × 8체 = 32체 구성으로 깔끔하게 분리됨.

### C-002. 학교 ↔ 심상 1:1 매핑 폐기 / 하나의 학교에서 여러 술사 출전 가능
- **결정**:
  - 기존 "학교 = 심상 1개, 같은 학교 술사는 같은 심상 공유" 규칙 **폐기**.
  - 이번 게임에는 **고양이를 다루는 학교** 출신 술사들만 등장.
  - 같은 학교 안에서도 술사마다 서로 다른 심상(고양이 품종)을 가짐.
  - 출전 편성에서 **같은 학교 출신 술사 여러 명 출전 가능**.
- **영향**:
  - [[00 기획서]] 2.1.3, [[04 세계관]] 16.7.1, [[07 게임 시스템]] 5.1·5.2.1 의 "같은 학교 = 같은 심상" 서술 전반 수정 필요.
  - 후속 결정 필요: 출전 편성 시 "같은 심상 중복 금지" 규칙(Q-003) 유지 여부.

### C-001. 심상 = 고양이 종족으로 통일
- **결정**: 이번 게임에 등장하는 모든 심상은 **고양이 종족(품종)** 으로 한다.
  - 예: 스코티시폴드 / 스핑크스 / 코리안숏헤어 등 실제 고양이 품종을 심상 단위로 사용.
- **소환술사**: 인간(기존 설정 유지 — 청소년~20대 술사)
- **소환수**: 모두 **고양이 캐릭터**. 냥코대전쟁처럼 전투에 적합한 형태로 변형된 귀여운 고양이.
- **세계관 본 설정 유지**: 외계체 침공 / 페이즈 입자 / PLGU(완드) / 바스티온 등 [[04 세계관 기획서]] 큰 틀은 그대로.
- **의도**:
  - 기획 단순화 (심상 카테고리가 한 축 — 고양이 품종 — 으로 정리됨)
  - 비주얼 통일감 + 귀여움 어필

### Q-027. 무한 모드 배수 상한 / 시간당 보상 균형 ([[#C-024]] 후속)
- 배수 상한이 필요한가? (예: 최대 ×3.0)
- 시간당 보상 비교:
  - 본편만 = 보상 × / 시간 T
  - 본편 + 무한 5라운드 = 보상 ×1.5 / 시간 T + Δ
  - 무한 모드의 시간당 보상이 본편보다 *항상 낮아야* V-2 자율성 보장.
  - 즉 "0.1 배수 추가" 가 무한 모드 1라운드 클리어에 드는 시간 대비 *불리해야* 안전.

### Q-026. 캠페인 무한 부분 ↔ 별도 방어전 모드의 차별화

### Q-012. ~~소환술사 ↔ 역할군 분포~~ → [[#C-011]] 로 결정됨 (술사 내 5 역할군 균등)

#### 문제 정의 — "원하는 심상만 쓰는 문제"

**플레이 시뮬레이션**:
- 배치 상한 10~12 (예정), 중복 배치 제한 없음.
- 한 게임에서 활용될 소환수 ≒ 10~12체.
- 소환술사 강화가 존재하므로 플레이어는 **선호 심상** 이 명확히 생김.
- 결과: **2가지 심상 효과만 주로 사용** → 출전한 4 술사 중 **2 술사는 사실상 버림패**.

**TFT와의 비교**:
- TFT: 기물 경쟁 + 시너지 밸런스 → 매판 다른 시너지 사용이 자연스러움.
- 본 게임: 소환술사 강화 + 기물 경쟁 X → **자기가 원하는 심상만 고정 사용** 하는 쏠림이 자연 발생.

---

#### 사용자가 검토한 안: "소환술사별로 역할군 몰기"
- 예: "A 술사 = 탱커 전문", "B 술사 = 딜러 전문"
- 효과: 플레이어가 역할군 다양성 확보를 위해 **여러 심상 효과를 강제로 함께 쓰게 됨**.

#### 위 안의 단점 (사용자 분석)
1. **여유 소환술사 필요**: 가챠로 뽑은 술사 3명이 모두 딜러 계열이면 그중 1명만 실제 활용 가능. 플레이어가 원하는 술사 조합을 제한.
2. **소환수 디자인 부담 + 플레이어 인지 부담**: 레어도 없음 → 한 술사 안에 같은 역할군 소환수 5체를 모두 디자인해야 함. 동일 심상·동일 역할군 소환수 5체는 플레이어에게도 구분 어려움.
3. **소환수 효과 랜덤성 저해**: 역할군 보정(탱커 효과는 탱커에게 갈 확률 ↑)을 주려 했으나, 역할군이 술사 단위로 몰리면 동일 효과가 한 술사 소환수들에 집중됨 → 보정 제거하고 **범용 효과만** 쓸 수밖에 없는 문제 발생.

#### 미결 / design-partner 후속 질문 (아래 캐치볼에서 다룸)

a. **"2 심상 고정 사용" 자체가 문제인가?** — M-1 (오래 즐기기) 관점에서 보면 "내가 좋아하는 심상을 깊게" 가 오히려 정합적일 수도. 강제 다양화가 진짜 필요한가?

b. **출전 술사 수 4명이 고정 전제인가?** — 어차피 2 심상만 쓴다면 출전을 2 술사로 줄이는 것도 옵션.

c. **버림패 술사의 가치 재정의** — 출전했지만 메인이 아닌 술사가 "부가 가치 (패시브 / 출격 보너스 / 완드 효과)" 를 제공하면 버림패가 아니게 됨.

d. **시스템적 인센티브** — "역할군 다양성 보너스" 같은 룰로 자연 유도. 예: 4 역할군 이상 배치 시 능력치 보너스.

e. **심상 효과를 역할군 비의존적으로** — "탱커 강화" 가 아니라 "전체 소환수 강화" 식으로 가면, 어떤 역할군 분포든 OK + 효과 보정 가능.

### Q-011. 마법사의 차별점은 무엇인가

  → 평타가 약하고 스킬이 강하다? / 스킬 쿨이 짧다? / 범위 스킬 위주? — 정의 필요.

### Q-010. 단순 스킬과 M-3(다양성) 트레이드오프

- 보완책: 단순하지만 **종류는 충분히 다양**하게 (단순 효과 N종 조합) — 합의 필요.

### Q-007. 5개 효과 × 20 소환수의 임계치 4 도달 난이도
- 효과 5개가 20 소환수에 균등 분배되면 평균 4체/효과. 임계치 4 트리거는 빠듯하거나 "균등 분배 시에만 가능".
- 분배 방식이 균등인지, 가중치/보정이 있는지 정의 필요 (기존 5.3에 "역할군 우선 보정" 언급 있었음 — 유지 여부).

### Q-005. M-1 ↔ M-3 균형
- M-1은 "오래 즐긴다(같은 덱 길게)" 방향, M-3은 "매번 다르다(빌드 다양)" 방향.
- 한 런 내에서는 M-1(오래), 런과 런 사이에는 M-3(다름) — 이 분리가 맞는지 확인 필요.

### Q-006. 소환수도 가챠 대상인가

- M-2(가챠 캐릭터 애정) 관점에서 이게 의도된 분리인지 확인 필요.
