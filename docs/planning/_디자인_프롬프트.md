# 택티카 디펜스 — 게임 체계도 인터랙티브 HTML 디자인 의뢰

## 역할

너는 게임 기획서를 시각화하는 웹 디자이너야. 첨부된 게임 기획서 문서들을 읽고, **이 게임의 전체 시스템을 한 눈에 파악할 수 있는 인터랙티브 단일 HTML 파일**을 디자인·구현해.

## 산출물

- **단일 HTML 파일 1개** (`택티카_디펜스_체계도.html`)
- 외부 의존성 없음 — CSS / JS 모두 인라인. 폰트는 시스템 폰트 또는 Google Fonts CDN 1개까지 허용.
- 브라우저로 바로 열어서 동작해야 함.

## 첨부 문서

다음 기획서 문서가 모두 첨부돼:

1. `00 기획서.md` — 게임 개요 전체, 플레이 시나리오, 메뉴 구조
2. `04 세계관 기획서.md` — 세계관, 심상·소환수 설정
3. `07 게임 시스템 기획서.md` — 전장 인게임 시스템 상세 (유닛/소환수/터미널/시너지/전술지원/출격모듈)
4. `08-1 컨텐츠 레벨디자인.md` — 캠페인 구조, 본편+무한 모드, 보상 정책
5. `08-2 육성 스케일링.md` — 메타 강화 축, 가챠, 육성, 재화 체계

**원칙**: 기획서의 모든 시스템 요소를 누락 없이 반영. 단, 본 문서의 "기획서와 달라진 점"은 본 문서를 우선해.

## 기획서와 달라진 점 (반드시 반영)

1. **방어전 모드 제외** — 캠페인 (스토리 모드) 만 다룸. 방어전 관련 내용은 모두 생략.
2. **출전 편성의 위치** — 출전 편성은 메인 메뉴 항목이 아니라 **전장 진입 직전 화면**. 메인 메뉴에서 직접 들어가는 메뉴가 아님.
3. **메인 메뉴의 전장 진입 명칭** — "전장 출격" 으로 통일. 스토리 모드 (캠페인) 만 진입.
4. 메인 메뉴 항목 예시: 전장 출격 / 모집 / 소환술사 (도감·육성) / 전술 지원 연구소 / 출격 모듈 / 설정 등.

## 핵심 요구사항 — "한 눈에 + 인터랙티브 드릴다운"

이 디자인의 가장 중요한 목적이야:

### 1. 첫 화면에서 게임 전체 시스템이 한 눈에 보여야 함

- 스크롤로 순서대로 읽게 만들지 마. 한 화면 (또는 첫 뷰포트) 에서 게임의 모든 요소·구성이 시야에 들어와야 해.
- 예: 메인 메뉴를 중심으로 한 마인드맵 / 게임 전체 시스템의 다이어그램 / 영역별 그리드 등.
- 보는 사람이 "아, 이 게임에는 이런 시스템들이 이렇게 구성돼 있구나" 를 첫 5초 안에 파악할 수 있어야 함.

### 2. 각 요소를 클릭/호버하면 세부 정보가 펼쳐지는 인터랙션

- 첫 화면에는 각 시스템 요소의 **이름·아이콘·한 줄 요약** 만 보여줘.
- 사용자가 특정 요소를 클릭하면 그 요소의 **세부 사항** 이 펼쳐지거나 모달로 열림.
- 인터랙션 방식은 자유 — 다음 중 어울리는 것을 선택하거나 조합:
  - 클릭 시 카드 expand (아코디언)
  - 클릭 시 사이드 패널 슬라이드
  - 클릭 시 모달 오픈
  - 호버 시 툴팁 미리보기 + 클릭 시 전체 상세
- 세부 정보 안에서 또 다른 요소를 클릭하면 이어서 드릴다운 가능하면 더 좋아.
- ESC / 외부 클릭으로 닫기, 모달 안 스크롤 등 기본 UX는 갖춰져야 해.

### 3. 보는 사람의 정신모델

이 페이지를 보는 사람의 목적은 두 가지야:

- **(A) 빠른 파악**: "어떤 게임이고 어떤 시스템들이 있는지" 를 첫 화면에서 안다.
- **(B) 깊이 보기**: 관심 가는 시스템 하나에 클릭해서 그 시스템의 규칙·수치를 본다.

이 두 모드가 자연스럽게 전환되어야 해. 한 화면 안에서 줌인/줌아웃 하는 느낌이 이상적.

## 디자인 톤

이 게임은 **고양이 소환수의 갭모에** 가 핵심 톤이야. 디자인도 그에 맞게:

- **밝고 발랄하고 귀여운** 분위기. 다크 테마 X.
- 추천 베이스: 크림 / 오프화이트 배경 + 파스텔 액센트 (핑크, 민트, 라벤더, 옐로, 하늘색).
- 라운디드 모서리, 부드러운 그림자, 둥글둥글한 형태.
- 적절한 이모지/아이콘 사용 가능 (🐾 🐱 🪄 🛡️ 등) — 단 과하지 않게.
- 진지한 SF 톤이 아니라 **귀엽고 다정한 톤**.

폰트 추천 (자유 선택):

- 한글: Pretendard / Noto Sans KR / Cafe24 Ssurround
- 영어: Quicksand / Nunito / Fredoka

## 게임 컨셉 빠른 요약 (배경 이해용)

- **장르**: 오토배틀러 + 타워디펜스 + 가챠 (싱글 PvE)
- **세계관**: 205X년 외계체 침공 근미래 SF. 인류는 "심상" 을 무기로 변환. 본 게임 등장 소환술사는 모두 **고양이 학교 출신** 이며, 모든 소환수는 다양한 **고양이 품종** 형태.
- **핵심 루프**: 메타 (모집·육성) → 전장 출격 → 출전 편성 (술사 4명) → 전장 인게임 (정비/전투 페이즈 교차) → 보상 → 다시 메타
- **전장 구조**: 본편 (약 20분, 패배 허용 3회) + 자율 무한 모드 (약 10분, 패배 1회 즉시 종료, 보상 ×1.0 + 무한 1라운드당 +0.1 배수)

## 페이지에 반드시 포함되어야 할 시스템 영역

아래 영역들이 모두 첫 화면에 나타나거나 1-depth 안에서 접근 가능해야 해:

### 메타 (게임 외부 — 메인 메뉴 진입)

- **전장 출격** — 캠페인 (전역 → 전장) 선택 진입
- **모집 (가챠)** — 소환술사 가챠 / 전술 지원 가챠
- **소환술사 (도감·육성)** — 레벨업 · 승급 · 심상 효과 강화
- **전술 지원 연구소** — 해금 / 강화
- **출격 모듈** — 해금 / 강화

### 전장 진입 전

- **출전 편성** — 보유 소환술사 중 4명 선택 (서로 다른 심상). 등장 소환수 풀 확인.

### 전장 인게임

- **라운드 구조** — 본편 + 자율 무한 모드 (방어전 X)
- **페이즈 구조** — 정비 페이즈 ↔ 전투 페이즈
- **소환 터미널** — 소환 / 스캔(리롤) / 잠금 / 환원 / 배치 상한 증가 / 마나
- **소환수** — 고양이, 합성 (3개→승급), 배치 (그리드 고정 배치)
- **유닛 시스템** — 능력치 / 스탯 구조 / 데미지 공식 / 역할군 5종 / 스킬
- **시너지 (2차원)** — 심상 효과 (이중 컴포넌트: 메인 + 글로벌 버프) + 소환수 효과 (전장 랜덤)
- **전술 지원** — 3라운드마다 3택 1, 새로고침
- **출격 모듈** — 전장 시작 시 자동 적용 (장착 단계 없음)
- **침략자 & 웨이브** — 라인 진군, 보스

### 보상 / 종료

- 본편 클리어 → 무한 모드 진입 선택 → 보상 정산 (배수 적용)

### 메타 시스템 공통

- **재화 체계** — 가챠 재화 / 강화 재화 2종 분리

### 세계관

- 시대·배경 / 심상·소환수 / 톤·스타일 (간략)

## 인터랙션 / 레이아웃 — 자유롭게

구체적인 인터랙션·레이아웃은 **너의 판단에 맡길게.** 위에 적은 두 목표 ("한 눈 파악" + "드릴다운 깊이 보기") 를 달성하는 방향이면 어떤 방식이든 좋아.

웹이 줄 수 있는 자유로운 시각 표현과 유저 반응성을 적극적으로 활용해. 클릭 / 호버 / 스크롤 / 키보드 / 애니메이션 / 레이어링 등 — 어떤 조합이든 목표 달성에 가장 효과적이라고 판단되는 방식을 골라.

종이 문서로는 못 만드는, 웹이기 때문에 가능한 디자인을 보고 싶어.

## 기술 제약

- **단일 HTML 파일**, CSS / JS 인라인.
- **외부 라이브러리 금지** (jQuery, React 등). 순수 HTML / CSS / Vanilla JS 만.
- 폰트는 Google Fonts CDN 1개까지 허용.
- 반응형: 데스크탑 기준 1400px 폭 최적화, 태블릿 (768px+) 까지 정상 표시. 모바일은 best-effort.
- 접근성: 키보드 탐색 가능, 모달은 ESC 닫기, 인터랙티브 요소에 적절한 hover/focus 표시.

## 출력 형식

작업 결과는:

1. 완성된 HTML 파일 (전체 내용)
2. 디자인 선택의 핵심 근거 3-5줄 (왜 이런 인터랙션·레이아웃을 골랐는지 — 너의 판단을 알 수 있게)

## 마지막 당부

- "한 눈에 + 인터랙티브 드릴다운" 이 가장 중요해. 스크롤로 읽게 만들지 마.
- 기획서 문서가 길지만, 페이지에 그대로 옮기는 게 아니라 **첫 화면은 간결하게, 깊이는 인터랙션 안에**.
- 첨부된 기획서에서 누락된 시스템이 없도록 체크. 단 위 "기획서와 달라진 점" 항목은 본 문서가 우선.
- 톤은 끝까지 **귀엽고 발랄하게**. 정보 밀도는 높되 차분한 파스텔 + 둥근 형태로 부드럽게.
