# 구현 이슈 작성 요령

> **구현** 에픽 하위의 Jira 이슈 작성 시 따르는 요령
>
> 본 문서는 업무 분류 체계 중 **구현** 단계의 에픽을 대상으로 한다.

---

## Jira 계층 구조

| Jira 유형 | 역할 | 크기 예시 |
|-----------|------|----------|
| **Epic** | 큰 기능 영역 | [전장] 소환수 |
| **Task** | 에픽 내 기능 묶음 | 소환수 합성 |
| **Sub-task** | 실제 이슈 / 스토리 단위 | 소환수를 드래그하여 합성할 수 있어야 한다 |

---

## Task 작성 요령

### 제목

- 형식: `[에픽 제목] - {기능명}`
- 에픽 내에서 독립적으로 완결되는 기능 단위를 나타낸다.
- 예: `[전장 - 소환수] - 합성 (승급)`, `[전장 - 소환수] - 배치/이동/스왑`, `[전장 - 소환수] - 환원 (판매)`

### 설명

- 이 Task가 다루는 기능의 범위를 간략히 기술한다.
- 하위 작업 목록이 범위를 대신하므로, 길게 쓸 필요 없다.

### 상태

- 하위 이슈 중 기획 누락이 있는 경우, Task 상태를 **"기획 필요"**로 설정한다.

### 댓글

Task 등록 후, 댓글로 해당 Task의 도출 근거를 아래 형식으로 남긴다:

```
## 도출 근거

💻 구현 대상 / 🧩 공통 에픽 소속 / ✅ 기존 구현 완료 / ❓ 기획 누락

### 동작 트리

{기능명}
├─ {상위 동작}
│   ├─ 💻 {말단 동작}
│   ├─ 💻 {말단 동작}
│   └─ ❓ {기획 누락 동작}
├─ {상위 동작}
│   ├─ 💻 {말단 동작}
│   └─ ✅ {기존 구현 완료 동작}
└─ ...

### 보조 관점 체크

**데이터/속성:**
- ❓ {기획 누락 항목}

**UI/정보 표시:**
- 🧩 {공통 에픽 소속 항목}
- ❓ {기획 누락 항목}

**기반 작업:**
- ✅ {기존 구현 완료 항목}
```

## Sub-task 작성 요령

### 유형 선택

| 유형 | 기준 | 제목 형식 |
|------|------|----------|
| **하위 스토리** | 플레이어 관점의 기능·동작 | `~할 수 있어야 한다` / `~하면 ~된다` |
| **하위 작업** | 기술적·내부 작업 | `{대상} {동작}` |

### 제목

- 형식: `[에픽 약칭 > 기능명] - {하위 이슈 제목}`
  - 에픽 제목과 Task의 기능명을 `>`로 연결하여 축약한다.
- **하위 스토리**: 플레이어가 경험하는 동작을 기술한다.
  - 좋은 예: `[전장 - 소환수 > 합성] - 동일 성급 소환수 2개를 합성하면 상위 성급으로 승급한다`
  - 나쁜 예: `MergeSystem 스크립트에서 성급 증가 로직 구현`
- **하위 작업**: 무엇을 만드는지 명확하게 기술한다.
  - 좋은 예: `[전장 - 소환수 > 합성] - 성급별 스탯 테이블 데이터 구조 제작`
  - 나쁜 예: `ScriptableObject 만들기`

### 설명 템플릿

```
## 작업 내용
- 구현해야 할 기능/동작을 구체적으로 기술

## 완료 기준
- 무엇이 되면 이 하위 작업이 끝난 것인지

## 참고
- 상위 이슈, 기획서 등 참조 정보
```

### 상태

- 기획 누락이 있는 Sub-task는 상태를 **"기획 필요"**로 설정한다.

### 작성 원칙

- **기능·동작 단위**로 작성한다. 개발 구현 단위(스크립트, 프리팹, 테스트 등)로 작성하지 않는다.
- 단, 기술적·설계적 단위로 작성하는 것이 적절한 경우에는 그렇게 작성한다. (예: 데이터 구조 설계)
- 구체적인 **산출물**(프리팹, ScriptableObject, 씬 등)은 설명에 명시해도 된다.
- **구현 방법**(특정 클래스명, 메서드 호출, 코드 패턴 등)은 작성하지 않는다.
- **코드 참고**(기존 코드 경로, 파일명, 코드 스니펫 등)는 작성하지 않는다.
