# 1. 유닛

## 1.1 능력치 목록 및 효과

- 모든 유닛(소환수/침략자)이 가지는 능력치 목록이다.
- 본 게임은 **공격 타입을 물리/마법으로 분리하지 않는다.** 능력치 표·데미지 공식이 통합되어 단순화된다.

| 분류 | 표기명 | 효과 |
|------|--------|------|
| 공격 | 공격력 | 일반 공격 및 스킬의 데미지 기반값 |
| 공격 | 공격 속도 | 초당 공격 횟수 |
| 공격 | 사거리 | 적을 감지하고 공격할 수 있는 거리 |
| 공격 | 치명타 확률 | 공격 시 치명타가 발생할 확률 (%) |
| 공격 | 치명타 배율 | 치명타 발생 시 데미지 배율 |
| 공격 | 피해 증가 | 가하는 최종 데미지를 비율로 증가 (%) |
| 방어 | 최대 체력 | 유닛이 가질 수 있는 최대 HP |
| 방어 | 방어력 | 받는 데미지 경감 (%, 곱연산 누적) |
| 방어 | 상태저항력 | 상태이상 저항 |
| 방어 | 피해 감소 | 받는 최종 데미지를 비율로 감소 (%) |
| 기타 | 이동 속도 | 유닛의 이동 속도 |
| 기타 | 쿨타임 감소 | 스킬 재사용 대기시간 감소 (%) |

## 1.2 스탯 구조 (기본/추가/최종, 수정자)

### 1.2.1 스탯 값 구조

- 유닛의 각 능력치는 **기본 스탯**과 **추가 스탯**으로 나뉜다.
- **기본 스탯**: 유닛이 원래 가지고 있는 능력치. 성급(별 등급)에 따라 정해진다.
- **추가 스탯**: 시너지, 전술지원, 버프 등 외부 효과로 인해 늘거나 줄어드는 능력치.
- **최종 스탯**: 기본 + 추가를 합산한 값. 실제 전투에서 이 값이 적용된다.

### 1.2.2 수정자

- 시너지, 버프, 전술지원 등 외부 효과는 **수정자**를 통해 유닛의 스탯을 바꾼다.
- 수정자는 두 종류가 있다.
  - **고정값**: 스탯에 정해진 수치를 더한다. (예: 물리 공격력 +20)
  - **비율**: 스탯을 비율만큼 올린다. (예: 공격 속도 +30%)
- 수정자에는 **출처**가 있다. 예를 들어 특정 버프가 해제되면, 그 버프가 준 수정자만 사라진다.
- 수정자가 붙거나 빠지면 최종 스탯이 자동으로 다시 계산된다.
- 최종 스탯이 바뀌면 유닛의 행동(공격속도, 이동속도, 사거리 등)에 바로 반영된다.

### 1.2.3 증감 분리 곱연산

- 곱연산으로 누적되는 스탯(물리 방어력, 마법 방어력 등)에 적용되는 계산 규칙이다.
- 증가 수정자와 감소 수정자를 **분리하여 각각 곱연산**한 뒤, 감소분을 빼서 최종값을 구한다.

**계산 방법:**
1. 증가 수정자끼리 곱연산하여 **총 증가분**을 구한다.
2. 감소 수정자끼리 곱연산하여 **총 감소분**을 구한다.
3. 최종값 = 총 증가분 - 총 감소분

**예시:** 물리 방어력에 증가 30%, 증가 20%, 감소 10%가 붙은 경우

| 구분 | 수정자 | 곱연산 결과 |
|------|--------|------------|
| 증가 | 30%, 20% | 1 - (1-0.3) × (1-0.2) = 44% |
| 감소 | 10% | 1 - (1-0.1) = 10% |
| **최종** | | **44% - 10% = 34%** |

## 1.3 데미지 계산 공식

### 1.3.1 계산 순서

1. 기본 데미지 결정
2. 치명타 적용
3. 방어력 적용
4. 피해 증가 적용 (공격자)
5. 피해 감소 적용 (피격자)
6. 최종 데미지 확정 (최소 0, 소수점 버림)

### 1.3.2 기본 데미지

- 모든 공격은 단일 **공격력** 능력치에 기반한다 (물리/마법 분리 없음).
- 일반 공격: `기본 데미지 = 공격력`
- 스킬: `기본 데미지 = 공격력 × 스킬 계수`

### 1.3.3 치명타 적용

- 치명타 확률에 따라 치명타가 발생한다.
- 치명타 발생 시 기본 데미지에 치명타 배율을 곱한다.

### 1.3.4 방어력 적용

- 방어력은 피해 감소율(%)에 직접 대응한다. (방어력 30 = 30% 감소)
- 모든 공격에 단일 **방어력** 능력치가 적용된다 (물리/마법 분리 없음).
- 방어력은 **증감 분리 곱연산**(1.2.3 참고)으로 계산한다.
  - 쌓을수록 효율이 점점 줄어들며, 피해를 완전히 0으로 만들 수는 없다.

### 1.3.5 피해 증가 (공격자)

- 공격자의 피해 증가 수치만큼 피해가 늘어난다.

```
데미지 = 데미지 × (1 + 피해 증가)
```

### 1.3.6 피해 감소 (피격자)

- 피격자의 피해 감소 수치만큼 피해가 줄어든다.

```
데미지 = 데미지 × (1 - 피해 감소)
```

### 1.3.7 최종 데미지

- 소수점을 버린다. 최종 데미지는 0 이상이다.

### 1.3.8 전체 공식 요약

```
최종 데미지 = 최소 0, 소수점 버림(
    공격력
    × 치명타 배율 (발생 시)
    × 방어력 곱연산 계수
    × (1 + 피해 증가)
    × (1 - 피해 감소)
)
```

| 역할군 | 컨셉 |

## 1.8 스킬 규칙

본 게임의 소환수 스킬은 다음 원칙을 따른다.

- **존재 의의**: 스킬이 없으면 소환수 간 차별점이 역할군에만 의존해 빈약해진다. 스킬은 소환수의 특색과 최소한의 전략적 선택을 만들기 위해 존재한다.

  - **쿨타임 기반** 으로 동작한다.
  - 한 줄로 설명될 수 있는 단순 효과 위주 (예: 범위 공격, 일정 시간 쉴드 부여 등).

# 2. 소환수

본 게임의 소환수는 모두 **고양이** 이다. 각 소환술사는 고유한 5체의 소환수를 보유한다.

## 2.0 소환수 풀 구성

### 2.0.1 술사당 소환수 / 전체 풀 크기

- 소환술사 1명당 **고유 소환수 5체**.
- 한 전장에 4명의 소환술사가 출전하므로, **한 전장의 총 소환수 풀 = 4 × 5 = 20체**.
- 소환수에는 **레어도(코스트)** 분리가 존재하지 않는다. 모든 소환수가 동일한 마나 비용으로 소환된다. 강도 차이는 합성 성급으로 만든다.

### 2.0.2 역할군 분포 규칙

각 소환술사의 소환수 5체는 1장의 5개 역할군에 분배되며, 다음 규칙을 따른다.

- **기본은 균형 분포** — 5개 역할군에 가능하면 골고루.
- **±1 수준의 작은 바리에이션 허용** — 술사마다 분포가 약간씩 달라 술사의 정체성을 만든다.
- **0체 허용** — 한 술사가 특정 역할군을 보유하지 않을 수 있다 (예: 힐러 0체). 이는 한 게임 활용 10~12체 기준에서 특정 역할군의 비율 과포화를 방지하고, 출전 편성 단계의 결정 깊이를 만든다.
- **극단 분포 금지** — 한 역할군이 3체 이상 몰리는 분포(예: 3 딜러 1 탱커 1 힐러)는 사용하지 않는다.
- 따라서 한 술사의 분포는 **각 역할군 0~2체** 범위에서 결정된다.

#### 분포 예시 (감 잡기용)

| 술사 | 탱커 | 근딜 | 원딜 | 마법사 | 힐러 |
|---|---|---|---|---|---|
| A | 1 | 2 | 1 | 1 | 0 |
| B | 1 | 1 | 1 | 1 | 1 |
| C | 2 | 1 | 1 | 0 | 1 |

#### 출전 편성과의 연결

- 출전 편성은 단순히 좋아하는 캐릭터 4명을 고르는 단계가 아니다.
- 4명의 소환술사 분포를 합산해서 **전체 역할군 분포가 균형 잡히는지** 를 플레이어가 고려하는 *빌딩 결정의 첫 단계* 가 된다.

## 2.1 소환수 라이프 사이클

### 2.1.1 라운드 간 상태 관리

- 라운드(전투 페이즈) 종료 시, 모든 소환수는 **HP가 전부 회복**된 상태로 다음 라운드를 시작한다.
- 다운된 소환수도 라운드 종료 시 **HP 전부 회복 상태로 복귀**한다.
- 스킬 쿨다운은 라운드 종료 시 **초기화**된다.
- 상태이상은 종류에 따라 라운드 종료 시 해제 여부가 다르다. (상태이상별 규칙은 1.9 참고)

## 2.2 소환수 합성(승급) 규칙

### 2.2.1 성급 체계

- 소환수는 **성급(별 등급)**을 가진다.
- 기본 합성: **동일 소환수 3개**를 모으면 **상위 성급으로 승급**한다.
  - 1성 × 3 → 2성
  - 2성 × 3 → 3성
- 강화 합성: 3성 이후에는 **2성 소환수를 하나씩 추가**하여 강화한다.
  - 3성 + 2성 1개 → 3성+1강
  - 3성+1강 + 2성 1개 → 3성+2강
  - 3성+2강 + 2성 1개 → 3성+3강 ...

### 2.2.2 합성 조건

- **동일 소환수 판정**: 같은 종류 + 같은 성급의 소환수 3개가 필요하다.
- **위치 조건**: 대기석과 전장에 있는 소환수 **모두 합성 대상**에 포함된다.
- **전투 중 합성 불가**: 합성은 **정비 페이즈에서만** 발생한다.

### 2.2.3 합성 실행

- 합성 조건이 충족되면 **자동으로 합성**된다.
- 합성 결과 소환수는 **가장 왼쪽 위에 있는 재료의 위치**에 생성된다.
- **연쇄 합성**: 합성 결과로 다시 3개가 충족되면 자동으로 연쇄 합성된다.

### 2.2.4 승급 효과

- 승급하면 소환수의 **기본 스탯이 강화**된다.
- 경우에 따라 **스킬이 강화**된다.

## 2.3 소환수 배치

### 2.3.1 배치 영역 규칙(아군 소환수)

- 아군 소환수는 **그리드 칸**에만 배치 가능하다.
- 아군 소환수는 전투 중 **이동하지 않는다(고정 배치)**.
- 한 칸에는 기본적으로 **1소환수**만 배치 가능하다.
- 배치 가능 소환수 수는 **배치 상한**에 의해 제한된다.

### 2.3.2 전장 배치/교체/합성(정비 페이즈 조작)

- **소환수 배치**: 보유한 소환수를 그리드 칸에 배치한다.
- **소환수 이동/스왑**: 소환수의 위치를 자유롭게 변경할 수 있다.
- **대기 슬롯(벤치) 관리**: 전장에 올리지 않은 소환수는 벤치(대기 공간)에 보관된다.
- **소환수 합성**: 합성 규칙을 만족하면 자동으로 합성된다.

# 3. 소환 터미널

## 3.1 터미널 구성(UI 레이아웃)

- 소환 슬롯
- 현재 마나
- 다음 마나 수입
- 재스캔 버튼
- 스캔 잠금 버튼
- 배치 상한 증가 버튼
- 터미널 열기 / 닫기

## 3.2 소환(구매)

- 제시되는 소환수 목록은 4개이다.
- 소환수를 소환하기 위해서는 마나가 필요하다
  - 1성 : 1마나
  - 2성 : 3마나
  - 3성 : 9마나
- 소환수가 소환되는 경우 대기석에 소환된다.
  - 대기석에 자리가 없는 경우 터치하여도 소환은 불가능하다.

- 배치 상한 증가에 따라서 소환수가 2성 또는 3성으로 등장하는 확률이 생긴다

## 3.3 스캔(리롤)

- 스캔 : 터미널에 등장하는 소환수 목록을 재구성 하는 것 (=리롤)
  - 스캔 시 출전 편성한 소환술사가 소환할 수 있는 소환수들 중에 4개가 선택되어 등장한다.
  - 소환수 풀은 소환술사당 **5체**, 총 **20체** (4명 × 5체)로 구성된다.
- 매 라운드 시작 시 터미널이 다시 스캔된다.
- 재스캔 버튼을 눌러 다시 스캔할 수 있다.
  - 재스캔 비용은 마나로 지불한다 (잠정 수치).

## 3.4 스캔 잠금

- 스캔 잠금 : 라운드 시작시에 터미널 재스캔을 막는다
- 스캔 잠금시에 라운드가 시작되면 재스캔이 진행되지 않고, 스캔 잠금이 풀린다

## 3.5 환원(판매)

- 환원 : 소환한 소환수를 소환 해제하여 마나를 다시 얻는 것
- 소환한 소환수를 상점 영역으로 드래그 앤 드롭하여 환원이 가능하다
- 환원 규칙
  - 1성 : 1마나
  - 2성 : 2마나 (제작 비용 3 - 1)
  - 3성 : 8마나 (제작 비용 9 - 1)
  - 3성+N강 : 8 + N마나 (강화 1회당 +1마나)
  - 마켓에서 고성급으로 등장한 소환수도 동일 규칙 적용


## 3.6 배치 상한 증가

- 배치 상한 증가 : 전장에 배치 할 수 있는 소환수를 늘리는 기능
- 초기 배치 상한 : 3개
- 배치 상한 증가 가격 규칙 
  - (4) 4 - (5) 6 - (6)10 - (7)15  - (8)20 - (9)30 - (10)45 - (11)70 - (12)100


## 3.7 마나 충전

- 매 라운드 시작시 마나가 지급된다.
- 마나 지급 계산식 : 기본 지급 (1 ~  10 : 라운드에 따라) + 이자 (0 ~ 3)

# 4. 침략자 및 라운드

## 4.1. 라운드

- 각 라운드마다 침략자의 무리가 등장한다.
- 침략자를 모두 처치하면 다음 라운드의 정비 페이즈가 시작된다.
- 캠페인의 한 전장은 **본편 라운드 + 무한 모드 라운드** 의 2단 구조를 가진다.

### 4.1.1 본편

- 캠페인의 정규 진행 구간. 라운드 수는 한 전장의 목표 플레이 타임(약 20분)에 맞춰 조정된다.
- 본편의 모든 라운드를 클리어하면 **본편 클리어 메시지** 가 표시되며, 무한 모드 진입 여부를 선택할 수 있다.
- 본편 단계의 패배 허용 횟수: **3회**.

### 4.1.2 무한 모드 (구 엑스트라 라운드)

- 본편 클리어 후 플레이어가 선택하여 진입하는 추가 라운드 구간.
- **무한 모드 진입 시 잔여 패배 횟수가 0으로 리셋된다.** 1회 패배 시 즉시 전장 종료.
- 라운드를 거듭할수록 난이도가 **가파르게** 상승. 본편 후반보다 명백히 어려운 수준에서 시작한다.
- 전역의 마지막 전장 등 특수 전장에서는 무한 모드가 비활성화될 수 있다.

### 4.1.3 보상

- 한 전장의 최종 보상 = **본편 클리어 보상 × 무한 모드 배수**
- 무한 모드 1라운드 클리어마다 **+0.1 배수**.
- 예: 본편만 클리어 = ×1.0 / 무한 5라운드 = ×1.5 / 무한 10라운드 = ×2.0.

## 4.2. 페이즈

### 4.2.1 페이즈의 구성

- 전장은 **정비 페이즈(준비)**와 **전투 페이즈(웨이브)**가 교차한다.
- 정비 페이즈에서 정비를 마치고 전투 시작을 터치하면 전투 페이즈가 시작한다

### 4.2.2 정비 페이즈 구성(가능 행동 요약)

- **소환 터미널(상점) 이용** → *3. 소환 터미널*
- **전장 배치/교체/합성** → *2. 소환수*
- **전술 지원 선택** → *6. 전술지원*
- **다음 웨이브 정보 확인(웨이브 프리뷰)** → *4.3 / 4.5.8*
- **전투 시작**: 정비가 끝나면 전투 시작을 터치해 전투 페이즈로 진입한다.

### 4.2.3. 전투 페이즈 구성

- **전투 시작**: 정비 페이즈에서 전투 시작을 터치하면 웨이브가 시작된다.
- **적 스폰**: 웨이브 구성에 따라 레인별로 적이 생성되어 진군한다.
- **소환수 자동 전투**: 배치된 소환수는 이동 없이 제자리에서 공격/스킬을 사용한다.
- **타겟팅/스킬 발동**: 소환수의 공격 및 스킬은 각 소환수의 규칙에 따라 자동으로 처리된다.
- **시너지 적용**: 활성화된 시너지 효과가 전투 중 계속 적용된다.
- **전술 지원 적용**: 선택한 전술 지원 효과가 전투 중 적용된다.

#### 4.2.3.1. 전투 중 제한 사항

- **배치 변경 불가**: 전투 페이즈 동안 소환수의 위치 변경/교체는 불가능하다.
- **소환 터미널 이용**: 전투 중에 소환, 리롤, 배치 상한 증가는 가능하지만, 전장에 배치는 불가능하다

#### 4.2.3.2. 전투 페이즈 종료 조건

- **전투 승리** : **적 전멸**: 등장하는 모든 적을 처치하면 전투 페이즈가 종료된다.
- **전투 패배** : 소환술사중 한명이라도 다운이 되면 전투 패배

#### 4.2.3.3. 전투 페이즈 종료 및 다음 단계

- **다음 정비 페이즈 진입**: 정비 페이즈로 전환된다.

## 4.3 전장 승리 / 패배 조건

### 4.3.1 승리

- **본편 클리어 (=임무 완수)**: 본편의 모든 라운드를 클리어하면 임무가 완수된다.
  - 본편 클리어 직후, 플레이어는 **무한 모드 진입** 또는 **전장 종료** 중 하나를 선택한다.
  - 전장 종료 선택 시 정산 화면이 표시된다.
  - 정산 화면에는 전투 정보 및 보상 (×1.0 또는 무한 모드 진행분 배수 적용)이 표시된다.

### 4.3.2 패배

- **본편 패배**: 전투 패배가 누적 3회가 되면 전장 패배 (=임무 실패).
- **무한 모드 패배**: 무한 모드 진입 시 잔여 패배 횟수가 0으로 리셋되므로, 1회 패배 시 즉시 전장 종료. 단 이 경우는 "실패" 가 아니라 **무한 모드 도달 라운드까지의 보상 배수를 챙기고 정산**으로 처리된다 (이미 본편은 클리어한 상태이기 때문).
- 임무 실패 시 재도전 또는 로비로 나갈 수 있다.


## 4.5 침략자 및 침략자 웨이브

### 4.5.1 등장 규칙

- 침략자는 **라인 오른쪽 끝**에 **무작위로 출현**한다.
- 침략자가 등장하는 **라인은 무작위**지만, **가능한 균일하게 등장**한다.
  - 한곳에 몰려 등장하지 않는다.

- 침략자는 **한번에 등장하지 않고**, 라운드마다 정해진 규칙에 따라 **순차적으로 등장**한다.
- 특별한 전장 또는 라운드의 경우 **보스가 등장**할 수 있다.

### 4.5.3 레벨 디자인(캠페인/방어전)

- 전장 레벨 디자인 문서 참고

### 4.5.4 웨이브 정보

- 정비 페이즈에서 **이번 라운드에 등장할 침략자를 미리 확인**할 수 있다.
  - 등장하는 침략자의 **정보와 숫자**를 표시해준다.
- 전투 페이즈에는 **남은 침략자 수**를 UI에 표시해준다.

# 5. 시너지

## 5.1 시너지 개요

본 게임의 시너지는 **2차원 구조**로 구성된다.

| 시너지 종류 | 결정 시점 | 결정 방식 |
|---|---|---|
| **심상 효과** | 출전 편성 시 | 소환술사 선택에 의해 고정 |
| **소환수 효과** | 전장 진입 시 | 매 전장 랜덤 부여 |

각 소환수는 전장 진입 시 **심상 효과 1개 + 소환수 효과 1개** 를 보유한 상태로 시작한다.

### 5.1.1 카운트 규칙

- 카운트 대상은 **전장에 배치된 서로 다른 종류의 소환수**이다. 동일 소환수는 중복 카운트하지 않는다.
- 전투 중 소환수가 **다운되어도 시너지 카운트는 유지** 된다.
- 소환수 **배치, 제거** 시 시너지를 재계산한다.

### 5.1.2 임계치

- **심상 효과**: 2 / 4 / 6 / 8 (4단계)
- **소환수 효과**: 2 / 3 / 4 (3단계)

### 5.1.3 효과 적용 대상

- 시너지 효과는 **해당 시너지를 가진 소환수에게만** 적용되는 것을 기본으로 한다. 단 심상 효과의 글로벌 버프 컴포넌트는 다른 심상에게도 적용된다 (5.2.1 참고).

## 5.2 심상 효과 (소환술사 귀속)

- 각 소환술사는 고유한 **심상 효과** 를 가진다.
- 소환술사의 소환수는 해당 소환술사의 심상 효과를 **자동으로 보유** 한다.
- 같은 심상(고양이 품종)을 가진 소환술사라도 심상 효과의 구체 내용은 서로 다를 수 있다.
- 소환술사의 **육성 상태** (레벨, 승급) 에 따라 심상 효과가 **강화** 될 수 있다.

### 5.2.1 심상 효과의 이중 컴포넌트 구조

각 심상 효과는 다음 두 가지 컴포넌트를 함께 가진다.

| 컴포넌트 | 적용 대상 | 역할 |
|---|---|---|
| **메인 효과** | 같은 심상의 소환수 | 본격 시너지 — 임계치가 높아질수록 (4/6/8) 강해진다 |
| **글로벌 버프** | 전체 소환수 (다른 심상 포함) | 약한 전체 강화 — 1티어(2 임계치) 만 켜도 발휘 |

**이 이중 구조의 의도**: 한 전장에서 플레이어가 주로 사용하는 심상은 2개로 수렴하는 경우가 많지만, 나머지 2 소환술사도 **글로벌 버프** 를 통해 전체 팀에 기여한다. 즉 출전 편성한 4명의 소환술사 모두에게 의미가 부여되어 "버림패 술사" 가 발생하지 않는다.

### 5.2.2 심상과 소환수 풀

- 심상은 소환술사가 어떤 형태의 소환수(고양이 품종)를 사용하는지를 결정하는 기준이다.
- 같은 심상의 소환수 풀은 일부 겹칠 수 있으나, 소환술사별로 완전히 동일할 필요는 없다.
  - 예: 스코티시폴드 심상 소환술사 A와 B가 일부 같은 소환수를 공유하면서, 일부는 다를 수 있다.
- **출전 편성에서는 같은 심상을 가진 소환술사를 중복 선택할 수 없다.**
  - 동일한 심상 소환수가 소환 터미널에 중복 등장하는 상황을 방지하기 위한 제한이다.

## 5.3 소환수 효과 (전장 랜덤 부여)

- 전장 시작 시 풀에서 **5개의 소환수 효과** 가 선택되며, 20체의 소환수에 랜덤으로 분배된다.
- 동일 소환수라도 전장마다 다른 효과를 가질 수 있다.
- 효과 부여 시 소환수의 **역할군에 따라 부여 확률이 보정** 된다 (탱커용 효과는 탱커에게, 딜러용 효과는 딜러에게 부여될 확률이 높다).
- 구체적인 효과 카탈로그·부여 확률 수치는 **레벨디자인 및 스케일링 / 콘텐츠 디자인** 단계에서 정의한다.

### 왜 랜덤 부여인가

- 매 전장마다 같은 소환수도 다른 효과를 가져 **반복 플레이의 다양성** 을 제공한다.
- 소환술사마다 소환수 구성이 다른 상황에서 "어느 소환수가 어떤 효과를 가지는가" 까지 고정해두면 플레이어의 학습 부담이 과도해진다 — 랜덤 부여가 이 부담을 자연스럽게 줄인다.

## 5.4 시너지 정보 UI

- 전장마다 소환수 효과 분배가 달라지므로, 플레이어가 현재 전장의 **심상 효과 + 소환수 효과 조합을 한눈에 파악** 할 수 있는 UI가 필요하다.
- UI에는 다음을 표시한다:
  - 출전 4 소환술사의 심상 효과 (메인 + 글로벌 버프)
  - 이번 전장의 5개 소환수 효과와 임계치 진행 상태
  - 각 소환수가 보유한 효과 (효과별 어느 소환수에 부여되었는지)

# 6. 전술지원

## 6.0 위상 — 팀 강화의 핵심 축

전술 지원은 **본 게임에서 팀의 출력을 결정하는 핵심 강화 축** 이다.

- 시너지 (심상 효과 + 소환수 효과) 가 팀의 **방향** 을 결정한다면, 전술 지원은 그 방향의 **출력 배율** 을 결정한다.
- 본 게임은 TFT 의 아이템을 별도로 두지 않는 대신, 그 역할까지 전술 지원이 흡수한다.
- 전술 지원의 알고리즘은 플레이어가 맞추고 있는 시너지·이전 선택에 따라 보정되어, 시너지가 빌드 방향을 결정하면 그에 어울리는 전술 지원이 등장하기 쉽도록 작동한다 (6.2 참고).

## 6.1 등장 타이밍

- 본편 진행 중 **일정 라운드마다 1회** 등장한다 (잠정: 약 3라운드마다).
- 정확한 빈도는 한 전장 목표 시간 (본편 약 20분) 과 카타르시스 구간 확보를 고려해 밸런싱 시 조정한다.

## 6.2 제시 로직

- 등장 시 **3개의 전술 지원 선택지** 가 제시되며, 플레이어는 **1개를 선택** 한다.
- **새로고침** 가능 — 마음에 들지 않는 3개를 다시 뽑을 수 있다.
  - 새로고침 비용은 **마나** 로 지불한다.
  - 한 번 새로고침할 때마다 비용이 **2배씩 증가** 한다 (무한 새로고침 방지).
- **알고리즘 보정** — 등장 후보는 다음 인자에 따라 가중치 보정된다:
  - 현재 플레이어가 맞추고 있는 시너지 (심상 효과 / 소환수 효과)
  - 이전에 선택한 전술 지원
  - → 빌드에 어울리는 전술 지원이 등장하기 쉽다.

## 6.3 효과 카테고리

전술 지원의 효과는 인게임 시스템 전반에 작용한다. 다음 카테고리들을 포함한다.

- **소환수 전체 강화**: 모든 소환수의 공격력/체력 등 능력치 강화.
- **특정 심상 강화**: 출전한 4 심상 중 하나의 효과를 추가 강화.
- **특정 소환수 효과 강화**: 이번 전장에 등장한 5개 효과 중 하나를 추가 강화.
- **자원 운용**: 마나 추가, 재스캔 비용 감소, 적 처치 보상 강화 등.
- **그 외 인게임 시스템 전반에 영향을 주는 효과**.

구체 효과 카탈로그는 콘텐츠 디자인 단계에서 정의한다.

## 6.4 누적 / 중첩 / 상한

- 선택한 전술 지원은 해당 전장이 끝날 때까지 누적 적용된다 (인게임 동안에만 유효, 전장 종료 시 사라진다).
- 동일 전술 지원이 중복 선택될 수 있는지, 중첩 시 효과 등은 콘텐츠 디자인 단계에서 정의.

## 6.5 수집 / 모집

- 전술 지원은 **가챠 재화** 로 모집할 수 있다 (소환술사 모집과 동일 재화, 잠정 별도 배너).
- 가챠 외에도 캠페인 진행 보상 등 조건부 해금 경로가 존재한다.
- 자세한 강화 시스템 구조는 [[08-2 육성 스케일링]] 참고.

# 7. 출격모듈

## 7.0 위상

출격 모듈은 **인게임 전반에 영향을 주는 영구 강화 시스템** 이다.

- TFT 에는 매칭되는 요소가 없으며, 스타크래프트2 유즈맵 (랜덤 조합 디펜스 류) 의 영구 강화 시스템이 모티프.
- 두 가지 존재 이유:
  1. **인게임 재화의 소비처** — 캠페인 진행으로 얻는 재화의 메타 진행감 형성.
  2. **덱 강화의 한 축** — 가챠·소환술사 육성·전술 지원과 다른 차원의 강화 축으로 작동.

## 7.1 작동 방식

- 출격 모듈은 **패시브** (상시 효과) 또는 **단발성 액티브** (특정 조건/타이밍에 발동) 형태로 구성된다.
- 단발성 액티브의 발동 방식은 모듈마다 다르다 (자동 트리거 / 조건부 자동 위주, 수동 발동도 가능).
- 한 번 강화하면 한 전장 안에서 **영구적으로 작동** 한다.

## 7.2 슬롯 / 장착 규칙

- **별도 슬롯 / 장착 단계 없음.** 보유한 **모든 모듈** 이 한 전장 시작 시 **자동 적용** 된다.
- 출격 모듈은 빌드 결정의 축에는 포함되지 않는다 — 출전 편성 단계의 결정 부담을 덜어준다.
- 빌드 결정의 깊이는 **시너지(출전 편성) + 전술 지원(인게임 중)** 에 집중시킨다.

## 7.3 효과 카테고리

다음과 같은 효과들을 포함한다.

- 시작 시 마나 추가 지급
- 시작 시 재스캔 횟수 추가
- 적 처치 시 일정 확률로 추가 마나 획득
- 그 외 인게임 시스템 전반에 영구 영향을 주는 효과

구체 효과 카탈로그는 콘텐츠 디자인 단계에서 정의한다.

## 7.4 강화

- 출격 모듈은 **강화 재화** 로 강화한다.
- 자세한 강화 시스템 구조는 [[08-2 육성 스케일링]] 참고.
