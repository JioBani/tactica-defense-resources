# 작업지시서

> 이 문서는 Claude Code 세션이 개발 작업 시 따라야 할 루틴이다.
> 참고 문서: `git.md`, `develop.md`, `CLAUDE.md`

---

## 작업 파일 구조

브랜치(일감)별로 작업 문서를 분리하여 관리한다.

```
.claude/work/
└── TACD-{번호}/
    ├── 작업정의.md       # 작업 내용 및 완료 정의 정리
    ├── 상세설계.md       # 해당 일감의 설계 명세
    └── 작업현황판.md     # 해당 일감의 구현 체크리스트
```

---

## 작업 루틴

### 1. 작업 내용 확인

1. 사용자가 지정한 Jira 하위일감의 작업 내용과 완료 목표 확인
2. 해당 일감의 브랜치 생성 및 전환 (`TACD-{번호}`)
3. 작업 디렉토리 생성 (`.claude/work/TACD-{번호}/`)

### 2. 작업정의 작성

1. Jira 일감의 작업 내용과 완료 목표 확인
2. 기획서(`Document/기획서/`)에서 관련 요구사항 수집
3. 기존 코드를 분석하여 현재 구조 파악 (`CLAUDE.md` 참고)
4. `.claude/work/TACD-{번호}/작업정의.md` 작성
   - 작업해야 할 내용 정리
   - 완료 정의 (무엇이 되면 이 일감이 끝난 것인지)
   - 관련 기획서 항목 및 기존 코드 참조
5. 사용자 확인 후 다음 단계로 진행

### 3. 상세설계 작성

1. 작업정의를 바탕으로 `.claude/work/TACD-{번호}/상세설계.md` 작성
   - 해당 일감에서 만들거나 수정할 대상에 한정
   - 클래스/메서드 명세, 기존 코드와의 연결점, 데이터 흐름 등
2. 사용자 확인 후 다음 단계로 진행

### 4. 작업현황판 작성

1. 상세설계에서 구현 항목과 테스트 항목을 도출
2. `.claude/work/TACD-{번호}/작업현황판.md`에 체크리스트 작성
   - `[ ]` 미완료 / `[~]` 진행 중 / `[x]` 완료
3. 사용자 확인 후 다음 단계로 진행

### 5. 구현

아래 사이클을 작업 단위마다 반복한다.

```
5.1 작업현황판에서 다음 항목 선택 → [~] 표시
5.2 코드 구현
5.3 Unity 컴파일 확인 (refresh_unity + read_console)
5.4 유닛 테스트 작성
5.5 유닛 테스트 실행
5.6 실패 시 수정 → 5.5로 복귀
5.7 통과 시 작업현황판에서 [x] 표시 → 다음 항목으로 이동
```

### 6. 완료

1. `git.md` 형식에 따라 커밋
2. 구현 결과가 상세설계와 다른 부분이 있으면 상세설계 수정
3. Jira 일감 상태 변경

---

## 작업 규칙

- **섹션 단위(예: 2, 3, 4, 5) 작업 완료 시마다 사용자에게 확인을 받는다.**
- 확인 없이 다음 섹션으로 넘어가지 않는다.
