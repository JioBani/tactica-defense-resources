# Jira 작업 지침

> 이 문서는 Claude Code 세션이 Jira 일감을 생성·관리할 때 따라야 할 지침이다.

## 하위 작업 분류 기준

- 하위 작업은 **기능·동작** 단위로 나누는 것을 우선한다.
- 개발 구현 단위(스크립트, 프리팹, 테스트 등)로 나누지 않는다.
- 단, 기술적·설계적 단위로 나누는 것이 적절한 경우에는 그렇게 나눈다.
  - 예: DB 설계, 공통 모듈 작성 등

### 좋은 예

```
TACD-191 [전장 - 유닛] 능력치 정보 UI
├── 능력치 정보 패널
├── 유닛 선택/해제로 패널 열기·닫기
└── 상태이상 표시
```

### 나쁜 예

```
TACD-191 [전장 - 유닛] 능력치 정보 UI
├── StatInfoPanel UI 프리팹 생성
├── 유닛 정보 데이터 바인딩 스크립트
├── 스탯 목록 동적 생성 로직
└── 유닛 클릭 이벤트 연동
```

## 일감 설명 작성 기준

- 구체적인 **산출물**(프리팹, ScriptableObject, 씬 등)은 명시해도 된다.
- 구현 방법(특정 클래스명, 메서드 호출, 코드 패턴 등)은 작성하지 않는다.

## 하위 작업 설명 템플릿

```
## 작업 내용
- 구현해야 할 기능/동작을 구체적으로 기술

## 완료 기준
- 무엇이 되면 이 하위 작업이 끝난 것인지

## 참고
- 관련 기존 코드, 상위 일감, 기획서 등 참조 정보
```
