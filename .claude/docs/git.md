# Git 커밋 메시지 작성 지침

커밋 시 아래 형식을 반드시 따른다.

## 형식

```
<branch 명> <type>: <커밋 제목>

## 전체 목표
<내용>

## 작업 필요 이유
<내용>

## 작업 세부 내용
<내용>
```

---

## 각 항목 설명

### 커밋 제목

- 이번 커밋에서 수행한 작업을 한 줄로 요약한다.
- conventional commit 형식을 따른다. (feat / fix / refactor / chore / docs / test 등)
- 예: `SEL-0000 feat: TODO 삭제 API 연결`

### 전체 목표

- 이 커밋이 속한 **상위 작업의 최종 목표**를 작성한다.
- 하나의 기능은 여러 커밋으로 나뉘어 진행된다. 이 항목은 개별 커밋이 아닌, 그 커밋들이 모여 완성하려는 큰 그림을 설명한다.
- 같은 기능에 속하는 커밋들은 모두 동일한 전체 목표를 가져야 한다.
- 한 줄로 간결하게 작성한다.
- 예: `TODO 리스트 삭제 기능 추가`
- 현재 전체 목표가 명확하지 않으면 사용자에게 질문한다

### 작업 필요 이유

- **이번 커밋**이 왜 필요한지를 설명한다.
- 전체 목표 달성 과정에서 이 커밋이 어떤 역할을 하는지, 어떤 문제를 해결하는지를 작성한다.
- 전체 목표와 중복되지 않도록 주의한다. 전체 목표는 "무엇을 만들 것인가"이고, 작업 필요 이유는 "이 커밋이 왜 지금 필요한가"이다.
- 1~3문장으로 작성한다.
- 예: `삭제 버튼 클릭 시 서버에 반영되도록 프론트엔드와 백엔드 API 연결이 필요.`

### 작업 세부 내용

- 이번 커밋에서 변경된 파일과 그 변경 내용을 구체적으로 나열한다.
- 각 항목은 `파일 경로 (변경 유형): 변경 내용 설명` 형식으로 작성한다.
- 변경 유형은 신규 / 수정 / 삭제 중 하나를 사용한다.
- 단순히 "어떤 파일을 수정했다"가 아니라, "왜 그렇게 변경했는지"를 포함한다.
- 예:
  - `src/api/todo.ts (수정): deleteTodo() 함수 추가. RESTful 규칙에 맞춰 DELETE 메서드 사용`
  - `src/hooks/useTodoDelete.ts (신규): 삭제 mutation hook. 낙관적 업데이트는 미포함`

## 작성 예시

```
feat: TODO 삭제 API 연결

## 전체 목표
TODO 리스트 삭제 기능 추가

## 작업 필요 이유
삭제 버튼 클릭 시 서버에 반영되도록 프론트엔드와 백엔드 API 연결이 필요
현재는 UI에서만 제거되고 새로고침하면 다시 나타나는 상태

## 작업 세부 내용
- src/api/todo.ts (수정): deleteTodo() 함수 추가. RESTful 규칙에 맞춰 DELETE /api/todos/:id 호출
- src/hooks/useTodoDelete.ts (신규): 삭제 mutation hook. useMutation 기반으로 에러 핸들링 포함
- src/components/TodoItem.tsx (수정): 삭제 버튼 onClick에 useTodoDelete 연결
```





## 추가 규칙

- 커밋은 작은 단위별로 쪼개서 작성한다
- `Co-Authored-By` 등 AI 작성 관련 내용은 생략한다
- 커밋은 한국어로 작성한다
