# Git 커밋 규칙

## 커밋 메세지 형식

```
[브랜치이름] 유형 : 작업 내용
```

### 유형
- `feat` : 새로운 기능 추가
- `fix` : 버그 수정
- `test` : 테스트 추가/수정
- `refactor` : 리팩토링
- `docs` : 문서 수정
- `chore` : 기타 작업

### 규칙

1. 커밋 메세지 앞에 `[브랜치이름]`을 붙인다
2. 브랜치 이름 뒤에 작업 유형을 작성한다 (feat, fix, test 등)
3. 커밋은 작업 단위로 분리한다 (하나의 커밋에 여러 작업을 섞지 않는다)
4. Co-Authored-By 등 AI로 작성했다는 표시를 생략한다
5. 여러 줄로 작성할 경우, 제목 작성 후 빈 줄을 두고 개괄식으로 작성한다

### 예시

한 줄:
```
[TACD-10] feat : 스킬 쿨다운 시스템 추가
```

여러 줄:
```
[TACD-10] feat : 스킬 쿨다운 시스템 추가

- 쿨다운 타이머 구현
- UI 쿨다운 표시 연동
```
