# 구현 이슈 분해

에픽 티켓 번호: $ARGUMENTS

## 참조 문서

먼저 `.claude/work/decompose/$ARGUMENTS/` 디렉토리에 기존 기록 파일이 있는지 확인한다.
- 파일이 있으면: 기록 파일을 읽고, 마지막 진행 상태부터 이어서 수행한다.
- 파일이 없으면: 아래 참조 문서를 읽고 처음부터 수행한다.

아래 문서를 읽고 숙지한 뒤, 워크플로우를 순서대로 수행한다:

1. `Document/프로젝트 관리/구현 이슈 분해 방법론.md` - 분해 방법론 (워크플로우)
2. `Document/프로젝트 관리/구현 이슈 작성 요령.md` - 이슈 작성 형식
3. `Document/프로젝트 관리/업무 분류 체계.md` - 업무 분류 정의
4. `.claude/docs/jira.md` - Jira 등록 지침

## 작업 파일

단계별 결과를 아래 파일에 기록하며 진행한다:

```
.claude/work/decompose/{에픽 티켓 번호}/
├── 분해결과.md                    # Phase A: Epic → Task 결과
├── {Task 티켓 번호}.md            # Phase B: Task Sub-task 분해 (Jira 등록 후 생성)
├── {Task 티켓 번호}.md
└── ...
```

### 기록 규칙

- **단계 완료 시마다** 결과를 파일에 기록한다
- 사용자 검증 후 수정된 내용은 해당 단계에 덮어쓴다
- 컨텍스트 컴팩트 발생 시, 해당 파일을 읽어 이전 결과를 복원한다

### 분해결과.md 형식 (Phase A)

```markdown
# 구현 이슈 분해: {에픽 티켓 번호} - {에픽 제목}

> 현재 Phase: A / B
> 현재 작업: {상태 설명}

---

## 1단계: 기획서 파악 ✅

**참조 기획서**: {경로}
**핵심 요약**: ...

## 2단계: 에픽 경계 확인 ✅

**대상 에픽**: {티켓 번호} - {제목}
**인접 에픽**: ...
**경계 판단**: ...

## 3단계: Task 도출 ✅

### 상위 동작 트리
(트리 구조)

### Task 목록
(경계 판단 결과)

## 4단계: Task Jira 등록 ✅

| # | Task | Jira | Sub-task 분해 |
|---|------|------|--------------|
| 1 | {기능명} | TACD-XXX | ✅ / 🔄 / - |
| 2 | {기능명} | TACD-XXX | - |
```

### Task별 파일 형식 (Phase B)

파일명: `{Task 티켓 번호}.md` (4단계에서 Jira 등록 후 생성)

```markdown
# {Task 티켓 번호}: {Task 제목}

> 상태: 진행중 / 사용자 검증 대기 / 완료

---

## 5단계: Sub-task 도출 {✅/🔄}

### 5-1. 동작 트리 상세화
(말단 노드까지 분해한 트리)

### 5-2. 보조 관점 체크
(데이터/속성, UI/정보 표시, 기반 작업)

### 5-3. 업무 분류 및 분류
(전체 항목 + 분류 결과)

## 6단계: 검증 {✅/🔄}

## 7단계: 이슈 목록 정리 {✅/🔄}

(구현 이슈 분해 방법론.md 7단계 형식으로 작성)

## 8단계: Sub-task Jira 등록 {✅/🔄}

(등록된 이슈 번호 기록)
```

## 워크플로우 요약

### Phase A: Epic → Task
1단계: 기획서 파악 → 에픽 관련 기획서를 읽는다
2단계: 에픽 경계 확인 → Jira 에픽 목록으로 범위를 구분한다
3단계: Task 도출 → 상위 동작 트리 + 경계 판단 → **사용자 검증**
4단계: Task Jira 등록

### Phase B: Task → Sub-task (Task마다 반복)
5단계: Sub-task 도출 → 동작 트리 상세화 + 보조 관점 체크 + 업무 분류 → **사용자 검증**
6단계: 검증 → 누락/중복 확인 → **사용자 검증**
7단계: 이슈 목록 정리 → 형식에 맞춰 정리하여 보여주기 → **사용자 검증**
8단계: Sub-task Jira 등록

## 주의사항

- 각 검증 단계에서 반드시 사용자 확인을 받은 후 다음 단계로 진행한다.
- 사용자가 수정을 요청하면 해당 단계를 수정한 뒤 다시 검증을 받는다.
- 기획서는 `Document/기획서/` 폴더에서 찾는다.
- 이슈 작성 시 `구현 이슈 작성 요령.md`의 형식을 따른다.
- Phase B 진행 시, `분해결과.md`와 해당 Task 파일만 읽으면 된다.
