## 6. TACD-196: 유닛 선택/해제로 패널 열기·닫기

### 이벤트 정의

```
OnObjectSelectedEvent : IGameEvent  (Common/Scripts/GlobalEventBus/)
└── GameObject SelectedObject  // 선택된 오브젝트 (범용)
```

### Selectable2D.cs (Common/Scripts/Selectable/)

- Unit.prefab 루트에 컴포넌트로 부착 (모든 유닛 변형이 자동 상속)
- `IPointerClickHandler` 구현
- `OnPointerClick`: 좌클릭 시 `GlobalEventBus.Publish(new OnObjectSelectedEvent(gameObject))`
- `EventTrigger`(IDragHandler)가 같은 GO에 있으므로 드래그 시 OnPointerClick 호출 안 됨 (Unity EventSystem 자동 구분)

### StatInfoPanel.cs — 이벤트 구독

- `Awake`: `GlobalEventBus.Subscribe<OnObjectSelectedEvent>(OnObjectSelected)` + `gameObject.SetActive(false)`
- `OnDestroy`: `GlobalEventBus.Unsubscribe<OnObjectSelectedEvent>(OnObjectSelected)`
- `OnObjectSelected` → `evt.SelectedObject.TryGetComponent<Unit>(out var unit)` 성공 시 `Show(unit)`
- CloseButton 클릭 → `Hide()`
- 주의: OnEnable/OnDisable이 아닌 Awake/OnDestroy 사용 — 패널이 비활성화 상태에서도 이벤트 수신 필요

### 패널 열기/닫기 흐름

```
유닛 클릭
→ Selectable2D.OnPointerClick (유닛 GO 자체에서 감지)
→ GlobalEventBus.Publish(OnObjectSelectedEvent(gameObject))
→ StatInfoPanel.OnObjectSelected: TryGetComponent<Unit> → Show(unit)
→ 유닛 정보 + 스탯 바인딩 + 패널 활성화

X 버튼 클릭
→ CloseButton.onClick → StatInfoPanel.Hide()
→ 동적 셀 Destroy + 패널 비활성화
```

## 7. TACD-198: 유닛 위치 기반 패널 배치

### 요구사항

- 유닛의 **월드 위치** 기준 오른쪽에 패널을 표시한다 (클릭 지점이 아닌 유닛 위치 기준)
- 패널이 화면 **하단**으로 넘어가는 경우, 위로 밀어올려서 온전히 보이도록 한다

### StatInfoPanel.cs — 위치 계산

- `Camera.main`을 Awake에서 캐싱하여 `_camera` 필드에 저장
- `Show(Unit unit)` 호출 시 유닛의 월드 좌표를 스크린 좌표로 변환:
  - `_camera.WorldToScreenPoint(unit.transform.position)` → 스크린 좌표
- 변환된 스크린 좌표를 `PositionAt()`에 전달 (기존 Canvas 로컬 좌표 변환 + 하단 잘림 보정 로직 재사용)

### 위치 계산 로직

```
1. 유닛 월드 좌표 → 스크린 좌표 변환
   screenPos = Camera.main.WorldToScreenPoint(unit.transform.position)

2. 스크린 좌표 → Canvas 로컬 좌표 변환
   RectTransformUtility.ScreenPointToLocalPointInRectangle(canvasRect, screenPos, null, out localPos)

3. 기본 배치: 유닛 위치의 오른쪽
   X = localPos.x + offset
   Y = localPos.y

4. 하단 잘림 보정
   panelBottom = Y - panelHeight
   if panelBottom < 0:
       Y = panelHeight

5. anchoredPosition = (X, Y)
```

### 데이터 흐름

```
유닛 클릭
→ Selectable2D.OnPointerClick(eventData)
→ GlobalEventBus.Publish(OnObjectSelectedEvent(gameObject, eventData.position))
→ StatInfoPanel.OnObjectSelected(evt)
   ├── TryGetComponent<Unit> → Show(unit)
   └── _camera.WorldToScreenPoint(unit.transform.position) → PositionAt(screenPos)
       ├── ScreenPoint → Canvas LocalPoint 변환
       ├── X: 유닛 위치 + 오프셋 (오른쪽)
       ├── Y: 유닛 높이 (하단 잘림 시 위로 보정)
       └── anchoredPosition 설정
```

## 8. 빈 화면 터치 시 패널 닫기

### 요구사항

- 패널이 열려 있는 상태에서, 유닛이나 패널이 아닌 **빈 화면**을 클릭하면 패널이 닫힌다
- 다른 유닛을 클릭하면 기존처럼 해당 유닛 패널로 전환된다 (닫히지 않음)
- 패널 자체를 클릭해도 닫히지 않는다 (닫기 버튼만 유효)

### 현재 입력 체계

- 유닛 클릭 감지: `Selectable2D` (`IPointerClickHandler`) → `OnObjectSelectedEvent` 발행
- 빈 화면 클릭: 감지 메커니즘 없음 (EventSystem이 히트하는 대상이 없어 아무 이벤트도 발생하지 않음)

### 설계 방향: LateUpdate 클릭 감지

StatInfoPanel 자체에서 매 프레임 마우스 클릭을 감지하고, 클릭 지점이 패널 밖이면 닫는다.

**LateUpdate를 사용하는 이유:**
- EventSystem은 `Update()` 단계에서 포인터 이벤트를 처리하므로, 유닛 클릭 시 `OnObjectSelected`가 `Update` 단계에서 먼저 실행된다
- `LateUpdate()`는 모든 `Update()` 이후에 실행되므로, 같은 프레임에서 유닛 클릭 → Show → `_showFrame` 설정이 완료된 이후에 닫기 판단이 가능하다

### StatInfoPanel.cs 변경

**필드 추가:**
- `int _showFrame` — `Show()` 호출 시점의 `Time.frameCount`를 기록

**Show() 변경:**
- `_showFrame = Time.frameCount`를 설정 (프레임 가드)

**LateUpdate() 추가:**
- 패널이 활성 상태일 때만 실행 (gameObject.SetActive(false)이면 LateUpdate 자동 비호출)
- `Input.GetMouseButtonDown(0)`이 true이고
- `Time.frameCount != _showFrame` (Show와 같은 프레임이 아닌 경우)이고
- 클릭 지점이 패널 영역 밖인 경우 → `Hide()`

**패널 영역 판정:**
- `RectTransformUtility.RectangleContainsScreenPoint(_rectTransform, Input.mousePosition)` 사용
- Screen Space Overlay Canvas이므로 camera 파라미터는 null

### 시나리오별 동작

```
1. 빈 화면 클릭
   → EventSystem: 히트 대상 없음, 이벤트 없음
   → LateUpdate: GetMouseButtonDown(0) = true, 패널 밖 → Hide()
   결과: 패널 닫힘 ✓

2. 다른 유닛 클릭
   → EventSystem Update: Selectable2D → OnObjectSelected → Show(newUnit) → _showFrame = 현재 프레임
   → LateUpdate: GetMouseButtonDown(0) = true, _showFrame == frameCount → 스킵
   결과: 새 유닛 패널 표시 ✓

3. 패널 위 클릭 (닫기 버튼 외)
   → LateUpdate: GetMouseButtonDown(0) = true, RectangleContainsScreenPoint = true → 스킵
   결과: 패널 유지 ✓

4. 닫기(X) 버튼 클릭
   → EventSystem Update: Button.onClick → Hide() → SetActive(false)
   → LateUpdate: 실행되지 않음 (GO 비활성)
   결과: 패널 닫힘 ✓

5. 같은 유닛 재클릭
   → EventSystem Update: OnObjectSelected → _currentUnit == unit → Hide() → SetActive(false)
   → LateUpdate: 실행되지 않음 (GO 비활성)
   결과: 패널 닫힘 (토글) ✓
```

## 9. 드래그 후 클릭 이벤트 방지

### 문제

- `Draggable2D`는 `IDragHandler` 인터페이스가 아닌 `EventTrigger` 컴포넌트를 통해 드래그를 처리
- Unity EventSystem은 press/release 대상이 같은 GameObject이면 드래그 여부와 무관하게 `OnPointerClick` 호출
- 결과: 유닛 드래그 후 놓으면 `Selectable2D.OnPointerClick`이 발생하여 StatInfoPanel이 열림

### Selectable2D.cs 변경

- `OnPointerClick`에서 `pressPosition`과 `position`의 거리를 비교
- 거리가 `EventSystem.current.pixelDragThreshold` (기본 10px)를 초과하면 클릭으로 간주하지 않음
- `EventSystem.current`가 null인 경우 (에디터 테스트 환경) 기본값 10 사용
- `sqrMagnitude` 비교로 `Mathf.Sqrt` 호출 비용 회피

### 데이터 흐름

```
유닛 드래그 후 놓기
→ EventSystem: OnPointerClick(eventData) 호출
→ Selectable2D.OnPointerClick
  ├── pressPosition = 드래그 시작 좌표
  ├── position = 드래그 종료 좌표
  ├── 거리² > threshold² → return (이벤트 미발행)
  └── 거리² ≤ threshold² → OnObjectSelectedEvent 발행 (정상 클릭)
```
