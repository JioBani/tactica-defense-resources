# TACD-122: [전장 - 유닛] 스탯

> 상위 에픽: TACD-80 [전장] 유닛

---

## 작업 내용

유닛(소환수/침략자)의 능력치 시스템을 전면 구축한다.

1. **능력치 목록 반영** — 기획서 1.1 기준 14종 능력치를 코드에 정의
2. **스탯 구조 구현** — 기본/추가/최종 스탯 분리, 수정자(고정값·비율) 시스템
3. **증감 분리 곱연산** — 방어력 등 곱연산 누적 스탯의 전용 계산 규칙
4. **데미지 계산 공식** — 치명타, 방어력, 피해증가/피해감소를 포함한 전체 공식 적용
5. **기존 컴포넌트 연동** — Attacker, Victim, Mover 등이 최종 스탯을 실시간 참조
6. **스킬 스탯 참조** — 스킬 실행 시 시전자의 최종 능력치 기반으로 효과 결정
7. **능력치 정보 UI** — 유닛 선택 시 능력치 확인 가능

---

## 완료 정의

Jira 일감에 명시된 완료 기준 7항목을 모두 충족하면 완료:

1. 기획서 1.1에 정의된 14종 능력치가 코드에 반영되어 있다
2. 유닛의 능력치가 기본값과 추가값으로 구분되어 동작한다
3. 외부 효과(버프/시너지 등)가 유닛의 능력치를 변경할 수 있다
4. 능력치 변화 시 유닛의 행동이 즉시 바뀐다
5. 데미지 계산 공식에 따라 피해가 산출된다
6. 스킬 효과가 시전자의 최종 능력치를 기반으로 결정된다
7. 유닛 선택 시 능력치 정보를 확인할 수 있다

## 현재 코드 상태 (갭 분석)

### 능력치 목록

| 항목 | 현재 상태 |
|------|-----------|
| UnitStatKind enum | 13종 정의됨. **피해 감소(DamageReduction)** 누락 |
| UnitStatsByLevelData | 별 등급 기반 기본값 제공 중 |

### 스탯 구조

| 항목 | 현재 상태 |
|------|-----------|
| UnitStats\<T\> | 단순 값 컨테이너 (CurrentValue/OriginalValue). 수정자 없음 |
| UnitStatSheet | 개별 스탯 프로퍼티 보유. Init 시 1회 로드, 동적 변경 미지원 |
| 수정자 시스템 | **미구현** — 버프/시너지 적용 불가 |

### 데미지 계산

| 항목 | 현재 상태 |
|------|-----------|
| Victim.Hit() | 데미지를 직접 빼기만 함 (공식 없음) |
| 방어력 적용 | PhysicalDefense/MagicDefense 정의만 있고 사용 안 함 |
| 치명타 | CriticalChance/CriticalDamageMultiplier 정의만 있고 계산 안 함 |
| 피해 증가/감소 | DamageDealtIncrease 정의만 있고 적용 안 함 |

### 스킬 연동

| 항목 | 현재 상태 |
|------|-----------|
| 스킬 → 스탯 참조 | 스킬 컨텍스트에 스탯 정보 없음 |
| RangeAttackExecutor | PhysicalAttack만 하드코딩 참조 |

### 기존 컴포넌트 스탯 사용

| 컴포넌트 | 현재 스탯 사용 방식 |
|-----------|---------------------|
| Attacker | AttackSpeed, AttackRange를 Init 시 1회 로드 (동적 변경 TODO) |
| Victim | Health 직접 감산 |
| Mover | MoveSpeed를 Init 시 1회 로드 |
| HealthBar | Health/MaxHealth OnChange 구독 (이미 반응형) |
