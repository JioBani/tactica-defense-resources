# TACD-200: [전장 - 소환수] - 배치/이동/스왑

> 상태: 완료

---

## 5단계: Sub-task 도출 ✅

### 5-1. 동작 트리 상세화

```
배치/이동/스왑
├─ 대기석 → 전장 배치
│   ├─ 대기석 소환수를 드래그하여 전장 그리드 칸에 배치
│   ├─ 배치 상한 미달 시에만 배치 가능
│   └─ 이미 소환수가 있는 칸에 놓을 때의 동작 → ⚠ 기획 누락
├─ 전장 내 이동/스왑
│   ├─ 전장 소환수를 빈 칸으로 이동
│   └─ 전장 소환수를 다른 소환수 위에 놓으면 스왑
├─ 전장 → 대기석 회수
│   ├─ 전장 소환수를 대기석으로 드래그
│   └─ 대기석에 빈 슬롯 필요
└─ 배치 제한
    ├─ 전투 중 배치/이동 불가
    └─ 배치 상한에 의한 제한
```

### 5-2. 보조 관점 체크

**데이터/속성:**
- 대기석 슬롯 수 → ⚠ 기획서에 정의 없음

**UI/정보 표시:**
- 현재 배치 수 / 배치 상한 → [공통: 소환터미널] UI에서 다룰 영역
- 드래그 중 배치 가능 영역 표시 → ⚠ 기획서에 정의 없음
- 배치 불가 시 피드백(사유 표시) → ⚠ 기획서에 정의 없음

**기반 작업:**
- 드래그 앤 드롭 시스템 → Draggable2D 이미 존재
- 그리드/대기석 → WaitingAreaReferences 이미 존재
