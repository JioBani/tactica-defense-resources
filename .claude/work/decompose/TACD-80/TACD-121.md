# TACD-121: [전장 - 유닛] 라이프사이클

> 상태: 완료

---

## 5단계: Sub-task 도출 ✅

### 5-1. 동작 트리 상세화

유닛이 게임에서 실제로 하는 행동의 흐름을 말단까지 분해.

```
라이프사이클 (유닛 공통 행동 흐름)
│
├─ 초기화한다
│   └─ 💻 유닛 생성 시 스탯/컴포넌트 초기화
│
├─ 대기한다 (Idle)
│   └─ 💻 적이 사거리에 없으면 대기, 감지하면 공격 전이
│
├─ 공격한다 (Attack) — 상태 전이 관점
│   ├─ 💻 적 감지 → 공격 상태 진입/유지
│   └─ 💻 대상 소실 → 대기 또는 새 타겟 탐색
│
├─ 피격당한다
│   ├─ 💻 데미지 수신 → HP 감소
│   └─ 💻 HP ≤ 0 → 다운 상태로 전이
│
├─ 다운된다 (Downed)
│   ├─ 💻 비활성화 (공격/스킬 중지)
│   ├─ 💻 침략자는 다운된 유닛을 무시하고 전진
│   └─ ❓ 라운드 간 다운 유닛 처리 (부활? HP 회복? — 기획 누락)
│
└─ 상태이상에 반응한다
    └─ 🧩 상태이상 상세 (TACD-123 소속)
```

### 5-2. 보조 관점 체크

**데이터/속성:**
- 💻 행동 상태 정의 (Idle, Move, Attack, Downed, Freeze 등 상태 타입)
- 💻 HP 관리 데이터 (현재 HP — 스탯 시스템에서 정의, 감소/판정은 라이프사이클 소속)

**UI/정보 표시:**

| 표시 가능 정보 | 표시 필요 여부 | 기획 상태 |
|---------------|--------------|----------|
| 현재 HP / 최대 HP | 필요 → 체력바 | 💻 명세 있음 |
| 다운 상태 | 필요 → 비활성화 비주얼 | ❓ 비주얼 미정 |
| 현재 행동 상태 (Idle/Attack 등) | ❓ 플레이어에게 표시 필요한가? | ❓ 기획 누락 |

**기반 작업:**
- ✅ 행동 상태 머신 프레임워크 (StateBase) — 이미 구현됨
