## 5단계: Sub-task 도출 ✅

### 5-1. 동작 트리 상세화

💻 구현 대상 / ✅ 기존 구현 완료 / ❓ 기획 누락 / 🔸 개별 스킬에서 구현 (이 이슈 범위 밖)

**참조 기획서**: `07 게임 시스템 기획서.md` - 섹션 1.8 스킬 규칙 (**기획 누락: 제목만 존재**)

**이 이슈의 범위**: 개별 스킬을 구현할 때 바로 적용할 수 있는 프레임워크(템플릿)를 만드는 것. 개별 스킬의 효과/조건 구현은 범위 밖.

```
스킬 사용 (유닛의 특수 능력 발동)
│
├─ 발동 가능 여부를 판정한다
│   ├─ ✅ 쿨타임이 완료되었는지 확인한다
│   ├─ ✅ 유닛이 다운 상태가 아닌지 확인한다
│   └─ 🔸 스킬별 발동 조건을 확인한다 — 개별 스킬이 CanCast()에서 구현
│
├─ 스킬을 시전한다
│   ├─ 🔸 시전 대상을 결정한다 — 개별 스킬이 Casting()에서 처리
│   ├─ 스킬 효과를 실행한다
│   │   ├─ 🔸 투사체 데미지 — 개별 스킬 (FireArrow에서 이미 구현)
│   │   ├─ 🔸 즉발 데미지 — 개별 스킬에서 구현
│   │   ├─ 🔸 범위 효과 — 개별 스킬에서 구현
│   │   ├─ 🔸 버프/디버프 — 개별 스킬에서 구현
│   │   └─ 🔸 기타 효과 — 개별 스킬에서 구현
│   └─ ✅ 시전 종료를 알린다 (이벤트)
│
└─ 쿨타임을 관리한다
    ├─ ✅ 시전 종료 후 쿨타임을 시작한다
    ├─ 💻 쿨타임 감소 스탯을 반영한다
    │    (능력치 1.1에 "쿨타임 감소" 정의됨, 적용 로직 미구현)
    └─ ✅ 쿨타임 완료 시 재사용 가능 상태가 된다
```
