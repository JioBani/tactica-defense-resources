# TACD-219: 스킬 정의 데이터 확장 구조 제작

## 작업 내용

개별 스킬이 고유 데이터(스킬 계수, 데미지 타입 등)를 ScriptableObject에서 정의할 수 있는 구조를 만들고, SkillDefinitionData에 맞는 스킬 인스턴스가 생성되도록 팩토리 구조를 개선한다.

### 현재 문제점

1. **SkillDefinitionData에 스킬 고유 데이터가 없음**
   - 현재 필드: id, displayName, description, coolTime, icon (표시 정보만 존재)
   - 스킬 계수(1.5f), 데미지 타입(Magical) 등이 FireArrowExecutor에 하드코딩됨

2. **SkillFactory가 하드코딩됨**
   - `CreateSkill()`이 항상 `new FireArrow(context.Attacker)` 반환
   - `context.Data`를 아예 사용하지 않음

3. **스킬 생성 시 데이터가 전달되지 않음**
   - FireArrow 생성자가 `Attacker`만 받음
   - SkillDefinitionData의 정보를 스킬 로직에서 활용할 수 없음

### 작업 대상

| # | 파일 | 변경 내용 |
|---|------|----------|
| 1 | SkillDefinitionData.cs | 스킬 고유 데이터 필드 추가 (계수, 데미지 타입 등) |
| 2 | SkillFactory.cs | 데이터 기반 스킬 인스턴스 생성 구조 개선 |
| 3 | FireArrow.cs | 데이터에서 값을 받도록 수정 |
| 4 | FireArrowExecutor.cs | 하드코딩 값을 데이터에서 받도록 수정 |
| 5 | SkillCaster.cs | 필요 시 데이터 전달 흐름 수정 |
| 6 | 불화살.asset | 확장된 필드에 기존 하드코딩 값 설정 |

## 설계 방향

### 스탯 계수 (Stat Scaling) 구조

- 계수 = (기준 스탯, 비율). 예: (공격력, 1.5) → 공격력의 150%
- 하나의 효과에 여러 스탯 계수가 붙을 수 있음 (복합 계수)
- 최종값 = 기본값 + Σ(스탯값 × 계수)

### 데이터 vs 코드 분리 원칙

- **데이터에 넣을 것** (ScriptableObject, 코드 수정 없이 변경 가능):
  - 스탯 계수 목록 (기준 스탯 + 비율)
  - 기본값, 데미지 타입 등 수치

- **코드에 남길 것** (스킬마다 로직이 다름):
  - 투사체/즉발/범위 등 시전 방식
  - 타겟 선정 로직
  - 효과 적용 순서와 조건

### 스킬별 데이터 확장 방식

**스킬 코드가 자기 데이터에서 계수를 꺼내 쓰는 방식** 채택:
- SkillDefinitionData를 상속하여 스킬별 데이터 SO 정의 (예: FireArrowDefinitionData)
- 스킬 코드가 자신의 데이터 타입을 알고, 필요한 계수를 직접 참조
- StatScaling은 공통 구조체로 제공하여 재사용

## 완료 정의

- 개별 스킬의 고유 수치(스킬 계수, 데미지 타입)를 코드 수정 없이 ScriptableObject에서 설정할 수 있다
- SkillDefinitionData에 따라 올바른 스킬 인스턴스가 생성된다
- 기존 FireArrow 스킬이 확장된 구조에서 동일하게 동작한다
