# TACD-212: 유닛 생성 시 스탯/상태 초기화 처리

## 작업 내용

유닛이 생성될 때 스탯, 행동 상태, 컴포넌트를 초기화한다.

## 현재 구현 상태

기존 코드가 완료 기준을 **이미 충족**하고 있다.

### 초기화 흐름 (구현 완료)

```
ObjectPooler.Spawn() → GameObject 활성화
  ↓
Unit.Awake()
  ├─ Draggable 캐싱
  └─ HealthBar OnHealthChange 구독
  ↓
Attacker.Awake() / Mover.Awake()
  ├─ OnSpawnEvent 구독
  └─ ActionStateController.RegisterListener(this)
  ↓
Unit.SetSpawn(UnitLoadOutData)
  ├─ StatSheet.Init(data.Stats) → 14종 스탯 초기화
  └─ OnSpawnEvent 발행
       ├─ Attacker.SetStats() → 공격범위, 공격속도, DynamicRepeater
       └─ Mover.SetSpeed() → 이동속도
  ↓
ActionStateController.OnEnable()
  └─ StartStateBase(canMove ? Move : Idle) → 초기 상태 진입
```

### 구현 완료 항목

| 항목 | 상태 | 위치 |
|------|------|------|
| 14종 스탯 기본값 설정 | ✅ | UnitStatSheet.Init() |
| 현재 체력 = 최대 체력 | ✅ | UnitStatSheet.Init() |
| 공격 범위/속도 설정 | ✅ | Attacker.SetStats() |
| 이동 속도 설정 | ✅ | Mover.SetSpeed() |
| HealthBar 연동 | ✅ | Unit.Awake() |
| Idle 상태 진입 (Defender) | ✅ | ActionStateController.OnEnable() |
| Move 상태 진입 (Aggressor) | ✅ | ActionStateController.OnEnable() |
| IStateListener 등록 | ✅ | Attacker/Mover Awake() |

## 남은 작업

기존 구현이 완료 기준을 충족하므로, **검증 테스트 작성**으로 완료한다.

- 유닛 생성 후 스탯 값이 데이터와 일치하는지 테스트
- 유닛 생성 후 Idle(또는 Move) 상태에 진입하는지 테스트

## 관련 파일

| 파일 | 역할 |
|------|------|
| Unit.cs | Awake, SetSpawn, OnSpawnEvent |
| UnitStatSheet.cs | Init(), 14종 스탯 관리 |
| ActionStateController.cs | OnEnable → StartStateBase |
| Attacker.cs | SetStats (OnSpawnEvent 수신) |
| Mover.cs | SetSpeed (OnSpawnEvent 수신) |

## 완료 정의

1. 유닛 생성 시 스탯이 UnitLoadOutData와 일치하게 설정된다
2. Defender는 Idle, Aggressor는 Move 상태로 진입한다
3. 위 동작을 검증하는 유닛 테스트가 통과한다
