# TACD-213: 적 감지 → 공격 전이, 대상 소실 → 대기 복귀

## 작업 내용

유닛이 사거리 내에 적을 감지하면 공격 상태로 전이하고, 대상이 소실되면 대기로 복귀하거나 새 타겟을 탐색한다.

## 현재 구현 상태

### 구현 완료

| 항목 | 위치 |
|------|------|
| 적 감지 (OnTriggerEnter2D) → _victim 설정 | Attacker.cs:71-86 |
| Idle/Move → Attack 상태 전환 | ActionStateController.cs:30-44 |
| Attack 상태 진입 시 DynamicRepeater 시작 | Attacker.cs:110-116 |
| 적 이탈 (OnTriggerExit2D) → _victim 해제 | Attacker.cs:88-98 |
| Attack → Idle/Move 전환 (타겟 없을 때) | ActionStateController.cs:46-52 |

### 버그: 타겟 다운 시 _victim 참조 미정리

**현상:** 공격 중인 타겟이 다운(HP=0)되면 `_victim`이 여전히 다운된 유닛을 참조한다.

**결과:** ActionStateController가 매 프레임 다음을 반복한다:
1. Attack 상태 → `_victim.CurrentState == Downed` → Idle/Move로 전환
2. Idle/Move 상태 → `attacker.Victim != null` (아직 참조 중) → Attack로 전환
3. 무한 루프 발생 (Idle ↔ Attack)

### 미구현: 새 타겟 자동 탐색

**현상:** 타겟이 소실된 후, 사거리 내에 이미 존재하는 다른 적을 탐색하지 못한다.

**원인:** `OnTriggerEnter2D`는 Collider가 **진입**할 때만 호출됨. 이미 범위 내에 있는 적에게는 재호출되지 않음.

## 수정 필요 사항

### A. 타겟 다운 시 _victim 해제

Attacker에서 타겟의 Downed 상태 전환을 감지하여 `_victim`을 null로 정리한다.

### B. 새 타겟 자동 탐색

타겟이 소실되었을 때 사거리 내에 있는 다른 적을 탐색하여 새 타겟으로 설정한다.

## 관련 파일

| 파일 | 역할 |
|------|------|
| `Attacker.cs` | 타겟 관리, 공격 반복, 물리 감지 |
| `ActionStateController.cs` | 상태 전환 조건 |

## 완료 정의

1. 적 감지 시 Idle/Move → Attack 전이 정상 동작
2. 타겟이 다운되면 _victim이 해제되고 Idle/Move로 복귀
3. 타겟 소실 후 사거리 내 다른 적이 있으면 자동으로 새 타겟 획득
4. Idle ↔ Attack 무한 루프 없음
5. 유닛 테스트 통과
