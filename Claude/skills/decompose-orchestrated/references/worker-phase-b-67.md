# 워커: 검증 + 이슈 목록 정리 (6+7단계)

이 파일은 서브에이전트가 직접 읽고 수행하는 지시서이다.
모든 Task별 Sub-task 도출과 교차 검증이 완료된 후 수행한다.

## 입력 파라미터

- **에픽 티켓 번호**: 프롬프트에서 전달받음
- **작업 경로**: `Claude/work/decompose/{에픽}/` 디렉토리

## 사전 읽기

1. **분해결과.md**: Phase A 결과 (에픽 정보, Task 목록, 기획서 경로)
2. **모든 Task 파일** ({Task 티켓}.md): 5단계 결과
3. **교차검증.md**: 플랫 리스트
4. **기획서**: 분해결과.md 1단계에 기록된 기획서 경로의 원문을 직접 읽는다.
5. **방법론**: `Document/프로젝트 관리/구현 이슈 분해 방법론.md`의 6단계, 7단계, 교차 검증 섹션을 읽는다.

## 수행 절차

### 6단계: 검증

1. 기획서를 다시 읽으며 Task/Sub-task에 빠진 내용이 없는지 대조한다.
2. 이슈 간 중복이 없는지 확인한다.
3. 발견된 누락/중복을 기록한다.

### 교차 검증 대조

교차검증.md의 플랫 리스트를 Task/Sub-task 구조와 대조한다:

1. 플랫 리스트의 각 항목이 어떤 Task > Sub-task에 매핑되는지 확인한다.
2. 매핑 안 되는 항목 → 누락 후보로 분류한다:
   - **Task 수준 누락**: 기존 Task 어디에도 속하지 않는 항목
   - **Sub-task 수준 누락**: 기존 Task에 속할 수 있지만 Sub-task로 도출되지 않은 항목
3. 각 누락 후보에 대해 추가/기각 판정과 사유를 작성한다.

교차검증.md에 대조 결과를 추가 기록한다.

### 7단계: 이슈 목록 정리

방법론 문서의 7단계 형식에 따라 각 Task별로 정리한다:

```
## Task: {Task 이름}

### 도출된 전체 항목

| # | 업무 분류 | 제목 |
|---|----------|------|

### → Sub-task (구현)

| # | 유형 | 제목 | 기획 상태 |
|---|------|------|----------|

### → 기획 누락 (게임플레이 기획)

| # | 누락 항목 | 상세 내용 |
|---|----------|----------|

### → 후속 작업 (기타 업무 분류)

| # | 업무 분류 | 제목 |
|---|----------|------|
```

### 7-1. 기획 누락 항목의 Sub-task 등록 판단

기획 누락 항목 각각에 대해:
- **구현이 확실한 항목** → Sub-task로 등록 대상, 상태 "기획 필요"
- **기획 판단에 따라 구현 여부가 달라지는 항목** → Sub-task에 포함하지 않음

## 산출물 원칙

Jira 등록 전 코드 레벨 필터링: 산출물에 클래스명, 메서드명, 파일 경로 등 코드 레벨 내용이 남아 있으면 기능/동작 수준으로 변환한다.

## 작업 파일 기록

- 교차검증.md에 대조 결과 섹션을 추가한다.
- 각 Task 파일에 6단계, 7단계 결과를 추가한다.
- 분해결과.md 상태를 업데이트한다:

```markdown
> 현재 Phase: B
> 다음 작업: ★검증 (Phase B 검증 + 사용자 검증)
```
