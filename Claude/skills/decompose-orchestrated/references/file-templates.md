# 작업 파일 형식 정의

워커 서브에이전트가 산출물을 기록할 때 참조하는 파일 형식이다.

## 분해결과.md 형식

```markdown
# 구현 이슈 분해: {에픽 티켓 번호} - {에픽 제목}

> 현재 Phase: A / B
> 다음 작업: {단계 코드, 예: A-2, B-1, ★검증}

---

## 1단계: 기획서 파악 ✅

**참조 기획서**: {경로}
**핵심 요약**: ...

## 2단계: 에픽 경계 확인 ✅

**대상 에픽**: {티켓 번호} - {제목}
**인접 에픽**: ...
**경계 판단**: ...

## 3단계: Task 도출 ✅

### 상위 동작 트리
(트리 구조)

### 상호작용 분석 (상위)
(대상 분류별 열거 + 주요 연결 지점 개요)

### Task 목록
(경계 판단 결과)

## 4단계: Task Jira 등록 ✅

| # | Task | Jira | Sub-task 분해 |
|---|------|------|--------------|
| 1 | {기능명} | TACD-XXX | ✅ / 🔄 / - |
| 2 | {기능명} | TACD-XXX | - |

## 8단계: Sub-task Jira 등록 ✅

### TACD-XXX: {Task 이름}

| Jira | 유형 | 제목 | 상태 |
|------|------|------|------|
| TACD-XXX | 하위 스토리 / 하위 작업 | ... | Backlog / 기획 필요 |

## 기획 누락 목록

| # | 누락 항목 | 관련 이슈 |
|---|----------|----------|

## 후속 작업

| # | 업무 분류 | 제목 |
|---|----------|------|
```

## Task별 파일 형식

파일명: `{Task 티켓 번호}.md`

```markdown
# {Task 티켓 번호}: {Task 제목}

> 상태: 진행중 / 5단계 완료, 검증 대기 / 완료

---

## 5단계: Sub-task 도출 {✅/🔄}

### 5-1. 동작 트리 상세화
(말단 노드까지 분해한 트리)

### 5-2. 상호작용 분석 (상세)
(대상별 × 내용별 매트릭스)

### 5-3. 보조 관점 체크
(데이터/속성, UI/정보 표시, 기반 작업)

### 5-4. 업무 분류 및 분류
(전체 항목 + 분류 결과)

## 6단계: 검증 {✅/🔄}

## 7단계: 이슈 목록 정리 {✅/🔄}

### 도출된 전체 항목

| # | 업무 분류 | 제목 |
|---|----------|------|

### → Sub-task (구현)

| # | 유형 | 제목 | 기획 상태 |
|---|------|------|----------|

### → 기획 누락 (게임플레이 기획)

| # | 누락 항목 | 상세 내용 |
|---|----------|----------|

### → 후속 작업 (기타 업무 분류)

| # | 업무 분류 | 제목 |
|---|----------|------|

## 8단계: Sub-task Jira 등록 {✅/🔄}

(등록된 이슈 번호 기록)
```

## 교차검증.md 형식

```markdown
# 교차 검증: {에픽 티켓 번호} - {에픽 제목}

## 분석 기반

- **기획서 참조**: {경로, 라인 범위}
- **에픽 경계**: {범위 요약}
- **제외 범위**: {인접 에픽 목록}

## 분해 과정

### 축 1: 동작 트리
(라이프사이클 기반 말단 동작 나열)

### 축 2: 상호작용
(대상별 연결 지점)

### 축 3: 보조 관점
(데이터/속성, UI/정보 표시, 기반 작업)

## 플랫 리스트

| # | 항목 | 출처 (동작/상호작용/보조관점) | 업무 분류 |
|---|------|---------------------------|----------|

## 대조 결과

(Phase B 완료 후 worker-phase-b-67 워커가 작성)

### 매핑된 항목

| # | 플랫 리스트 항목 | 매핑 대상 (Task > Sub-task) |
|---|-----------------|---------------------------|

### 누락 후보 — Task 수준

| # | 플랫 리스트 항목 | 판정 (추가/기각) | 사유 |
|---|-----------------|----------------|------|

### 누락 후보 — Sub-task 수준

| # | 플랫 리스트 항목 | 대상 Task | 판정 (추가/기각) | 사유 |
|---|-----------------|----------|----------------|------|
```
