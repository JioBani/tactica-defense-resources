# 워커: 교차 검증

이 파일은 서브에이전트가 직접 읽고 수행하는 지시서이다.
Task별 분해와 **독립적으로** 수행하여, 누락을 검출한다.

## 입력 파라미터

- **에픽 티켓 번호**: 프롬프트에서 전달받음
- **Phase A 결과**: 분해결과.md 경로 (기획서 경로, 에픽 경계 포함)
- **작업 파일**: 교차검증.md 경로

## 사전 읽기

1. **Phase A 결과** (분해결과.md)를 읽어 기획서 경로와 에픽 경계를 확인한다.
2. **기획서**: 분해결과.md 1단계에 기록된 기획서 경로의 원문을 직접 읽는다.
3. **방법론**: `Document/프로젝트 관리/구현 이슈 분해 방법론.md`의 "분해 방법" 섹션 (동작 트리 + 상호작용 + 보조 관점)을 읽는다.

## 독립성 보장

- Phase A의 Task 목록을 **읽지 않는다**.
- Phase B의 Sub-task 결과를 **읽지 않는다**.
- 기획서와 에픽 경계만으로 독립적으로 분해한다.

## 수행 절차

### 플랫 리스트 도출

모든 분해 축(동작 트리 + 상호작용 + 보조 관점)을 **한번에** 적용하여 플랫 리스트를 도출한다.

각 항목에 기록할 것:
- 항목 내용
- 출처 (동작 / 상호작용 / 보조관점)
- 업무 분류

## 산출물 원칙

산출물은 **기능/동작 수준**으로만 작성한다.

- 코드베이스 탐색은 맥락 파악을 위해 **허용**한다.
- 산출물에는 클래스명, 메서드명, 코드 패턴, 파일 경로, 구현 용어를 포함하지 않는다.
- 플레이어/기획자가 읽어도 이해할 수 있는 게임 동작 수준으로 작성한다.
- **예외**: 기술적 설계 작업은 기술 용어 허용.

## 작업 파일 기록

파일 형식 참조: `Claude/skills/decompose-orchestrated/references/file-templates.md`의 "교차검증.md 형식" 섹션.

작업 파일에 기록한다:
- 분석 기반 (기획서 참조, 에픽 경계, 제외 범위)
- 분해 과정 (축별 분석)
- 플랫 리스트

**대조 결과**는 이 단계에서 작성하지 않는다. Phase B 완료 후 worker-phase-b-67 워커가 작성한다.
