# 워커: 기획서 파악 + 에픽 경계 확인 (1+2단계)

이 파일은 서브에이전트가 직접 읽고 수행하는 지시서이다.

## 입력 파라미터

- **에픽 티켓 번호**: 프롬프트에서 전달받음
- **작업 파일**: 결과를 기록할 분해결과.md 경로

## 수행 절차

1단계와 2단계는 순차로 수행한다. 2단계가 1단계의 기획서 파악 결과에 의존한다.

### 1단계: 기획서 파악

1. Jira MCP 도구로 해당 에픽의 정보를 조회한다.
2. `Document/기획서/` 폴더에서 해당 에픽과 관련된 기획서를 찾아 읽는다.
3. 기획서의 전체 그림을 이해하고, 핵심 요약을 작성한다.

### 2단계: 에픽 경계 확인

기획서에서 인접 에픽을 도출한 뒤, Jira 에픽에 매핑한다.

#### 2-1. 키워드 선정

분해 대상 에픽과 관련된 키워드를 여러 개 선정한다.
- 에픽의 핵심 대상 (예: "소환술사")
- 동의어, 약칭 (예: "소환사")
- 관련 장소/개념 (예: "방어선", "라인 끝")

#### 2-2. 기획서 문맥 수집

`Document/기획서/` 전체에서 선정된 키워드를 검색하여, 등장하는 모든 문맥(해당 문단/섹션)을 수집한다.

#### 2-3. 연결 시스템 추출

각 문맥에서 대상 에픽 **외에** 언급된 시스템/기능을 추출한다.

예시:

| 문맥 | 추출된 연결 시스템 |
|------|-------------------|
| "소환술사가 전장에 등장할 소환수 풀을 결정한다" | 소환수 |
| "소환술사의 HP가 0이 되면 전투에서 패배한다" | 라운드/승패 |
| "소환술사별로 시너지 효과를 보유한다" | 시너지 |

#### 2-4. Jira 에픽 매핑

1. Jira MCP 도구로 같은 프로젝트의 에픽 목록을 조회한다.
2. 추출된 연결 시스템을 에픽 목록에 매칭하여 인접 에픽을 도출한다.

#### 2-5. 경계 판단

각 인접 에픽에 대해 이 에픽의 범위와 인접 에픽의 범위를 구분한다:
- 공통 시스템은 공통 에픽에 포함
- 고유 동작만 이 에픽에 포함

경계 판단 결과를 표로 정리한다.

## 작업 파일 기록

파일 형식 참조: `Claude/skills/decompose-orchestrated/references/file-templates.md`의 "분해결과.md 형식" 섹션.

작업 파일에 아래를 기록한다:
- 파일 상단 헤더 (에픽 번호, 제목, 현재 상태)
- 1단계 결과 (참조 기획서 경로, 핵심 요약)
- 2단계 결과 (대상 에픽, 인접 에픽 경계 표)

상태 표시를 아래와 같이 설정한다:

```markdown
> 현재 Phase: A
> 다음 작업: A-2 (Task 도출)
```
