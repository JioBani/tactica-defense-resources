# 워커: Sub-task Jira 등록 (8단계)

이 파일은 서브에이전트가 직접 읽고 수행하는 지시서이다.

## 입력 파라미터

- **에픽 티켓 번호**: 프롬프트에서 전달받음
- **작업 경로**: `Claude/work/decompose/{에픽}/` 디렉토리

## 사전 읽기

1. **분해결과.md**: Task 목록 (Jira 티켓 번호)
2. **각 Task 파일**: 7단계 이슈 목록 (Sub-task 목록, 기획 누락 판정 결과)
3. **이슈 작성 요령**: `Document/프로젝트 관리/구현 이슈 작성 요령.md`를 읽는다.
4. **Jira 지침**: `.claude/docs/jira.md`를 읽는다.

## 수행 절차

### Sub-task 등록

각 Task에 대해:

1. 7단계 Sub-task 목록을 기반으로, 해당 Task의 하위 작업으로 Jira에 등록한다.
2. 이슈 작성 요령의 **Sub-task 작성 요령**을 따른다:
   - **유형 선택**: 하위 스토리 (`~할 수 있어야 한다`) / 하위 작업 (`{대상} {동작}`)
   - **제목**: `[에픽 약칭 > 기능명] - {하위 이슈 제목}`
   - **설명**: 작업 내용 + 완료 기준 + 참고
   - **상태**: 기획 누락이 있으면 "기획 필요"
3. 7-1에서 구현이 확실하다고 판정된 기획 누락 항목도 Sub-task로 등록하고, 상태를 "기획 필요"로 설정한다.

### Task 댓글 등록

각 Task에 댓글로 도출 근거를 남긴다. 이슈 작성 요령의 **댓글** 섹션 형식을 따른다:

```
## 도출 근거

💻 구현 대상 / 🧩 공통 에픽 소속 / ✅ 기존 구현 완료 / ❓ 기획 누락

### 동작 트리
(Task 파일 5-1의 동작 트리)

### 상호작용 분석
(Task 파일 5-2의 매트릭스 요약)

### 보조 관점 체크
(Task 파일 5-3의 결과)
```

### Task 설명 업데이트

각 Task의 Jira 설명에 이 Task가 다루는 범위를 간략히 기술한다.

## Jira MCP 도구 사용 시 주의

- description 필드에 `\n` 이스케이프를 사용하지 않는다. 실제 줄바꿈을 사용한다.
- MCP 도구가 markdown을 Jira wiki markup으로 자동 변환한다.

## 작업 파일 기록

- 분해결과.md에 8단계 결과를 추가한다 (등록된 Jira 티켓 번호 테이블).
- 각 Task 파일에 8단계 결과를 추가한다.
- 분해결과.md 상태를 업데이트한다:

```markdown
> 현재 Phase: B 완료
> 다음 작업: 완료
```
