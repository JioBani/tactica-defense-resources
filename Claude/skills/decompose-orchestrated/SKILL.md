# 구현 이슈 분해 (오케스트레이터)

에픽 티켓 번호: $ARGUMENTS

## 설계 원칙

이 스킬은 **오케스트레이터-워커 패턴**으로 동작한다:

| 역할 | 담당 | 기억할 것 |
|------|------|-----------|
| **메인 (오케스트레이터)** | 워크플로우 순서, 서브에이전트 디스패치, 사용자 검증 | 다음에 누구를 호출할지 |
| **워커 서브에이전트** | 각 단계의 실제 작업 수행 | reference 파일에서 직접 읽음 |
| **검증 서브에이전트** | Phase 전환 시 산출물 품질 검증 | validator.md에서 직접 읽음 |

**핵심 규칙:**
- 메인 에이전트는 방법론 문서를 직접 읽지 않는다.
- 서브에이전트에게 내용을 요약해서 전달하지 않는다. **파일 경로만 전달**한다.
- 서브에이전트가 지시 파일 + 프로젝트 문서를 직접 읽고 수행한다.

## 경로 규칙

```
스킬 경로: Claude/skills/decompose-orchestrated
작업 경로: Claude/work/decompose/{에픽 티켓 번호}/
```

## 이력 확인

`Claude/work/decompose/{에픽}/` 디렉토리에 기존 기록이 있는지 확인한다.
- **있으면**: `분해결과.md` 상단의 `현재 Phase`, `다음 작업`을 읽고 해당 단계부터 이어서 진행한다.
- **없으면**: Phase A부터 시작한다.

## 워크플로우

### Phase A: Epic → Task

#### A-1. 워커: 기획서 파악 + 에픽 경계 (1+2단계)

Agent 서브에이전트를 실행한다:

```
지시 파일: {스킬 경로}/references/worker-phase-a-12.md
에픽 티켓 번호: {에픽}
작업 파일: {작업 경로}/분해결과.md
```

완료 후 → 분해결과.md에 1+2단계 결과가 기록됨.

#### A-2. 워커: Task 도출 (3단계)

```
지시 파일: {스킬 경로}/references/worker-phase-a-3.md
에픽 티켓 번호: {에픽}
작업 파일: {작업 경로}/분해결과.md
```

완료 후 → 분해결과.md의 3단계 결과를 읽어 사용자에게 보여준다.

#### ★ 검증 + 사용자 검증

**검증 서브에이전트** 실행:

```
검증 지시: {스킬 경로}/references/validator.md
검증 대상: Phase A
작업 파일: {작업 경로}/분해결과.md
```

검증 결과를 사용자에게 전달 → **사용자 검증**을 받는다.
수정 요청 시 → A-2 워커를 재실행한다.

#### A-3. 워커: Task Jira 등록 (4단계)

사용자 검증 통과 후:

```
지시 파일: {스킬 경로}/references/worker-phase-a-4.md
에픽 티켓 번호: {에픽}
작업 파일: {작업 경로}/분해결과.md
```

### Phase B: Task → Sub-task

#### B-1. 워커들: Sub-task 도출 + 교차 검증 (5단계, 병렬)

분해결과.md에서 Task 목록(티켓 번호, 이름)을 읽는다.

**Task별 워커** (Task 수만큼 병렬) + **교차 검증 워커** (동시)를 한 번에 실행:

Task별:
```
지시 파일: {스킬 경로}/references/worker-phase-b-5.md
에픽 티켓 번호: {에픽}
Task 티켓 번호: {Task 티켓}
Task 이름: {Task 이름}
Phase A 결과: {작업 경로}/분해결과.md
작업 파일: {작업 경로}/{Task 티켓}.md
```

교차 검증:
```
지시 파일: {스킬 경로}/references/worker-cross-validate.md
에픽 티켓 번호: {에픽}
Phase A 결과: {작업 경로}/분해결과.md
작업 파일: {작업 경로}/교차검증.md
```

#### B-2. 워커: 검증 + 정리 (6+7단계)

모든 B-1 워커 완료 후:

```
지시 파일: {스킬 경로}/references/worker-phase-b-67.md
에픽 티켓 번호: {에픽}
작업 경로: {작업 경로}/
```

완료 후 → 결과를 사용자에게 보여준다.

#### ★ 검증 + 사용자 검증

**검증 서브에이전트** 실행:

```
검증 지시: {스킬 경로}/references/validator.md
검증 대상: Phase B
작업 경로: {작업 경로}/
```

검증 결과를 사용자에게 전달 → **사용자 검증**을 받는다.
수정 요청 시 → 해당 워커를 재실행한다.

#### B-3. 워커: Sub-task Jira 등록 (8단계)

사용자 검증 통과 후:

```
지시 파일: {스킬 경로}/references/worker-phase-b-8.md
에픽 티켓 번호: {에픽}
작업 경로: {작업 경로}/
```

## 서브에이전트 디스패치 방법

Agent 도구를 호출할 때, 아래 형식으로 프롬프트를 구성한다:

```
아래 지시 파일을 읽고 수행하라.

{위 각 단계에 명시된 파라미터들}
```

**금지 사항:**
- 메인 에이전트가 방법론이나 기획서 내용을 요약하여 서브에이전트에게 전달하지 않는다.
- 서브에이전트 프롬프트에 방법론 세부사항을 인라인으로 적지 않는다.
- 항상 파일 경로만 전달하고, 서브에이전트가 직접 Read하게 한다.

## 컨텍스트 복구

대화가 길어져 컨텍스트가 압축되면, `분해결과.md` 상단의 상태 표시를 읽어 현재 위치를 복원한다:

```markdown
> 현재 Phase: A / B
> 다음 작업: {다음 실행할 단계 코드, 예: A-2, B-1, ★검증}
```

이 정보만으로 워크플로우의 어느 지점에서 재개할지 결정할 수 있다.
