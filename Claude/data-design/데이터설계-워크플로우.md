# 데이터 구조 설계 워크플로우

> 데이터 구조 설계 작업 시 사용자와 AI가 협업하는 워크플로우.

---

```

├── 데이터 설계 작업 현황.md   # 진행 추적 체크리스트 + 진행 기록
├── 컨텍스트 수집.md           # Phase 1 산출물
```

## Phase 1: 컨텍스트 수집

> 설계에 필요한 정보를 모으는 단계. 1~3은 순서가 고정되지 않으며, 상황에 따라 병렬 진행하거나 순서를 바꿀 수 있다.

### 1. 요구사항 수집

**입력**: Jira 티켓 (작업 목표, 설명), 기획서 (도메인 규칙, 용어 정의)

**수행**
- Jira 티켓에서 작업 범위와 완료 정의를 파악한다
- 관련 기획서에서 도메인 규칙, 용어, 제약 조건을 수집한다
- 수집한 내용을 `컨텍스트 수집.md`에 정리한다

**산출물**: 요구사항 요약 → `컨텍스트 수집.md`

### 2. 기존 코드 분석

**입력**: CLAUDE.md (아키텍처 개요), 기존 코드베이스

**수행**
- 사용자에게 분석이 필요한 코드나 참고할 기존 구현을 추천받는다
- 추천받은 코드 + CLAUDE.md 기반으로 관련 데이터 구조, 패턴, 컨벤션을 파악한다
- 새 데이터 구조가 연결될 기존 시스템의 인터페이스를 확인한다
- 활용 가능한 기존 인프라(enum, 수정자 시스템 등)를 식별한다
- 결과를 `컨텍스트 수집.md`에 추가한다

**산출물**: 기존 코드 참조 요약 → `컨텍스트 수집.md`

### 3. 외부 리서치 (필요 시)

**수행**
- 사용자에게 외부 리서치가 필요한지 확인한다
- 필요한 경우, 사용자에게 참고할 레퍼런스(게임, 시스템, 문서 등)를 추천받는다
- 추천이 없으면 유사 시스템의 데이터 구조를 자동으로 검색한다
- 결과를 `컨텍스트 수집.md`에 추가한다

**산출물**: 외부 참조 요약 → `컨텍스트 수집.md`

### Phase 1 완료 조건

- `컨텍스트 수집.md`를 사용자에게 제시하고 확인을 받는다
- 확인 없이 Phase 2로 넘어가지 않는다

---

## Phase 2: 데이터 정의

> 수집한 정보를 정리하고, 구체적인 필드로 확정하는 단계.

### 4. 데이터 케이스 분류

**수행**
- 수집한 정보를 아래 3가지로 분류한다:
  - **표현 데이터**: UI에 보여줄 정보 (이름, 설명, 아이콘 등)
  - **로직 데이터**: 런타임에서 계산/판정에 사용하는 정보 (타입, 임계치, 상수 등)
  - **추정 데이터**: 기획서/이슈에 명시되지 않았지만 일반적으로 필요한 정보
- 추정 데이터는 사용자와 범위 확인 → 포함 여부를 결정한다
- 결과를 `데이터 정의.md`에 정리한다

**산출물**: 데이터 케이스 분류표 → `데이터 정의.md`

### 5. 필드 목록 확정

**수행**
- 4단계의 분류 결과를 구체적인 필드(이름, 타입)로 변환한다
- 각 필드가 어느 구조체/클래스에 속할지 1차 배치한다
- 결과를 `데이터 정의.md`에 추가한다

**산출물**: 확정된 필드 목록 → `데이터 정의.md`

### Phase 2 완료 조건

- `데이터 정의.md`를 사용자에게 제시하고 확인을 받는다
- 확인 없이 Phase 3으로 넘어가지 않는다

---

## Phase 3: 설계 및 구현

> 확정된 필드를 구조화하고, 코드로 만드는 단계.

### 6. 초안 설계

**수행**
- 확정된 필드를 struct/class/enum으로 구조화한다
- 각 구조체/클래스의 역할과 관계를 명시한다
- **HTML 파일로 ERD를 생성**하여 시각화한다 (`데이터구조-설계.html`)
- 설계 결정의 근거를 기록한다

**산출물**: `데이터구조-설계.html` (ERD + 설계 결정 사항)

- HTML ERD를 사용자에게 제시하고 피드백을 받는다
- 피드백을 반영하여 설계를 수정한다 (필요 시 반복)

### 7. 구현

**수행**
- **plan 모드**로 구현 계획을 작성한다
- plan 모드 승인 후 코드를 구현한다
- 컴파일 확인 (refresh_unity + read_console)

**산출물**: 구현된 코드

## 작업 규칙

- 작업 시작 시 `데이터 설계 작업 현황.md`를 생성하고, 진행 상황을 체크리스트로 추적한다
- **Phase 완료 시 해당 산출물 파일을 기반으로 사용자 확인을 받는다.**
  - Phase 1 → `컨텍스트 수집.md`
  - Phase 2 → `데이터 정의.md`
  - Phase 3 초안 → `데이터구조-설계.html`
  - Phase 3 구현 → plan 모드 승인
- 확인 없이 다음 Phase로 넘어가지 않는다.
- `데이터 설계 작업 현황.md`의 체크리스트를 실시간으로 갱신한다.
