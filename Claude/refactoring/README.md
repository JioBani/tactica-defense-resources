# Refactoring Decision Log

## 목적

리팩토링·설계 수정을 진행할 때 *어떤 기준으로 의사결정을 내렸는지*를 사례 단위로 누적한다.
나중에 사례가 충분히 모이면 정제·일반화하여 설계 컨벤션 또는 코드 컨벤션으로 승격시킨다.

## 작성 규칙

- 한 사례 = 한 파일. 파일명: `<YYYY-MM-DD>-<티켓번호>-<짧은-식별자>.md`
- 각 사례는 다음 4 섹션으로 구성:
  - **상황** — 무엇이 문제였는가 (객관적 사실, 코드·구조 기반)
  - **결정** — 어떻게 바꿨는가 (Before/After 가능하면 둘 다)
  - **결정 근거** — 왜 그렇게 정했는가 (의사결정 기준)
  - **재사용 가능 원칙 후보** — 본 사례에서 추출 가능한 일반 원칙 (잠정 — 누적되면 정제)

## 정제 트리거

다음 시점에 본 폴더의 사례들을 훑어 일반 원칙을 추출·정제:

- 사례 수가 5 개 이상 누적되었을 때
- 비슷한 결정 근거가 2 회 이상 반복 등장할 때
- 새 설계·리팩토링 결정 시 본 폴더를 참고했는데 기준이 분산되어 있어 비교가 어려울 때

정제된 일반 원칙은 `.claude/docs/clean-architecture.md` / `code-convention.md` 등으로 승격하거나
별도 컨벤션 문서로 분리한다.
