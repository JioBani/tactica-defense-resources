# 소환술사 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 기능 — 대응 UR 노드: `전장 > 라운드 & 페이즈` (전장 승리·패배 조건을 이 노드가 보유). 근거: 본 모듈의 전장 내 책임은 레인 끝 고정 배치 유닛으로서 피격·교전하고, 다운 시 라운드 패배를 통지하며, 정비 페이즈에 회복하는 것 — 즉 패배 조건과 페이즈 라이프사이클의 실현체다. 편성 선택(전장 입장 전 4명 고름)은 `출전 편성` 노드 소관이며 본 모듈은 그 결과(`SummonerFormation`)를 소비할 뿐이라 1차 매핑에서 제외한다.

## 1. 한 줄 정의

편성으로 결정된 소환술사를 레인 끝 스폰 포인트에 고정 배치하고, 각 소환술사 유닛이 다운되면 라운드 패배를, 정비 대기 진입 시 전체 회복을 일으키는 전장 내 소환술사 유닛/관리 컴포넌트.

## 3. 구조 맵 (위치 중심)

진입점부터.

- **배치 진입점**: `SummonerManager.Start` → `SpawnSummoners` — `Summoner/SummonerManager.cs` (편성 읽어 스폰 포인트에 배치)
- **편성 주입**: `SummonerManager.SetFormation` — `Summoner/SummonerManager.cs` (편성 UI 완성 전까지는 인스펙터 `testSummoners` 폴백)
- **유닛 인스턴스**: `Summoner : Units.Unit, IStateListener<ActionStateType>` — `Summoner/Summoner.cs`
- **상태 반응**: `Summoner.OnStateEnter` (Downed→패배 통지 / Waiting→회복) — `Summoner/Summoner.cs`
- **씬 프리팹**: `Summoner.prefab` — `Summoner/Summoner.prefab`

### 3.2 의존 관계

- **밖으로 의존**: 유닛 기반(`Units.Unit`·`UnitGenerator.GenerateSummoner` — `Feature/Unit/Unit/`), 상태 머신(`ActionStateController`·`ActionStateType`·`IStateListener` — `Feature/Unit/ActionState/`, `Common.Scripts.StateBase`), 스탯(`StatSheet.RecoverFullHealth`), 이벤트 버스(`GlobalEventBus` + `OnRoundLoseEventDto` — `Common.Scripts.GlobalEventBus`, `Feature/Events/RoundEvents/`), 씬 싱글톤(`Common.Scripts.SceneSingleton`), 편성·정의 데이터(`Common.Data.Summoners.*` — `SummonerFormation`·`SummonerLoadOutData`·`SummonerDefinitionData`)
- **밖에서 의존**: 라운드/페이즈 시스템(`OnRoundLoseEventDto` 구독자), 편성 선택 UI(`SetFormation` 호출 예정), 편성 술사 목록을 읽는 소비처(시너지의 심상 효과 등)

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도 (Summoner ↔ SummonerManager)**: `Summoner`는 유닛 1체의 정체성·상태 반응만, `SummonerManager`는 편성→배치 라이프사이클만 담당한다. 다중 소환술사 배치·편성 주입 책임을 유닛 본체에서 떼어낸 것.
- **도메인적 의도 (다운 = 패배)**: 소환술사가 직접 전투를 수행하기보다, 그 다운(`Downed`)이 곧 전장 패배 조건이라는 기획을 상태 전이 한 분기로 인코딩했다. 소환술사는 "지켜야 할 대상" 역할이다.
- **비자명한 제약/관례**:
  - `SummonerDefinition`은 `OnSpawn`에서 `SummonerLoadOutData`로 캐스팅 성공해야 채워진다 — 잘못된 로드아웃 타입은 에러 로그만 남고 정의가 비게 된다.
  - 배치 수는 `_formation.Summoners.Count`와 인스펙터 `spawnPoints.Count`의 **더 작은 쪽**으로 제한된다(스폰 포인트가 상한).
  - 편성 선택 UI 미완성 단계라 `testSummoners` 인스펙터 폴백과 `SetFormation` 주입이 공존한다(코드 주석에 명시된 잠정 구조).
