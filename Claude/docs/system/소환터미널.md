# 소환터미널 — 시스템 관점

> 이 문서는 **AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록** 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실(single source of truth)이다. 이 문서는 코드를 재설명하지 않는다 — **"인덱스 + α"** 에 한정한다.
>
> - **분류**: 기능(단순 구현) — 대응 UR 노드: `전장 > 소환 터미널`
> - **이 문서가 다루는 것**: ① 이 대상이 시스템에서 무엇인가(한 줄) ② 도메인 개념 ↔ 코드(파일 경로)의 매핑 ③ 무엇이 어디에 있는가(구조 맵) ④ 다른 코드가 가져다 쓸 헬퍼 ⑤ 코드만 봐선 모르는 의도(+α).
> - **이 문서가 다루지 않는 것**: 코드의 동작을 풀어쓴 설명서, 알고리즘·자료구조 해설, 메서드 본문 요약.

---

## 1. 한 줄 정의

소환 터미널은 정비 페이즈 동안 마나 경제(소환·환원·스캔·레벨업)를 관장하고, 마켓 슬롯·배치 상한·스캔 잠금 같은 소환 인터페이스의 상태를 보유하는 전장 서브시스템이다.

## 2. 도메인 ↔ 코드 매핑

| 도메인 개념 | 코드 식별자 | 위치(파일 경로) | 책임 (한 줄) |
|------------|------------|----------------|-------------|
| 소환 터미널(경제 로직) | `MarketManager` | `Assets/Scenes/Battle/Feature/Market/Scripts/MarketManager.cs` | 마나·소환·환원·스캔·레벨업을 관장하는 진입점 컴포넌트 |
| 마나 | `MarketManager.Mana` | 위 파일 | 보유 마나의 관측 가능 상태 |
| 소환 (구매) | `MarketManager.BuyDefender` | 위 파일 | 마켓 슬롯의 소환수를 마나로 소환해 대기석에 배치 |
| 환원 (판매) | `MarketManager.Sell` | 위 파일 | 소환수를 환원하고 성급·강화 단계에 따른 마나를 회수 |
| 스캔 (리롤) | `MarketManager.Reroll` | 위 파일 | 마나를 지불해 마켓 등장 목록을 재구성 |
| 스캔 잠금 | `MarketManager.IsScanLocked` / `ToggleScanLock` | 위 파일 | 라운드 시작 시 자동 재스캔 스킵 여부의 상태·토글 |
| 배치 상한 / 레벨 | `MarketManager.DefenderPlacementLimit` / `Level` / `LevelUp` | 위 파일 | 배치 가능 수와 레벨을 마나로 상향(상위 성급 확률 동반 해금) |
| 상위 성급 확률 | `StarProbabilityConfig` (`PickStar`) | `Assets/Common/Data/Configs/StarProbabilityConfig.cs` | 배치 상한 레벨별 등장 성급 확률 테이블 |
| 마나 충전·이자 | `ManaIncomeConfig` | `Assets/Common/Data/Configs/ManaIncomeConfig.cs` | 라운드 시작 기본 마나·이자 규칙의 수치 출처 |
| 배치 상한 테이블 | `PlacementConfig` | `Assets/Common/Data/Configs/PlacementConfig.cs` | 초기 배치 상한·레벨업 비용/상한 테이블의 수치 출처 |
| 마켓 슬롯 (등장 1칸) | `MarketDefenderSlot` | `Assets/Scenes/Battle/Feature/Market/Scripts/MarketDefenderSlot.cs` | 등장한 소환수 데이터·초기 성급·판매 여부를 담는 값 객체 |
| 등장 목록 뽑기 | `MarketUnitRoller` | `Assets/Scenes/Battle/Feature/Market/Scripts/MarketUnitRoller.cs` | 코스트·성급 확률에 따라 마켓 슬롯들을 추첨 |
| 환원 영역 (드래그 판매) | `DefenderSellZone` | `Assets/Scenes/Battle/Feature/Market/Scripts/DefenderSellZone.cs` | 소환수 드래그가 판매 영역에 들어왔는지 판정해 환원 위임 |

> `OnManaNotEnoughDto` (`OnManaNotEnoughDto.cs`), `OnDefenderPlacementLimitDto` (`OnDefenderPlacementLimitDto.cs`) 는 도메인 개념이 아닌 통지용 빈 DTO/enum 자리표시자라 §3 구조 맵에만 둔다.

## 3. 구조 맵 (위치 중심)

- **진입점**: `MarketManager` (`SceneSingleton`) — `Assets/Scenes/Battle/Feature/Market/Scripts/MarketManager.cs`
- **페이즈 연동**: `MarketManager` 는 `IStateListener<PhaseType>` 로 `RoundManager` 에 등록 — 정비 페이즈에서만 상점 이용 가능(`IsMarketAvailable`) 상태가 된다
- **추첨 서브영역**: `MarketUnitRoller` — `MarketUnitRoller.cs` (코스트 분류·성급 확률 기반 슬롯 추첨)
- **슬롯 값 객체**: `MarketDefenderSlot` — `MarketDefenderSlot.cs`
- **환원 영역**: `DefenderSellZone` — `DefenderSellZone.cs` (드래그 종료 시 판매 영역 히트 판정)
- **통지 자리표시자**: `OnManaNotEnoughDto.cs`, `OnDefenderPlacementLimitDto.cs` (현재 빈 DTO)
- **수치 설정(외부)**: `ManaIncomeConfig` / `PlacementConfig` / `StarProbabilityConfig` — `Assets/Common/Data/Configs/`

### 3.2 의존 관계

- **밖으로 의존**:
  - `DefenderManager` (`Scenes.Battle.Feature.Unit.Defenders`) — 소환수 생성/제거·배치 수 조회
  - `RoundManager` / `PhaseType` (`Scenes.Battle.Feature.Rounds`) — 정비 페이즈 진입·라운드 인덱스
  - `GlobalEventBus` + `OnDefenderDragEventDto` (`Common.Scripts.GlobalEventBus`, `Scenes.Battle.Feature.Events`) — 소환수 드래그 이벤트 수신(환원 영역 토글)
  - `UnitLoadOutData` (`Common.Data.Units.UnitLoadOuts`) — 소환수 코스트·성급별 비용·환원 마나
  - `ManaIncomeConfig` / `PlacementConfig` / `StarProbabilityConfig` (`Common.Data.Configs`) — 마나·배치·성급 수치 출처
  - `BubbleMessageSpawner` (`Common.Scripts.BubbleMessage`) — 마나 부족 메시지 표시
  - `SceneSingleton`, `RxValue`, `IStateListener`, `RepeatX` (`Common.Scripts.*`) — 인프라 기반
- **밖에서 의존**: 마켓 UI(`Scenes.Battle.Feature.Ui` 의 MarketUiManager 등) 가 `MarketManager.Instance` 의 상태·액션·콜백을 소비한다.

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 경제 로직(`MarketManager`)과 등장 추첨(`MarketUnitRoller`)을 분리한 것은 "무엇을 뽑는가"(확률·코스트·성급)와 "그것으로 무엇을 하는가"(마나 정산·배치)를 별개 책임으로 둔 것이다. UI 표시는 의도적으로 `MarketUiManager` 쪽에 둔다(이 모듈은 상태와 콜백만 노출).
- **도메인적 의도**: 스캔 잠금(`IsScanLocked`)은 라운드 시작 시의 *자동* 재스캔만 스킵하기 위한 플래그다 — 잠금은 다음 라운드 자동 리롤을 한 번 막고 해제되는 일회성 의미를 가진다(코드상 라운드 시작 시 잠금이면 리롤을 건너뛰고 잠금을 푼다).
- **비자명한 제약/관례**: 모든 액션(`Reroll`/`BuyDefender`/`LevelUp`/`ToggleScanLock`)은 `IsMarketAvailable` 가 true(정비 페이즈)일 때만 동작한다 — 전투 페이즈에는 소환 터미널이 잠긴다. 또한 연승/연패 보너스, 코스트별 확률 분배는 현재 주석으로 보류된 미완 영역이라 실제 동작은 단순화돼 있다(콘텐츠 추가 시 복원 예정).
