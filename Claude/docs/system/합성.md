# 합성 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 기능(단순 구현) — 대응 UR 노드: `전장 > 소환수 > 합성(승급)`.

## 1. 한 줄 정의

정비 페이즈에서 배치된 소환수들을 검사하여 동일 종류·동일 성급 3개를 상위 성급으로 자동·연쇄 승급시키고, 3성 이상 타겟에 드래그 앤 드롭으로 동일 종류 재료를 소모해 강화(reinforcement)를 부여하는 합성 시스템.

## 3. 구조 맵 (위치 중심)

진입점부터.

- **자동 합성 진입점**: `DefenderFusionManager` 가 정비 페이즈 진입(`IStateListener<PhaseType>.OnStateEnter`) 및 디펜더 스폰 이벤트(`OnDefenderChange`)에서 `TryExecuteBasicFusion` 호출 — `Scripts/DefenderFusionManager.cs`
- **강화 합성 진입점**: `DefenderFusionManager.TryEnhanceFusion(material, target)` (public, UI 드래그 앤 드롭에서 호출) — `Scripts/DefenderFusionManager.cs`
- **기본 합성 그룹 탐지(순수 로직)**: `FusionGroupDetector.FindFusionGroup` / `HasFusionGroup` — `Scripts/FusionGroupDetector.cs`
- **강화 합성 쌍 탐지(순수 로직)**: `EnhancementFusionDetector.FindEnhancementPair` / `HasEnhancementPair` / `IsMaterialStar` — `Scripts/EnhancementFusionDetector.cs`
- **결과 표면**: `OnDefenderFusedEventDto` 발행(`GlobalEventBus`) + `BubbleMessageSpawner` 월드 메시지 — `Scripts/DefenderFusionManager.cs`

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도 (Detector ↔ Manager)**: 그룹/쌍 탐지(순수 C# 로직, Unity 무의존)를 `FusionGroupDetector`·`EnhancementFusionDetector`로 분리하고, 승급 대상 선정·제거·이벤트 발행 등 Unity 의존 동작만 `DefenderFusionManager`에 둔 것은 탐지 로직을 에디터 테스트로 검증 가능하게 하려는 의도(각 Detector 주석에 명시).
- **도메인적 의도 (승급 대상 선정 규칙)**: 기본 합성 시 3개 중 어느 것을 승급체로 남길지는 탐지 결과 인덱스가 아니라 **실시간 transform 위치**(위쪽 우선 → 왼쪽 우선)로 정한다. 합성 결과가 항상 일관된 자리에 남도록 하는 배치/연출상 약속으로, 탐지를 위치와 분리해 Manager에서 처리한다.
- **비자명한 제약/관례**:
  - 기본 합성은 `MaxBasicFusionStar`(현재 3) **미만** 성급에서만 일어난다 — 1성·2성만 자동 승급, 3성부터는 강화 합성 경로로 전환되는 기획(§2.2)을 코드 상수로 인코딩.
  - 강화 합성 재료 성급→강화량 매핑(2성→+1, 3성→+2)은 `GetReinforcementAmount`에 고정.
  - 합성은 정비 페이즈 전용. 전투 중 합성 조건이 충족되면 실행 대신 "전투 중에는 합성이 불가능합니다" 버블 메시지만 띄운다(사용자 피드백 목적).
  - 기본 합성은 스폰 이벤트(`DefenderChanges.Spawn`)에만 반응 — 신규 배치로 그룹이 완성되는 경우만 자동 트리거.
