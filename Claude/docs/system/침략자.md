# 침략자 — 시스템 관점

> 이 문서는 **AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록** 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실(single source of truth)이다. 이 문서는 코드를 재설명하지 않는다 — **"인덱스 + α"** 에 한정한다.
>
> - **분류**: 기능 — 대응 UR 노드: `전장 > 침략자 & 웨이브` (침략자(적) 진군·재타겟·재진군)
> - **이 문서가 다루는 것**: ① 이 대상이 시스템에서 무엇인가(한 줄) ② 도메인 개념 ↔ 코드(파일 경로)의 매핑 ③ 무엇이 어디에 있는가(구조 맵) ④ 다른 코드가 가져다 쓸 헬퍼 ⑤ 코드만 봐선 모르는 의도(+α).
> - **이 문서가 다루지 않는 것**: 코드의 동작을 풀어쓴 설명서, 알고리즘·자료구조 해설, 메서드 본문 요약. 그건 코드를 보면 된다.

---

## 1. 한 줄 정의

침략자는 레인을 따라 진군하는 적 유닛으로, 공용 유닛 컴포지션(이동·타게팅·행동 상태) 위에 "쓰러지면 풀로 회수"와 "웨이브 프리뷰 시 정지·등장 수 표시"라는 침략자 고유 행동만 얹은 얇은 컴포넌트 집합이다.

## 2. 도메인 ↔ 코드 매핑

| 도메인 개념 | 코드 식별자 | 위치(파일 경로) | 책임 (한 줄) |
|------------|------------|----------------|-------------|
| 침략자 본체 | `Aggressor` | `Assets/Scenes/Battle/Feature/Unit/Aggressor/Scripts/Aggressor.cs` | 행동 상태를 구독해 Downed 진입 시 자신을 풀로 환원하고, 비활성화 시 잔여 속도를 정리 |
| 침략자 쓰러짐(처치) → 퇴장 | `Aggressor.OnDown` → `Poolable.DeSpawn` | `Assets/Scenes/Battle/Feature/Unit/Aggressor/Scripts/Aggressor.cs` | 처치된 침략자를 파괴 대신 오브젝트 풀로 회수하는 도메인 퇴장 처리 |
| 웨이브 프리뷰 표시 | `AggressorPreview` | `Assets/Scenes/Battle/Feature/Unit/Aggressor/Scripts/AggressorPreview.cs` | 프리뷰 모드 진입 시 Freeze로 정지시키고 해당 그룹 등장 수(`x N`)를 표시 |
| 등장 수 라벨(단순 표기) | `AggressorSample` | `Assets/Scenes/Battle/Feature/Unit/Aggressor/Scripts/AggressorSample.cs` | 샘플/표기용 오브젝트에 등장 수 텍스트를 세팅 |
| 진군(이동) | `Mover` (Move 상태에서 좌향 진군) | `Assets/Scenes/Battle/Feature/Unit/Mover/Mover.cs` | 침략자가 의존하는 공용 이동 컴포넌트 — 진군의 실제 주체 |
| 재타겟·재진군 판정 | `ActionStateTransitionService.CheckTransition` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateTransitionService.cs` | 사거리 내 타겟 유무에 따라 Move↔Attack 전환(재타겟 시 멈춤, 타겟 소실 시 재진군)을 판정 |
| 행동 상태(진군/공격/쓰러짐 등) | `ActionStateType` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs` | 침략자가 거치는 행동 상태 enum(Idle/Move/Attack/Downed/Freeze/Waiting) |

> 진군·재타겟·재진군의 실제 동작 주체는 침략자 디렉토리가 아니라 공용 유닛 인프라(`Mover`·`Attacker`·`ActionStateController`)다. 침략자는 이를 컴포지션으로 끌어다 쓰고, 위 표 하단 3행은 그 진입점을 가리키는 네비게이션 항목이다.

- **진군의 주체(공용)**: `Mover` (Move 상태에서 `Vector2.left` 방향 진군) — `Assets/Scenes/Battle/Feature/Unit/Mover/Mover.cs`

- **타겟 보유·재탐색(공용)**: `Attacker.Victim` / `_victimsInRange` — `Assets/Scenes/Battle/Feature/Unit/Attacker/Attacker.cs`

### 3.2 의존 관계

- **밖으로 의존**:
  - `Common.Scripts.ObjectPool` — `Poolable` (처치 시 풀 환원)
  - `Common.Scripts.StateBase` — `IStateListener<T>`, `StateBaseController<T>` (행동 상태 구독 기반)
  - `Scenes.Battle.Feature.Units.ActionStates` — `ActionStateController`, `ActionStateType`, `ActionStateTransitionService` (진군/재타겟/재진군 상태 머신)
  - `Scenes.Battle.Feature.Units` (`Mover`) / `...Attackers` (`Attacker`) — 진군·타게팅의 실제 동작 주체
  - `TMPro` — `TextMeshPro` (등장 수 라벨)
- **밖에서 의존**:
  - 침략자 프리팹/풀 스폰 경로(웨이브 등장 로직)가 `Aggressor`·`AggressorPreview` 를 컴포넌트로 보유 — 프리뷰 활성/해제 및 처치 회수의 소비처
  - *주의*: 침략자 디렉토리 자체에는 외부 C# 소비처가 거의 없다. 진군·전투는 공용 인프라가, 스폰·웨이브 편성은 웨이브 측 코드가 담당한다.

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 침략자는 "적 전용 동작"을 새로 구현하지 않고, 디펜더·소환수와 동일한 공용 컴포지션(`Unit`+`Mover`+`Attacker`+`ActionStateController`) 위에 *차이점만* 얹는다. 침략자 고유 코드가 처치 시 풀 환원(`OnDown`)과 프리뷰 표시(`AggressorPreview`)뿐인 이유다. 진군·재타겟·재진군 로직을 이 디렉토리에서 찾으려 하지 말고 ActionState/Mover/Attacker 로 가야 한다.
- **도메인적 의도**: "진군"은 별도 경로 추종 로직이 아니라 Move 상태에서 좌향(`Vector2.left`) 등속 이동으로 표현된다 — 침략자는 레인을 따라 한 방향으로 밀려오는 존재라는 도메인 전제가 코드에 직접 박혀 있다. "재타겟→정지→재진군"은 사거리 내 타겟 유무에 따른 Move↔Attack 전환으로, 별도 AI가 아니라 공용 상태 전환 규칙의 산물이다.
- **비자명한 제약/관례**: 처치된 침략자는 `Destroy` 가 아니라 `Poolable.DeSpawn` 으로 회수된다(웨이브당 대량 스폰 전제의 풀링 관례). 프리뷰는 행동 상태를 `Freeze` 로 강제해 정지시키므로, 프리뷰 중에는 진군·전투 상태 전환이 일어나지 않는다.
