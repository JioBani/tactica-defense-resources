# 스킬 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 기능(단순 구현) — 대응 UR 노드: `전장 > 소환수 > 소환수 동작 (전투)` 중 스킬(쿨타임) 부분.

## 1. 한 줄 정의

소환수의 능동 행동(평타·스킬)을 "캐스트(Castable) → 실행(Executable)" 2단계로 분리하고, 스킬은 쿨타임 게이트와 시전 가능 조건(`CanCast`)을 거쳐 정의 데이터별 효과를 부여하는 시전(Casting) 시스템.

## 2. 도메인 ↔ 코드 매핑

| 도메인 개념 | 코드 식별자 | 위치(파일 경로) | 책임 (한 줄) |
|------------|------------|----------------|-------------|
| 시전 1회의 추상(평타·스킬 공통) | `Castable` (추상) | `Casting/Castable/Castable.cs` | 시전 1회를 `Casting()`으로 만들고 캐스트/실행/실행종료 이벤트를 발행하는 베이스 |
| 시전이 만들어 낸 효과 처리 단위 | `Executable` (추상) | `Casting/Executable/Executable.cs` | 캐스트가 생성한 실제 효과(데미지 등) 적용·종료를 책임지는 베이스 |
| 평타(원거리 공격) 시전 | `AttackCast` | `Casting/Castable/AttackCast.cs` | 평타 캐스트 — 발사체 생성 후 명중 시 실행기 가동 |
| 평타 데미지 적용 | `RangeAttackExecutor` | `Casting/Executable/RangeAttackExecutor.cs` | 물리 데미지 계산 후 피격자에 적용 |
| 평타 주기 구동(사거리·공속) | `AttackCaster` | `Casting/Caster/AttackCaster.cs` | 최근접 타겟 포착·공속 주기로 평타 캐스트 반복 |
| 스킬 시전(쿨타임 기반) | `SkillCaster` | `Casting/Caster/SkillCaster.cs` | 스킬 쿨다운 관리·시전 가능 시 캐스트 트리거 |
| 스킬 캐스트 추상 | `SkillCast` (추상) | `Skills/Scripts/Skills/SkillCast.cs` | `Castable`을 스킬 도메인으로 좁힌 베이스 |
| 개별 스킬(예: 불화살) | `FireArrow` / `FireArrowExecutor` | `Skills/Scripts/Skills/FireArrow/` | 타겟 존재 조건 검사 후 정의 계수로 마법 데미지 부여 |
| 스킬 정의 → 인스턴스 라우팅 | `SkillFactory.CreateSkill` | `Skills/Scripts/SkillFactory.cs` | 정의 데이터 타입별 `SkillCast` 생성 (switch 분기) |
| 스킬 생성 입력 묶음 | `SkillCreateContext` | `Skills/Scripts/Skills/SkillCreateContext.cs` | 팩토리 생성 시 정의·시전자·공격자 전달 |

> `CanExecuteContext` / `ExecuteContext` / `InitializeContext` (`Skills/Scripts/Contexts/`) 는 현재 위 흐름에서 직접 소비되지 않는 컨텍스트 정의로 §3 구조 맵에만 둔다.

## 3. 구조 맵 (위치 중심)

진입점부터.

- **스킬 진입점**: `SkillCaster.Update` → `_skill.CanCast()` 충족 시 `Cast()` — `Casting/Caster/SkillCaster.cs` (쿨타임 준비 + 시전 조건이 동시에 참일 때만)
- **평타 진입점**: `AttackCaster.Attack` (공속 `DynamicRepeater` 주기) — `Casting/Caster/AttackCaster.cs`
- **캐스트 베이스**: `Castable.Cast` → `Casting()` → `OnCastEvent` — `Casting/Castable/Castable.cs`
- **실행 베이스**: `Executable.Execute` → `Executing()` / `EndExecute` → `EndExecuting()` — `Casting/Executable/Executable.cs`
- **평타 구현**: `AttackCast` + `RangeAttackExecutor` — `Casting/Castable/`, `Casting/Executable/`
- **스킬 구현**: `SkillCast` 상속 (`FireArrow`) + `Executable` 상속 (`FireArrowExecutor`) — `Skills/Scripts/Skills/`
- **스킬 생성 라우팅**: `SkillFactory` (SceneSingleton, 정의 타입 switch) — `Skills/Scripts/SkillFactory.cs`
- **미사용/예비 컨텍스트**: `CanExecuteContext`·`ExecuteContext`·`InitializeContext` — `Skills/Scripts/Contexts/`

### 3.2 의존 관계

- **밖으로 의존**:
  - 발사체 — `ProjectileGenerator` (`Feature/Projectiles`)
  - 데미지 계산 — `DamageCalculator`, `SkillCoefficientCalculator` (`Feature/Units/Damage`, `Common/Data/Skills`)
  - 스킬 정의 데이터(SO) — `SkillDefinitionData` / `FireArrowDefinitionData` (`Common/Data/Skills/SkillDefinitions`)
  - 타이머 인프라 — `TimerManager`/`Timer` (쿨타임), `DynamicRepeater` (공속 주기)
  - 페이즈·행동 상태 — `RoundManager`/`PhaseType`, `ActionStateController`/`ActionStateType` (`IStateListener` 구독)
  - 타겟·피격 — `Attacker`/`Victim` (`Feature/Units/Attackers`, `Feature/Units/Attackables`)
- **밖에서 의존**: 소환수 유닛 프리팹(`Unit`)에 `SkillCaster`·`AttackCaster`를 부착해 사용. UI/타 시스템의 직접 소비처는 없음.

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: `Castable`(시전 1회 트리거·이벤트 발행)과 `Executable`(효과 적용)을 분리한 것은 시전 시점과 효과 발생 시점을 떼어내기 위함이다 — 발사체가 날아가 명중하는 순간에 데미지가 발생하므로, 캐스트 즉시 쿨다운을 멈췄다가(`OnCast`) 실행 종료 시 재개(`OnExecuteEnd`)하는 타이밍 제어가 가능하다.
- **도메인적 의도**: 평타와 스킬을 `Castable` 공통 추상 아래 두어, "소환수의 능동 행동"이라는 동일 도메인 개념으로 일원화했다. 차이는 게이트뿐이다 — 평타는 공속 주기, 스킬은 쿨타임 + `CanCast` 조건.
- **비자명한 제약/관례**:
  - 쿨타임 게이트는 "타이머 준비(`_isSkillReady`)"와 스킬별 "시전 가능(`CanCast`)"의 AND다. `FireArrow.CanCast`는 타겟(Victim) 존재를 요구하므로, 쿨타임이 차도 타겟이 없으면 시전되지 않는다.
  - 다운(`Downed`) 상태 진입 시 스킬은 차단되고 타이머가 정지한다 — 상태 머신 구독으로 처리되며 스킬 코드 자체엔 드러나지 않는다.
  - 새 스킬 추가는 `SkillCast` 상속 + `SkillFactory` switch 분기 등록 두 곳을 함께 손대야 한다 (미등록 타입은 `ArgumentException`).
  - `AttackCast.Casting()`은 발사체에 실행기를 연결한 뒤 별도의 `RangeAttackExecutor`를 새로 만들어 반환한다 — 반환 인스턴스와 명중 시 실행되는 인스턴스가 다르다(코드 외 의도라기보단 주의 지점).
