# 비동기와타이밍 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 인프라 — UR 미매핑 횡단 기반. **여러 작은 비동기 실행·타이머·주기 반복 유틸을 한 묶음으로 인덱싱한 문서**다. 특정 사용자 기능에 매핑되지 않고, 다수 기능이 공통으로 올라타는 타이밍·비동기 실행 기반이다. 각 유틸은 본래 클래스 단위라 일반 SR 에는 안 들어갈 내용이나, 자주 찾아오는 공통 기반이라 인덱스 형태로 묶어 문서화한다.

## 1. 한 줄 정의

전투/UI 로직이 공통으로 쓰는 카운트다운 타이머, 순차 비동기 큐, 가변 주기 반복기, 비동기 작업 핸들, 활성화 수명 콜백 바인더 등 **작은 비동기·타이밍 유틸의 모음(인덱스)** — 각 유틸은 독립적이며 서로 의존하지 않는다.

## 2. 도메인 ↔ 코드 매핑 (주요 추상)

인프라 묶음이라 도메인 개념보다 "유틸 → 한 줄 책임"의 인덱스로 읽는다. 설명 열은 SR 수준(책임), 위치 열은 네비게이션(경로·타입).

| 유틸 (주요 추상) | 코드 식별자 | 위치(파일 경로) | 책임 (한 줄) |
|------------|------------|----------------|-------------|
| 카운트다운 타이머 | `Timer` | `Assets/Common/Scripts/Timer/Timer.cs` | MaxTime→0 으로 흐르는 단일 카운트다운의 상태·만료 통지를 가진 순수 C# 타이머 |
| 타이머 수명/구동 관리 | `TimerManager` | `Assets/Common/Scripts/Timer/TimerManager.cs` | 씬 싱글턴으로 Timer 들을 생성·삭제하고 매 프레임 구동하는 MonoBehaviour |
| 채널별 전역 순차 큐 | `GlobalTaskQueue` | `Assets/Common/Scripts/TaskQueue/GlobalTaskQueue.cs` | 채널 단위로 비동기 태스크를 순차 실행하도록 큐를 분배하는 정적 진입점 |
| 인스턴스 순차 큐 | `TaskQueue` | `Assets/Common/Scripts/TaskQueue/TaskQueue.cs` | 등록된 비동기 태스크를 한 번에 하나씩 순차 실행하는 인스턴스 큐 |
| 큐잉 대상 계약 | `IQueuedTask` / `QueuedTask` | `Assets/Common/Scripts/TaskQueue/IQueuedTask.cs`, `QueuedTask.cs` | 큐에 넣을 비동기 작업의 인터페이스와 람다 래퍼 |
| 큐 채널 구분 | `TaskQueueChannel` | `Assets/Common/Scripts/TaskQueue/TaskQueueChannel.cs` | 전역 큐를 분리하는 채널 식별 enum |
| 비동기 작업 핸들 | `UniTaskHandle` / `UniTaskHandle<T>` | `Assets/Common/Scripts/UniTaskHandle/UniTaskHandle.cs` | 단발 await 되는 UniTask 의 완료·실패·결과 상태를 여러 곳에서 폴링하게 감싸는 핸들 |
| 가변 주기 반복기 | `DynamicRepeater` | `Assets/Common/Scripts/DynamicRepeater/DynamicRepeater.cs` | 외부가 정하는 현재 간격마다 비동기 작업을 반복하는 취소 가능 반복기 |
| 횟수 반복 헬퍼 | `RepeatX` | `Assets/Common/Scripts/RepeatX/RepeatX.cs` | N회 반복 실행·수집을 인덱스 콜백으로 표현하는 정적 헬퍼 |
| 활성화 수명 콜백 바인더 | `CallbackLifetimeBinder` | `Assets/Common/Scripts/CallbackLifetimeBinder/CallbackLifetimeBinder.cs` | 오브젝트 활성/비활성에 짝지은 콜백을 등록·자동 호출하는 MonoBehaviour 베이스 |

> 이 묶음은 도메인 개념 매핑이 아니라 기술 유틸 인덱스다. 각 유틸의 "이럴 때 쓴다" 진입점은 §3.1 을 본다.

## 3. 구조 맵 (위치 중심)

각 유틸은 독립 폴더이며 상호 의존이 없다. 폴더별로 본다.

- **Timer** — `Timer`(순수 카운트다운) + `TimerManager`(씬 싱글턴 구동) 2종. 진입점: `TimerManager.Make(maxTime, onTimeOutChange)` → `Timer/TimerManager.cs`
- **TaskQueue** — 채널 분배(`GlobalTaskQueue`) → 인스턴스 큐(`TaskQueue`) → 작업 계약(`IQueuedTask`)·람다 래퍼(`QueuedTask`)·채널 enum(`TaskQueueChannel`). 진입점: `GlobalTaskQueue.Enqueue(channel, task)` → `TaskQueue/GlobalTaskQueue.cs`
- **ActionQueue** — `Assets/Common/Scripts/ActionQueue/` 폴더는 **비어 있다** (소스 없음). 현재 순차 실행은 TaskQueue 가 담당한다.
- **UniTaskHandle** — `UniTaskHandle`/`UniTaskHandle<T>` 와 변환 확장 `UniTaskHandleExtensions`. 진입점: `task.ToHandle()` 확장 → `UniTaskHandle/UniTaskHandle.cs`
- **DynamicRepeater** — 단일 클래스 `DynamicRepeater`(`IDisposable`). 진입점: `new DynamicRepeater(intervalNow, job)` + `Start()` → `DynamicRepeater/DynamicRepeater.cs`
- **RepeatX** — 단일 정적 클래스 `RepeatX`. 진입점: `RepeatX.Times(count, action)` → `RepeatX/RepeatX.cs`
- **CallbackLifetimeBinder** — 단일 MonoBehaviour 베이스. 진입점: 상속 후 `Bind(onEnable, onDisable)` → `CallbackLifetimeBinder/CallbackLifetimeBinder.cs`

### 3.1 제공 헬퍼 (다른 코드가 찾아오는 곳)

| 헬퍼 | 위치(파일 경로) | 이럴 때 쓴다 |
|------|----------------|-------------|
| `TimerManager.Make` | `Assets/Common/Scripts/Timer/TimerManager.cs` | 만료 콜백이 달린 카운트다운 타이머가 필요할 때 (매니저가 구동·정리까지 담당) |
| `Timer` | `Assets/Common/Scripts/Timer/Timer.cs` | 엔진 루프와 무관하게 직접 Update 를 돌릴 순수 타이머가 필요할 때 |
| `GlobalTaskQueue.Enqueue` | `Assets/Common/Scripts/TaskQueue/GlobalTaskQueue.cs` | 채널(예: 전투 UI 연출)별로 비동기 작업을 겹치지 않게 순차 실행하고 싶을 때 |
| `TaskQueue` | `Assets/Common/Scripts/TaskQueue/TaskQueue.cs` | 전역 채널 없이 한정된 범위에서만 순차 비동기 큐가 필요할 때 |
| `QueuedTask` | `Assets/Common/Scripts/TaskQueue/QueuedTask.cs` | `Func<UniTask>` 람다를 큐에 바로 넣고 싶을 때 (IQueuedTask 구현 생략) |
| `task.ToHandle()` | `Assets/Common/Scripts/UniTaskHandle/UniTaskHandle.cs` | UniTask 를 한 번만 await 하면서도 완료/실패/결과를 여러 지점에서 조회해야 할 때 |
| `DynamicRepeater` | `Assets/Common/Scripts/DynamicRepeater/DynamicRepeater.cs` | 런타임에 바뀌는 주기로 비동기 작업을 반복하고, 토큰/Dispose 로 안전 종료하고 싶을 때 |
| `RepeatX.Times` | `Assets/Common/Scripts/RepeatX/RepeatX.cs` | for 루프 대신 "N회 실행" 또는 "N개 생성·수집"을 선언적으로 쓰고 싶을 때 |
| `CallbackLifetimeBinder` | `Assets/Common/Scripts/CallbackLifetimeBinder/CallbackLifetimeBinder.cs` | OnEnable/OnDisable 짝을 수동 작성하지 않고 활성 수명에 콜백을 묶고 싶을 때 상속 |

### 3.2 의존 관계

- **밖으로 의존**: `Cysharp.Threading.Tasks`(UniTask — TaskQueue·UniTaskHandle·DynamicRepeater), `Common.Scripts.SceneSingleton`(`SceneSingleton<T>` — TimerManager), UnityEngine
- **밖에서 의존**: 전투 UI·연출 흐름(TaskQueue `BattleUi` 채널), 주기 발생 로직(DynamicRepeater), 카운트다운 표시(Timer) 등 다수 기능 코드. 유틸 간 상호 의존은 없다.

## 4. +α — 코드만 봐선 모르는 것

- **묶음 의도**: 이들은 기능적으로 한 시스템이 아니라 "작고 독립적인 비동기·타이밍 유틸"이라는 공통 성격만 공유한다. 흩어진 폴더를 개별 SR 문서로 만들면 과하므로 한 인프라 인덱스로 묶었다 — 검색 진입점 통합이 목적이다.
- **분할 의도 (Timer ↔ TimerManager)**: 타이머 로직을 MonoBehaviour 에서 떼어 순수 C# 으로 둔 것은 엔진 루프·씬에서 독립시키기 위함이고, 매 프레임 구동·수명 관리라는 엔진 책임은 `TimerManager` 한쪽에 모았다.
- **분할 의도 (TaskQueue 채널화)**: 순차 실행을 전역 단일 큐가 아니라 채널별로 나눈 것은, 서로 무관한 비동기 흐름(예: 전투 UI 연출)이 한 큐에서 불필요하게 서로를 기다리지 않게 격리하기 위함이다.
- **ActionQueue 부재**: `ActionQueue` 폴더는 자리만 있고 비어 있다. 순차 비동기 실행 수요는 TaskQueue 가 담당한다 — 새 큐를 만들기 전에 TaskQueue 재사용을 먼저 검토한다.
- **DynamicRepeater 의 가변 간격**: 간격을 값이 아니라 `Func<TimeSpan>` 로 받는 것은, 반복 도중 외부 상태 변화(예: 가속·감속)를 다음 틱에 즉시 반영하기 위함이다.
- **비자명한 관례**: `DynamicRepeater.RunAsync` 안에 디버그 로그(`Debug.Log("RunAsync")`)가 남아 있다 — 운영용 로그가 아니라 개발 흔적으로 보인다.
