# 씬수명과접근 — 시스템 관점

> 이 문서는 **AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록** 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실(single source of truth)이다. 이 문서는 코드를 재설명하지 않는다 — **"인덱스 + α"** 에 한정한다.
>
> - **분류**: 인프라 — UR 미매핑 횡단 기반 (매니저 싱글톤 베이스·씬 간 데이터 전달).
> - **이 문서가 다루는 것**: ① 이 대상이 시스템에서 무엇인가(한 줄) ② 주요 추상 ↔ 코드(파일 경로)의 매핑 ③ 무엇이 어디에 있는가(구조 맵) ④ 다른 코드가 가져다 쓸 헬퍼 ⑤ 코드만 봐선 모르는 의도(+α).
> - **이 문서가 다루지 않는 것**: 코드의 동작을 풀어쓴 설명서, 알고리즘·자료구조 해설, 메서드 본문 요약.

---

## 1. 한 줄 정의

씬 단위로 단 하나만 존재하는 매니저류 컴포넌트에 **전역 접근점과 수명(중복 차단·씬 전환 유지)** 을 부여하는 베이스(`SceneSingleton<T>`)와, 그 위에 올라타 **씬 경계를 넘어 데이터를 전달**하는 전역 컨테이너(`SceneDataManager`)를 묶은 인프라다.

## 3. 구조 맵 (위치 중심)

- **진입점(접근)**: `SceneSingleton<T>.Instance` — `Assets/Common/Scripts/SceneSingleton/SceneSingleton.cs`
- **진입점(데이터)**: `SceneDataManager.Instance.selectedBattlefield` — `Assets/Common/Scripts/SceneDataManager/SceneDataManager.cs`
- **수명 관리**: `SceneSingleton<T>` 의 `Awake` / `OnDestroy` / `OnApplicationQuit` 와 `_dontDestroyOnLoad` 직렬화 필드 — 씬 유지 여부·종료 시 접근 차단 — `Assets/Common/Scripts/SceneSingleton/SceneSingleton.cs`
- **전달 셀**: `SceneData<T>` — 값 + 보유 플래그 한 쌍, Inspector 노출 — `Assets/Common/Scripts/SceneDataManager/SceneData.cs`
- **일괄 초기화**: `SceneDataManager.ClearAll()` — `Assets/Common/Scripts/SceneDataManager/SceneDataManager.cs`

### 3.2 의존 관계

- **밖으로 의존**: UnityEngine (`MonoBehaviour`, `DontDestroyOnLoad`, `FindFirstObjectByType`); `SceneDataManager` 는 `SceneSingleton<T>` 와 `Common.Data.Battlefields.BattlefieldData` 에 의존.
- **밖에서 의존**: `SceneSingleton<T>` 를 상속하는 다수 매니저 — `MarketManager`·`SynergyManager`·`SummonerManager`·`DefenderFusionManager`·`SkillFactory`·`SynergyControllerFactory`·`ObjectPooler`·`TimerManager`·`AlertManager`·`ProjectileGenerator`·`BubbleMessageSpawner`·`WaitingAreaReferences`·`ObjectPoolingReferences` 등. `SceneDataManager` 소비처 — `RoundManager`(전장 정보 읽기), `BattleWinUiManager`·`BattleLoseUiManager`(`ClearAll`).

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 접근/수명(`SceneSingleton<T>`)과 데이터 운반(`SceneDataManager`/`SceneData<T>`)을 분리했다. 전자는 "단일 인스턴스 + 전역 접근"이라는 횡단 관심사를 모든 매니저에 재사용시키는 베이스이고, 후자는 그 베이스에 올라탄 단 하나의 구체 사용처(씬 간 값 전달)다 — 데이터 전달이 늘면 `SceneDataManager` 에 슬롯이 추가되는 구조.
- **도메인적 의도**: `selectedBattlefield` 는 전장 선택→전투 씬 진입이라는 게임 흐름(전장 입장)의 컨텍스트를 씬 경계 너머로 넘기기 위한 채널이다. 도메인 자체가 아니라 도메인 흐름을 잇는 배관.
- **비자명한 제약/관례**: ① 파생 매니저는 `Awake` 를 직접 쓰지 말고 `OnAwakeSingleton()` 훅을 쓴다 (베이스가 `Awake` 에서 싱글톤 확정·`DontDestroyOnLoad` 를 처리하므로). ② `_dontDestroyOnLoad` 기본값이 true 라 별도 설정 없이 씬 전환에도 살아남는다 — 씬 한정 매니저는 Inspector 에서 꺼야 한다. ③ `Instance` 는 종료 중이거나 씬에 없으면 `null` 을 줄 수 있다(보장된 자동 생성이 아님). ④ 테스트는 `SceneSingleton<T>._instance` 정적 필드를 리플렉션으로 초기화한다 (`Assets/Tests/Editor/`), 즉 정적 상태가 테스트 간 누수될 수 있다는 전제.
