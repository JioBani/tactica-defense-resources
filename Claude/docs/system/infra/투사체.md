# 투사체 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 인프라 — UR 미매핑 횡단 기반. 원거리 공격(`AttackCast`)과 투사체형 스킬(`FireArrow`) 등 여러 전투 행동이 공통으로 올라타는 "타겟까지 날아가 명중 시점을 알리는" 전송 기반이다. 특정 사용자 기능 한 잎에 귀속되지 않고, 데미지·효과 적용은 호출자(executor)에게 맡긴다. 본래 클래스 단위라 일반 SR 에는 안 들어갈 내용이나, 공통 기반이라 별도 문서화한다.

## 1. 한 줄 정의

발사 지점에서 타겟 Transform 까지 이동하다 근접하면 명중 시점을 `OnHit` 이벤트로 통지하고 스스로 풀에 반환하는, 풀링된 투사체 전송 기반.

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 투사체는 "전송과 명중 시점"만 책임지고 데미지·효과는 `OnHit` 구독자(executor)에게 위임한다. 그래서 평타든 스킬이든 같은 투사체를 재사용하고, 결과는 호출자가 갈아끼운다.
- **도메인적 의도**: 명중을 충돌 물리가 아니라 타겟 근접 거리(`hitRadius`)로 판정한다 — 호밍 투사체가 빗나가지 않고 도착을 보장해야 하는 타워디펜스 전투 톤을 반영한다. `maxLifetime` 은 타겟이 사라진 경우의 누수 방지 안전장치다.
- **비자명한 제약/관례**: 현재 `Shot` 은 `Attacker` 전용에서 `Transform` 기반으로 일반화하는 과도기다(주석 처리된 구버전 시그니처·TODO 참조) — 비-attacker 투사체로 확장 예정. `OnHit` 은 `OnDisable` 에서 해제되므로 풀 반환 후 재사용 시 구독이 남지 않는다.
