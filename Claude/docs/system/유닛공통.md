# 유닛공통 — 시스템 관점

> 이 문서는 AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실이다. 이 문서는 코드를 재설명하지 않는다 — "인덱스 + α"에 한정한다.
>
> - **분류**: 기능(단순 구현) — 대응 UR 노드: `전장 > 유닛 스탯 & 데미지 > 능력치 & 스탯 구조`.

## 1. 한 줄 정의

소환수·소환술사·침략자가 공유하는 유닛 베이스로, 능력치 시트(기본/추가/최종·수정자)를 단일 진실로 보유하고 그 변경을 체력바·이동·애니메이션 컴포넌트에 이벤트로 흘려보내는 책임을 진다.

## 3. 구조 맵 (위치 중심)

무엇이 어디에 있는가. 진입점부터.

- **스폰 진입점**: `Unit.SetSpawn(UnitLoadOutData, star)` → `StatSheet.Init` — `Unit/Scripts/Unit.cs` (스탯·성급·체력바·역할군 아이콘 초기화, `OnSpawnEvent` 발행)
- **생성 라우팅**: `UnitGenerator.GenerateDefender / GenerateAggressor / GenerateSummoner / GenerateAggressorSample` — `Unit/UnitGenerator.cs` (종류별 프리팹 + 오브젝트 풀)
- **스탯 구조(핵심)**: `Unit/UnitStats/` — `UnitStat`(단일 능력치·수정자·합산식), `StatModifier`/`StatModifierType`(수정자), `UnitStatSheet`(14종 집합·성급/강화·현재 체력)
  - 합산식: `StatCalculationMode.Additive`(기본) / `SeparatedMultiplicative`(방어력에 사용)
  - 성급/강화 갱신: `UnitStatSheet.UpgradeStar` / `Reinforce` → 기존 수정자 보존 후 기본값만 갱신
- **체력바**: `HealthBar.Display / SetStarGrade / SetRoleGroupIcon` — `HealthBar/Scripts/HealthBar.cs`
- **이동**: `Mover` (`IStateListener<ActionStateType>`) — `Mover/Mover.cs`
- **애니메이션**: `UnitAnimator : StateAnimator<ActionStateType>` — `Animation/UnitAnimator.cs`
- **빈 스텁**: `UnitManager` — `Unit/Scripts/UnitManager.cs`

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 능력치 1종(`UnitStat`)을 "기본값 + 출처 귀속 수정자 → 최종값" 단위로 독립시켜, 시너지·상태이상·전술지원이 자기 출처로만 수정자를 붙였다 떼며 서로의 효과를 깨지 않게 하려는 것. 현재 체력만 수정자 체계 밖(`RxValue`)에 둔 것은 그것이 누적 상태 값이지 합산되는 능력치가 아니기 때문이다.
- **도메인적 의도**: 소환수·소환술사·침략자가 같은 14종 능력치·성급 체계를 공유한다는 기획을 단일 `Unit`/`UnitStatSheet` 베이스로 인코딩한 것. 방어력만 증감 분리 곱연산(`SeparatedMultiplicative`)을 쓰는 것은 방어 감소가 무한 누적되지 않도록 증가/감소를 분리해 곱하는 도메인 규칙을 반영한다.
- **비자명한 제약/관례**: 성급 + 강화 단계를 합친 `effectiveStar`로 기본 스탯을 조회하며, 승급(`UpgradeStar`)은 강화 단계를 0으로 리셋한다. `UpgradeStar`/`Reinforce`는 기존 수정자를 보존한 채 기본값만 다시 깐다. `UnitStatSheet.Init`은 중복 구독 방지를 위해 `MaxHealth.OnChange` 핸들러를 해제 후 재등록한다(재사용 풀 객체 대비).
