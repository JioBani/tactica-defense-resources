# 공격과타겟팅 — 시스템 관점

> 이 문서는 **AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록** 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실(single source of truth)이다. 이 문서는 코드를 재설명하지 않는다 — **"인덱스 + α"** 에 한정한다.
>
> - **분류**: 기능(단순 구현) — 대응 UR 노드: `전장 > 소환수 > 소환수 동작 (전투)`
> - **이 문서가 다루는 것**: ① 이 대상이 시스템에서 무엇인가(한 줄) ② 도메인 개념 ↔ 코드(파일 경로)의 매핑 ③ 무엇이 어디에 있는가(구조 맵) ④ 다른 코드가 가져다 쓸 헬퍼 ⑤ 코드만 봐선 모르는 의도(+α).
> - **이 문서가 다루지 않는 것**: 코드의 동작을 풀어쓴 설명서, 알고리즘·자료구조 해설, 메서드 본문 요약.

---

## 1. 한 줄 정의

소환수가 사거리 안의 적을 감지·타겟팅하고, 공격 속도에 맞춰 반복 공격을 발사하며, 적중 사실을 외부(상태이상 등)에 통지하는 전투 동작의 진입 컴포넌트다.

## 3. 구조 맵 (위치 중심)

- **진입점**: `Attacker` 컴포넌트 — `Assets/Scenes/Battle/Feature/Unit/Attacker/Attacker.cs`
- **타겟 감지/이탈**: `OnTriggerEnter2D` / `OnTriggerExit2D` (`CircleCollider2D` 트리거 기반) — `Attacker.cs`
- **공격 실행 위임**: `Attack` → `AttackCast` — `Assets/Scenes/Battle/Feature/Unit/Casting/Castable/AttackCast.cs` (투사체 발사 후 `RangeAttackExecutor`로 적중 처리; 본 기능 범위 밖)
- **상태 연동**: `IStateListener<ActionStateType>` 구현 — `Attack` 상태 진입/이탈에 반복 시작·정지, 실행 중 타겟 다운 감지 시 재획득. `ActionStateController` — `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateController.cs`
- **피격 처리**: `Victim.Hit(AttackContext)` — `Assets/Scenes/Battle/Feature/Unit/Attacker/Victim.cs`
- **적중 데이터**: `AttackContext` — `Assets/Scenes/Battle/Feature/Unit/Attacker/AttackContext.cs`

### 3.2 의존 관계

- **밖으로 의존**:
  - `DynamicRepeater` (`Common/Scripts/DynamicRepeater/DynamicRepeater.cs`) — 공격 속도 기반 반복 구동
  - `ActionStateController` / `IStateListener<ActionStateType>` (`Scenes/Battle/Feature/Unit/ActionState/`) — `Attack` 상태와 동기화
  - `AttackCast` (`Scenes/Battle/Feature/Unit/Casting/Castable/AttackCast.cs`) — 실제 공격 캐스팅 위임
  - `Unit.StatSheet`(`AttackRange`, `AttackSpeed`, `Health`) — 사거리·공격속도·체력 스탯 소스
- **밖에서 의존**:
  - **상태이상 프레임워크 (StatusEffect) — `OnAttackHitEvent` 훅 소비**: `AttackHitHookBinder` / `OnAttackHitHookProvider` / `IOnAttackHitHook` (`Common/Scripts/StatusEffect/HookProvider/AttackHitHookBinder.cs`) 가 `Attacker.OnAttackHitEvent` 를 구독하여 `IOnAttackHitHook` SE들에게 `OnAttackHit(context)` 로 전달한다. → infra: [StatusEffect](./infra/StatusEffect.md)
  - 이 훅을 사용하는 대표 SE: `GunslingerSynergyStatusEffect`, `WarmongerSynergyStatusEffect`, `FreljordSynergyStatusEffect` (`Scenes/Battle/Feature/Synergy/Scripts/SynergyEffects/`)

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: `Attacker`(주체)와 `Victim`(피격)을 별도 컴포넌트로 분리한 것은 "공격에 성공함"과 "피격당함"을 서로 다른 이벤트 표면으로 두기 위함이다 — 현재 `Victim` 측 피격 이벤트(`OnGetHit`)는 미구현이며 반격·가시·피격 VFX 등을 위한 확장 지점으로 남겨져 있다(`Victim.cs` TODO).
- **도메인적 의도**: 사거리를 `CircleCollider2D` 반경으로 인코딩한 것은 기획의 "원형 거리" 사거리 모델을 물리 트리거로 구현한 것이다. 타겟 선정은 "최근접"이 아니라 사거리 진입 순서 기반 첫 유효 후보로 구현되어 있어, 기획의 최근접 타겟팅과는 차이가 있다(현 구현 한계).
- **비자명한 제약/관례**: `AttackContext.Source` 의 null/non-null 구분이 일반 공격과 SE 유발 추가 피해를 가르는 관례적 신호다 — SE가 다시 적중 훅을 듣고 추가 피해를 넣을 때 이 필드로 무한 루프/재귀 증폭을 구분하도록 의도되어 있다. 또한 `AttackContext.Victim` 은 현재 non-null 전제이나 논타겟/범위 공격 도입 시 nullable 화가 필요하다(`AttackContext.cs` TODO).
