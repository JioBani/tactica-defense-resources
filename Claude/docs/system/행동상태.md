# 행동상태 — 시스템 관점

> 이 문서는 **AI 에이전트가 코드 레이어에 맞게 이해하고 빠르게 코드를 찾아가도록** 하는 시스템 관점 컨텍스트 문서다.
> 코드가 단일 진실(single source of truth)이다. 이 문서는 코드를 재설명하지 않는다 — **"인덱스 + α"** 에 한정한다.
>
> - **분류**: 기능 — 대응 UR 노드: `전장 > 소환수 > 소환수 동작 (전투)`. 유닛 단위 행동 상태(Idle/Move/Attack/Downed/Freeze/Waiting) 머신으로, 횡단 인프라인 `StateBase` 위에 올라타되 소환수(및 유닛) 전투 행동에 특화되어 있어 기능으로 분류한다. `StateBaseController<T>` 를 상속·소비할 뿐 그 자체를 제공하지 않으므로 인프라가 아니다.
> - **이 문서가 다루는 것**: ① 이 대상이 시스템에서 무엇인가(한 줄) ② 도메인 개념 ↔ 코드(파일 경로)의 매핑 ③ 무엇이 어디에 있는가(구조 맵) ④ 다른 코드가 가져다 쓸 헬퍼 ⑤ 코드만 봐선 모르는 의도(+α).
> - **이 문서가 다루지 않는 것**: 코드의 동작을 풀어쓴 설명서, 알고리즘·자료구조 해설, 메서드 본문 요약.

---

## 1. 한 줄 정의

`행동상태` 는 한 유닛의 전투 행동을 6개 상태(Idle/Move/Attack/Downed/Freeze/Waiting)로 표현하고, 매 프레임 조건 판정과 라운드 페이즈 통지에 따라 그 상태를 전환할 책임을 진다.

## 2. 도메인 ↔ 코드 매핑

| 도메인 개념 | 코드 식별자 | 위치(파일 경로) | 책임 (한 줄) |
|------------|------------|----------------|-------------|
| 유닛 행동 상태(대기·이동·공격·다운·정지·대기석) | `enum ActionStateType` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs` | 한 유닛이 가질 수 있는 행동 상태의 도메인 어휘를 정의 |
| 유닛 행동 상태 머신(유닛 부착 컴포넌트) | `ActionStateController` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateController.cs` | 상태 머신을 유닛에 부착하고, 매 프레임 전환·페이즈 통지를 상태 변경으로 연결 |
| 상태 전환 규칙(체력·타겟·이동가능·페이즈) | `ActionStateTransitionService` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateTransitionService.cs` | 전환 판정 규칙을 MonoBehaviour 밖 순수 C# 로 보유(테스트 가능) |
| 전투 종료 후 정지 | `ActionStateType.Freeze` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs` | 결과 표시 중 다운 외형/HP바를 유지하는 정지 상태 |
| 정비 중 무적 대기 | `ActionStateType.Waiting` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs` | Maintenance 페이즈 동안 무적·회복하며 다음 전투를 준비하는 상태 |

## 3. 구조 맵 (위치 중심)

- **진입점**: `ActionStateController.OnEnable` — `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateController.cs` (초기 상태로 머신 기동, `RoundManager` 페이즈 리스너 등록)
- **상태 어휘**: `ActionStateType` — `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs`
- **자동 전환 판정**: `ActionStateController.CheckStateTransition` → `ActionStateTransitionService.CheckTransition` — 체력·타겟 보유·이동가능을 입력받아 다음 상태 산출
- **페이즈 연동 전환**: `ActionStateController` 가 구현한 `IStateListener<PhaseType>.OnStateEnter` → `ActionStateTransitionService.ResolvePhaseTransition` — 라운드 페이즈를 행동 상태로 매핑

### 3.1 제공 헬퍼 (다른 코드가 찾아오는 곳)

| 헬퍼 | 위치(파일 경로) | 이럴 때 쓴다 |
|------|----------------|-------------|
| `ActionStateTransitionService.CheckTransition` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateTransitionService.cs` | 체력/타겟/이동가능 입력으로 다음 행동 상태를 MonoBehaviour 없이 판정·테스트할 때 |
| `ActionStateTransitionService.ResolvePhaseTransition` | `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateTransitionService.cs` | 라운드 페이즈에 대응하는 행동 상태를 얻을 때(전환 불필요 시 null) |

> 상태 변화에 반응하는 동작(HP 회복·외형 변경 등)은 이 모듈이 제공하지 않는다 — 외부가 `IStateListener<ActionStateType>` 를 `ActionStateController` 에 구독해 처리한다.

### 3.2 의존 관계

- **밖으로 의존**:
  - `Common.Scripts.StateBase` (`StateBaseController<T>`, `IStateListener<T>`) — 상태 머신 인프라. 라이프사이클(Enter→Run→Exit)·리스너 알림·전환 루프를 제공받는다.
  - `Scenes.Battle.Feature.Rounds` (`RoundManager`, `PhaseType`) — 페이즈 변경 통지를 받기 위한 리스너 등록 대상.
  - `Scenes.Battle.Feature.Units` (`Unit`, `Attacker`) — 전환 판정 입력(체력·공격 대상) 출처.
- **밖에서 의존**: 유닛 프리팹에 부착되어 동작하며, `IStateListener<ActionStateType>` 구독자(외형/HP/연출 처리 컴포넌트)가 상태 통지를 소비한다.

## 4. +α — 코드만 봐선 모르는 것

- **분할 의도**: 전환 규칙을 `ActionStateTransitionService` 순수 C# 로 떼어낸 것은 MonoBehaviour 의존 없이 전환 판정을 유닛 테스트하기 위함이다(파일 헤더 주석이 그 의도를 명시).
- **도메인적 의도**: `Freeze`/`Waiting` 은 전투 자체보다 라운드 페이즈 전환(결과 표시·정비)에 의해 들어가는 상태로, 행동 상태가 전투 루프뿐 아니라 라운드 페이즈 흐름의 종속 변수임을 반영한다. 전환 판정에서 체력 0 → `Downed` 가 다른 전환보다 우선하는 것은 사망 처리를 다른 행동보다 앞세우려는 도메인 우선순위다.
- **비자명한 제약/관례**: 동작 반응은 이 모듈에 작성하지 않고 외부 리스너로 분리한다는 관례가 있다(`ActionStateController` 헤더 주석 참조).
