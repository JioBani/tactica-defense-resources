# 게임 시스템 — 사용자 관점 계층

> 기획서를 바탕으로 게임 시스템을 **사용자가 인식하는 기능 단위**로 계층화한 트리다.
> AI 에이전트가 "이 게임에 어떤 기능이 있는가"를 레이어로 파악하고, 각 기능의 도메인 정의(기획서)와 시스템 구현(system 문서)으로 찾아가는 진입점이다.
>
> - **이 문서가 다루는 것**: 사용자 관점 기능의 계층(부모-자식) 구조, 각 노드의 한 줄 도메인 설명, 기획서·시스템 문서로의 라우팅.
> - **이 문서가 다루지 않는 것**: 도메인 규칙·수치·상세(→ 기획서), 코드·시스템 구조(→ 시스템 문서).

> 라우팅 표기: `[전체기획]` = `00 기획서.md`, `[시스템기획]` = `07 게임 시스템 기획서.md`. 잎(leaf) 기능 하나가 하나 이상의 시스템 관점 문서에 대응한다. `system: 미구현` 은 코드가 아직 없는(개발 예정) 기능이다.

## 계층 트리

- **전장 (인게임)** — 한 판(전장)을 플레이하는 핵심 게임플레이 루프. 정비/전투 페이즈가 교차하는 라운드들로 구성. · [전체기획](../../../../tactica-defense-resources/docs/planning/00%20기획서.md) §2.1.4
  - **라운드 & 페이즈** — 정비 페이즈(준비) ↔ 전투 페이즈(웨이브)의 교차, 정규/엑스트라 라운드, 전장 승리·패배 조건. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §4 · system: [라운드와페이즈](../system/라운드와페이즈.md), [생명크리스탈](../system/생명크리스탈.md)
  - **소환 터미널** — 마나로 소환수를 소환/판매(환원)하고 목록을 스캔(리롤)·잠금하며 배치 상한을 늘리는 인터페이스. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §3 · system: [소환터미널](../system/소환터미널.md)
    - 소환 / 환원 — 소환수 소환(성급별 마나) 및 판매. · §3.2, §3.5 · system: ↑ 소환터미널 문서에 포함
    - 스캔 / 스캔 잠금 — 등장 목록 재구성(리롤)과 라운드 시작 재스캔 잠금. · §3.3, §3.4 · system: ↑ 소환터미널 문서에 포함
    - 배치 상한 / 마나 — 배치 가능 수·상위 성급 확률 해금, 마나 충전·이자. · §3.6, §3.7 · system: ↑ 소환터미널 문서에 포함
  - **소환수** — 소환술사가 소환하는 전투 유닛(고양이). 풀 구성·합성·배치의 대상. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §2 · system: [소환수](../system/소환수.md)
    - 소환수 풀 & 역할군 분포 — 술사당 6체/전장 24체, 4역할군(탱커·평타·스킬·서포터) 분포 유형. · §2.0, §1.4 · system: [소환수](../system/소환수.md), [유닛공통](../system/유닛공통.md)
    - 소환수 동작 (전투) — 공격 규칙·타겟팅(최근접)·사거리(원형 거리)·스킬(쿨타임)·행동 상태. · §1.5~§1.8 · system: [공격과타겟팅](../system/공격과타겟팅.md), [스킬](../system/스킬.md), [행동상태](../system/행동상태.md)
    - 합성 (승급) — 동일 소환수 3개 → 상위 성급, 3성 이후 강화, 자동·연쇄 합성. · §2.2 · system: [합성](../system/합성.md)
    - 배치 & 라이프사이클 — 그리드 고정 배치·이동/스왑·벤치, 라운드 간 회복·쿨다운 초기화. · §2.1, §2.3 · system: [대기석](../system/대기석.md), [배치드롭존](../system/배치드롭존.md)
  - **유닛 스탯 & 데미지** — 모든 유닛(소환수·소환술사·침략자)의 능력치·스탯 합성·데미지 계산. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §1 · system: [유닛공통](../system/유닛공통.md)
    - 능력치 & 스탯 구조 — 능력치 목록, 기본/추가/최종 스탯, 수정자(고정값/비율, 출처), 증감 분리 곱연산. · §1.1, §1.2 · system: [유닛공통](../system/유닛공통.md)
    - 데미지 계산 — 치명타·방어력·피해 증가/감소 순서의 단일 공격력 기반 공식. · §1.3 · system: [데미지계산](../system/데미지계산.md)
    - 상태이상 — 유닛에 부여되는 상태 효과·우선순위. *(도메인 정의 추후 — 콘텐츠 단계)* · §1.9, §1.10 · system: 인프라 [StatusEffect](../system/infra/StatusEffect.md)
  - **시너지** — 팀의 *방향*을 결정하는 2차원 강화 구조(심상 효과 + 소환수 특성). · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §5 · system: [시너지](../system/시너지.md)
    - 심상 효과 (소환술사 귀속) — 출전 술사가 가진 고유 시너지, 메인 효과 + 글로벌 버프 이중 컴포넌트. · §5.2 · system: ↑ 시너지 문서에 포함
    - 소환수 특성 (전장 랜덤) — 전장마다 5개 효과가 소환수 종류에 랜덤 배정. · §5.3 · system: [소환수특성](../system/소환수특성.md)
    - 시너지 카운트 & UI — 종류 단위 카운트·임계치, 현재 전장 시너지 표시 UI. · §5.1, §5.4 · system: [시너지](../system/시너지.md), [전투UI](../system/전투UI.md)
  - **소환술사 (전장 내)** — 각 레인 끝에 1:1 배치되어 침략자와 교전하는 플레이어 유닛. 다운 시 전투 패배의 주체. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §1.1.1 · system: [소환술사](../system/소환술사.md)
  - **전술 지원 (인게임)** — 일정 라운드마다 등장하는 3택 1 강화. 팀 *출력*을 결정하는 핵심 축. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §6 · system: 미구현 (코드 없음 — 개발 예정)
  - **출격 모듈 (전장 적용)** — 보유한 모든 모듈이 전장 시작 시 자동 적용되는 영구 강화(인게임 측면). · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §7 · system: 미구현 (코드 없음 — 개발 예정)
  - **침략자 & 웨이브** — 레인을 따라 진군하는 적의 등장·웨이브 규칙·웨이브 프리뷰. · [시스템기획](../../../../tactica-defense-resources/docs/planning/07%20게임%20시스템%20기획서.md) §4.5 · system: [침략자](../system/침략자.md)
  - **전투 UI (횡단)** — 전투 화면의 시너지 정보·스탯 패널·라운드 정보 등 표시 레이어. · system: [전투UI](../system/전투UI.md)
- **출전 편성 (덱 빌딩)** — 전장 입장 전 소환술사 4명을 선택해 등장 소환수 풀과 역할군 분포를 설계하는 단계. · [전체기획](../../../../tactica-defense-resources/docs/planning/00%20기획서.md) §2.1.3 · system: 미구현 (편성 데이터만 존재: `Assets/Common/Data/Summoners/SummonerFormations`)
- **메타 (수집 · 육성)** — 전장 밖에서 전력을 영구히 키우는 4개 강화 축. · [전체기획](../../../../tactica-defense-resources/docs/planning/00%20기획서.md) §2.1.1, §3.2 · system: 미구현 (코드 없음 — 개발 예정)
  - 가챠 / 모집 — 단일 가챠로 소환술사·전술 지원·재화를 획득, 중복 시 심상조각. · §3.2.1 · system: 미구현
  - 소환술사 육성 — 레벨업(수치)·심상 각성(추가 효과), 최대 60레벨. · §3.2.2 · system: 미구현
  - 전술 지원 연구 · 강화 — 연구소에서 해금·강화, 인게임 등장 강도 상승. · §3.2.3 · system: 미구현
  - 출격 모듈 강화 (메타) — 강화 재화로 모듈 강화. · §3.2.4 · system: 미구현
- **캠페인 (컨텐츠)** — 전역 → 전장(일반/중간보스/보스)으로 구성된 스토리 모드, 보상 스케일링. · [전체기획](../../../../tactica-defense-resources/docs/planning/00%20기획서.md) §6 · system: 미구현 (코드 없음 — 개발 예정)

## 시스템 인프라 (UR 미매핑)

> 특정 사용자 기능에 매핑되지 않지만 시스템 전반을 떠받치는 인프라는 위 트리에 넣지 않는다. `system/infra/` 에 별도 문서화되며 [index.md](../index.md) 에서도 관리한다.

- **상태이상 프레임워크** (`StatusEffect`) — 유닛 상태이상/효과 관리 + 게임 이벤트 전달 플러그인 기반. · [infra](../system/infra/StatusEffect.md)
- **투사체** (`Projectiles`) — 원거리 공격·스킬의 발사체 풀링·발사·명중. · [infra](../system/infra/투사체.md)
- **전역 이벤트 버스** (`GlobalEventBus`) — 컴포넌트 간 느슨한 결합 이벤트 통지. · [infra](../system/infra/GlobalEventBus.md)
- **상태 머신 베이스** (`StateBase`) — 열거형 기반 범용 상태 머신(행동 상태·페이즈 등). · [infra](../system/infra/StateBase.md)
- **오브젝트 풀** (`ObjectPool`) — 투사체·침략자·UI 등 재사용 풀링. · [infra](../system/infra/ObjectPool.md)
- **관찰 가능 값** (`Rx`/`RxValue`) — 마나·티어·체력 등 변경 통지. · [infra](../system/infra/Rx.md)
- **씬 수명과 접근** (`SceneSingleton`·`SceneData`) — 매니저 싱글톤·씬 간 데이터 전달. · [infra](../system/infra/씬수명과접근.md)
- **드래그 앤 드롭** (`Draggable`) — 배치·대기석·드롭존 상호작용. · [infra](../system/infra/Draggable.md)
- **비동기와 타이밍** (Timer·TaskQueue·DynamicRepeater 등) — 타이머·순차 비동기·주기 반복. · [infra](../system/infra/비동기와타이밍.md)
- **공용 유틸리티** (직렬화·인스펙터·확장 등 묶음) — 소규모 범용 헬퍼 색인. · [infra](../system/infra/공용유틸리티.md)
