# TACD-305 구현 결과 — 고해상도 대응 (레터박스 고정)

> 작성: `impl` 멤버 · 2026-06 · 근거: [작업정의.md](작업정의.md), [해상도-조사.md](해상도-조사.md)
> 방향: 기준 종횡비 **20:9 고정 + 레터박스**, 기준 도면값 **2400×1080**
> 상태: 코드/씬 구현 완료. Play mode 통합 시각검증은 후속(§6).

---

## 1. 개요

- 카메라 렌더 해상도 변경이 아니라(이미 기기 물리 해상도로 렌더됨 — 조사 경우 A), **반응형 UI + 레터박스 종횡비 고정**이 실제 작업.
- 종횡비 계산을 **순수 C#(테스트 가능)** 과 **얇은 MonoBehaviour(Unity 연동)** 로 분리해 클린 아키텍처 준수.
- 레터박스 띠는 URP 단일 카메라 rect 축소만으로는 검정 보장이 안 되어 **검정 배경 카메라**를 추가.

---

## 2. SR-1 — 레터박스 카메라 (완료)

신규 폴더: `Assets/Scenes/Battle/Feature/CameraControl/Letterbox/`

| 파일 | 책임 | 비고 |
|---|---|---|
| `AspectRatioPolicy.cs` | 기준 종횡비 상수 한 곳 관리 | `ReferenceWidth=2400`, `ReferenceHeight=1080`, `TargetAspect`(20:9). const(데이터 콘텐츠 아닌 단일 표시 정책). |
| `LetterboxViewport.cs` | (targetAspect, w, h) → 카메라 뷰포트 `Rect` 계산 | **순수 함수**, MonoBehaviour/Time 비의존 → EditMode 단위테스트 가능. 비정상 입력 `ArgumentException`(조용한 실패 금지). |
| `LetterboxCameraController.cs` | 화면 변동 감지 → `Camera.rect` 적용 | 얇은 어댑터. 계산은 Viewport에 위임. `[RequireComponent(typeof(Camera))]`. Main Camera 부착. |

### 계산 로직
- `screenAspect < targetAspect`(덜 와이드, 예 16:9) → 상하 띠(letterbox): `Rect(0, (1-s)/2, 1, s)`
- `screenAspect > targetAspect`(더 와이드, 예 21:9) → 좌우 띠(pillarbox): `Rect((1-s)/2, 0, s, 1)`
- 정확히 일치하면 전체 화면(띠 없음).

### 검정 배경 카메라 (보완)
- **발견**: URP에서 단일 카메라 `rect` 축소 시 띠 영역이 검정 클리어되지 않음(Main Camera ClearFlags=Skybox).
- **대응**: 씬에 `LetterboxBackgroundCamera` 추가 — depth `-2`, ClearFlags `SolidColor=black`, cullingMask `Nothing`, rect 전체, URP `Base`. 매 프레임 전체를 검정 클리어 → 그 위에 Main Camera(depth -1)가 20:9 영역만 렌더.
- `CameraControlManager`는 `Camera.main`(MainCamera 태그)만 제어하므로 배경 카메라와 책임/충돌 없음. 레터박스(`rect`)와 페이즈 연출(`orthoSize`/`position`)도 독립.

## 3. SR-2 — UI 반응형 + 레터박스 정합 (완료)

대상은 Screen Space 캔버스 1개(`Canvas`). World Space 캔버스(`WorldSpaceCanvas`)는 미변경(작업정의 Q4 확정).

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| `Canvas.renderMode` | Screen Space-Overlay | **Screen Space-Camera** (worldCamera=Main Camera) |
| `Canvas.planeDistance` / `sortingOrder` | – / 0 | 100 / 100 |
| `CanvasScaler.uiScaleMode` | ConstantPixelSize | **ScaleWithScreenSize** |
| `CanvasScaler.referenceResolution` | 800×600(무시되던 값) | **2400×1080** |
| `CanvasScaler.screenMatchMode` / `match` | MatchWidthOrHeight / 0 | MatchWidthOrHeight / **1 (Height)** |

- Screen Space-Camera 전환으로 캔버스가 Main Camera의 뷰포트(레터박스 `rect`)에 정합 → UI가 검은 띠 위로 새지 않음.
- EditMode 20:9 캡처: 상단 UI 바·버튼 정상 표시, 깨짐 없음 확인.

## 4. SR-3 — 월드 오브젝트 배치 점검 (완료, 변경 불필요)

- Main Camera: pos `[0, 1.24, -10]`, **orthographic size 3.3**(가시 높이 6.6유닛), 20:9 시 가시 폭 ≈ 14.67유닛.
- 종횡비가 20:9로 **불변**이고 레터박스로 가시 영역이 항상 20:9로 고정되므로, 기존 800×360 프리뷰와 콘텐츠 배치가 그대로 일치. **재배치 불필요.**
- `CameraControlManager`의 페이즈별 orthoSize(3.3↔4.0)·position.x(0↔1.5↔3) 연출도 비율 영향 없이 유지.

## 6. 후속 / 미검증 항목

- **Play mode 통합 시각검증** (test 멤버 권장):
  - 비표준 비율(16:9 등) GameView에서 검은 띠 생성 + 게임 내용 안 잘림.
  - 띠가 실제 검정으로 렌더(배경 카메라 합성) — EditMode game_view 캡처는 Main Camera 직접 렌더라 합성 미반영, Play mode에서만 확인 가능.
  - 다양한 비율에서 UI가 20:9 영역에 정합.
  - `LetterboxViewport.Calculate` EditMode 단위테스트(16:9/20:9/21:9 → 기대 Rect).
- **아트/기획 확정**: Terrain 배경·Illustration·Summoner 일러스트 흐릿 후보 재제작 여부.
