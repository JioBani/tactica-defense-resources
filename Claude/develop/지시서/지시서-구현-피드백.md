# 지시서-구현-피드백

> 이 문서는 구현 에이전트가 테스트 실패 후 피드백 루프에서 따라야 할 절차다.

---

## 입력

- 테스트 결과 보고서 경로

---

## 절차

### 1. 보고서 분석

1. 테스트 결과 보고서를 읽는다
2. 실패 원인을 파악한다

### 2. 비즈니스 로직 검토

실패 내용을 바탕으로 원인을 분류한다.

- **테스트 코드 버그로 판단되는 경우**: 구현 코드를 수정하지 않고, "테스트 코드 버그"로 판단 사유와 함께 메인 에이전트에 반환한다. 메인이 테스트 에이전트에 수정을 지시한다.
- **요구사항에 대한 사용자 확인이 필요한 경우**: 검토 내용과 질문을 메인 에이전트에 반환한다. 사용자 응답이 SendMessage로 전달되면 반영한다.
- **요구사항 반영이 안 된 것이 확실한 경우**: 바로 수정한다.

### 3. 코드 수정

1. 실패 원인에 따라 코드를 수정한다
2. Unity 컴파일 확인 (`refresh_unity` + `read_console`)

### 4. 반환

메인 에이전트에 아래를 반환한다.

- 변경된 파일 목록
- 컴파일 결과
