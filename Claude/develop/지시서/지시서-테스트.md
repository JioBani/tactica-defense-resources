# 지시서-테스트

> 이 문서는 테스트 에이전트가 따라야 할 절차다.
> 구현된 코드에 대해 유닛 테스트를 작성·실행하고 결과 보고서를 산출한다.

---

## 입력

- 작업정의 경로: `Claude/develop/TACD-{번호}/작업정의.md`
- 대상 단위 번호: `{N}`
- 변경 파일: `{구현 에이전트가 반환한 파일 목록}`

## 산출물

- 테스트 코드 (`Assets/Tests/Editor/{씬명}/`에 작성. 예: `Assets/Tests/Editor/Battle/`)
- `Claude/develop/TACD-{번호}/테스트-보고서-{단위번호}.md`
- git commit

---

## 참조 문서

- `.claude/docs/develop.md` — 개발 지침
- `.claude/docs/code-convention.md` — 코드 컨벤션
- `.claude/docs/git.md` — Git 커밋 지침

---

## 절차

### 1. 테스트 케이스 분리

1. 작업정의.md에서 대상 단위의 완료 정의를 확인한다
2. 변경 파일 목록을 확인하고 구현 코드를 읽는다
3. 완료 정의를 바탕으로 테스트 케이스를 도출한다.
   1. 아래 방법론 중 해당되는 것만 적용하여 케이스를 나열한다:
      - **정상 경로**: 완료 정의가 기대하는 동작이 정상적으로 수행되는 케이스
      - **경계값**: 최솟값, 최댓값, 0, 빈 컬렉션 등 경계 조건
      - **예외/에러 경로**: 잘못된 입력, null, 불가능한 상태 등
      - **상태 전이**: 상태 변경 전후의 동작 차이
   2. 중복되는 케이스를 제거한다
   3. 비즈니스 관점에서 관련 케이스를 묶는다

### 2. 테스트 코드 작성

케이스별로 하나씩 테스트 코드를 작성한다.

- **테스트 가능한 케이스** (순수 C# 로직, 서비스 클래스, 계산, 판정 등): 테스트 코드를 구현한다
- **테스트 불가능한 케이스** (MonoBehaviour·씬 의존 로직, UI 구성, 프리팹 설정 등): `[Ignore("사유")]` 속성을 붙여 스킵 처리한다. 성공/실패/스킵이 구분되도록 한다.

#### 데이터 기반 테스트

같은 로직을 여러 데이터로 검증할 때는 `[TestCase]`/`[TestCaseSource]`를 적극 활용하여 중복 코드를 없애고 케이스를 가독성 좋게 그룹화한다.

#### 테스트 코드 주석 규칙

- **파일 최상단**: 1단계에서 분리한 테스트 케이스 목록을 주석으로 정리한다
- **각 테스트 함수**: 어떤 케이스인지, 무엇을 검증하는지 주석을 작성한다

#### 작성 후

Unity 컴파일 확인 (`refresh_unity` + `read_console`)

### 3. 테스트 실행 및 오류 수정

1. MCP `run_tests`로 테스트를 실행한다 (mode: EditMode, include_failed_tests: true)
2. `get_test_job`으로 결과를 폴링한다 (wait_timeout: 30 권장). 5회 폴링 후에도 완료되지 않으면 사유와 함께 메인 에이전트에 반환한다.
3. 실패한 테스트를 분류한다:
   - **테스트 코드 오류**: 테스트 코드 자체의 버그 (잘못된 setup, mock 누락, assertion 오류 등)
   - **요구사항 미충족**: 비즈니스 로직이 완료 정의를 만족하지 못하는 경우
4. **테스트 코드 오류는 직접 수정**하고 재실행한다. 모든 테스트 코드 오류가 해소될 때까지 반복한다.
5. **요구사항 미충족은 로직을 절대 수정하지 않는다.** 분석만 수행한다.

### 4. 보고서 작성

`Claude/develop/TACD-{번호}/테스트-보고서-{단위번호}.md`에 작성한다 (템플릿: `Claude/develop/템플릿/테스트-보고서-템플릿.md`)

요구사항 미충족이 있는 경우, 보고서에 **미충족 분석** 섹션을 추가한다:
- 어떤 완료 정의가 충족되지 않았는지
- 테스트가 기대한 동작과 실제 동작의 차이
- 이 분석은 테스트 에이전트의 판단이며, 구현 에이전트가 검증한다

### 5. 반환

메인 에이전트에 아래를 반환한다.

- 테스트 결과 보고서 경로
- 성공/실패/스킵 건수

### 5. 커밋

SendMessage로 커밋 지시가 오면, `.claude/docs/git.md` 형식에 따라 커밋한다.

- 커밋 해시를 메인 에이전트에 반환한다
