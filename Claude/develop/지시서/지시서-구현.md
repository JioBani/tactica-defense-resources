# 지시서-구현

> 이 문서는 구현 에이전트가 따라야 할 절차다.
> 코드베이스를 탐색하고 plan을 작성한 뒤, 승인된 plan에 따라 구현 및 커밋한다.

---

## 입력

- 작업정의 경로: `Claude/develop/TACD-{번호}/작업정의.md`
- 대상 단위 번호: `{N}`

## 산출물

- `Claude/develop/TACD-{번호}/plan-{단위번호}.md`
- 코드 변경 (스크립트 생성/수정)
- git commit

---

## 참조 문서

아래 문서를 읽고 숙지한다.

- `.claude/docs/develop.md` — 개발 지침
- `.claude/docs/code-convention.md` — 코드 컨벤션
- `.claude/docs/git.md` — Git 커밋 지침

---

## 절차

### 1. 작업 단위 파악

1. 작업정의.md를 읽는다
2. 대상 단위 번호에 해당하는 작업 단위와 완료 정의를 확인한다

**주의: 자신이 담당하는 단위의 완료 정의만 대상으로 한다. 다른 단위의 완료 정의는 절대 plan이나 구현에 포함하지 않는다.**

### 2. Plan 작성

**plan 작성이 완료되어 반환하기 전까지 코드 수정을 절대 하지 않는다.**

1. 코드베이스 탐색
   - 커밋 히스토리 우선: `git log --all --grep="TACD-{번호}"`로 해당 일감의 기존 커밋 확인
   - 커밋으로 부족하면 코드베이스 탐색 (`CLAUDE.md` 참고)
   - **충분히 탐색한 뒤 계획을 작성한다.** 관련 코드를 얕게 훑고 바로 계획을 세우지 않는다. 변경 대상 코드, 호출자, 의존 관계를 파악한 뒤에 계획을 작성한다.
2. **대상 단위의 완료 정의만** 만족시키는 구현 계획을 작성한다. 다른 단위의 내용을 선행 구현하거나 함께 다루지 않는다.
3. `Claude/develop/TACD-{번호}/plan-{단위번호}.md`에 저장한다

### 3. 반환 (plan)

메인 에이전트에 plan 파일 경로를 반환한다.
사용자 검토 후 SendMessage로 피드백이 올 수 있다. 피드백에 따라 plan을 수정하고 다시 반환한다.
사용자 승인 시 SendMessage로 구현 지시가 온다.

### 4. 코드 구현

1. 승인된 plan에 따라 코드를 구현한다
2. Unity 컴파일 확인 (`refresh_unity` + `read_console`)
3. 컴파일 오류가 있으면 수정한다

### 5. 반환 (구현)

메인 에이전트에 아래를 반환한다.

- 변경된 파일 목록
- 컴파일 결과

사용자 피드백이 올 수 있다. 피드백에 따라 구현을 수정하고 다시 반환한다.

### 6. 커밋

SendMessage로 커밋 지시가 오면, `.claude/docs/git.md` 형식에 따라 커밋한다.

- 커밋 해시를 메인 에이전트에 반환한다
