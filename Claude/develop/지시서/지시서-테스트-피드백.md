# 지시서-테스트-피드백

> 이 문서는 테스트 에이전트가 피드백 루프에서 따라야 할 절차다.
> 구현 수정 후 테스트를 재실행하고 보고서를 갱신한다.

---

## 입력

- 변경 파일: `{구현 에이전트가 반환한 변경 파일}`

---

## 절차

### 1. 테스트 코드 확인

변경된 파일을 확인하고, 테스트 코드 수정이 필요한지 판단한다.

- **구현 코드의 시그니처 변경으로 인한 컴파일 에러**: 테스트 코드를 수정한다
- **테스트 로직 자체의 버그** (구현-피드백에서 지적받은 경우): 테스트 코드를 수정한다
- **요구사항 해석 변경**: 메인 에이전트에 확인을 요청한다

### 2. 테스트 재실행

초기 절차와 동일한 방법으로 실행한다 (MCP `run_tests` + `get_test_job` 폴링).

### 3. 보고서 갱신

기존 테스트 결과 보고서를 **덮어쓰기**한다. 이전 결과는 남기지 않는다.

지시서-테스트의 보고서 작성 절차(테스트 코드 오류 수정, 요구사항 미충족 분석)와 동일하게 수행한다.

### 4. 반환

메인 에이전트에 아래를 반환한다.

- 테스트 결과 보고서 경로
- 성공/실패/스킵 건수
