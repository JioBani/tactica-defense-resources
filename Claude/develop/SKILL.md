# develop 스킬

Jira 일감 기반 오케스트레이터 개발 워크플로우.

## 사용법

```
/develop TACD-{번호}
```

## 실행

1. `지시서/지시서-메인.md`를 읽고 워크플로우를 수행한다.
2. 서브에이전트에 지시서를 전달할 때는 이 스킬의 디렉토리 기준 경로를 사용한다.
3. 작업 산출물(작업정의, plan, 보고서 등)은 프로젝트 디렉토리 `Claude/develop/TACD-{번호}/`에 저장한다.

## 지시서 구조

| 파일 | 역할 |
|------|------|
| `지시서/지시서-메인.md` | 메인 에이전트 (오케스트레이터) |
| `지시서/지시서-작업정의.md` | 작업정의 에이전트 |
| `지시서/지시서-구현.md` | 구현 에이전트 (plan + 구현) |
| `지시서/지시서-테스트.md` | 테스트 에이전트 |
| `지시서/지시서-구현-피드백.md` | 구현 피드백 루프 |
| `지시서/지시서-테스트-피드백.md` | 테스트 피드백 루프 |
| `지시서/지시서-작업정의-복구.md` | 작업정의 에이전트 복구 |

## 템플릿

| 파일 | 용도 |
|------|------|
| `템플릿/작업정의-템플릿.md` | 작업정의 작성 시 사용 |
| `템플릿/테스트-보고서-템플릿.md` | 테스트 결과 보고서 작성 시 사용 |
