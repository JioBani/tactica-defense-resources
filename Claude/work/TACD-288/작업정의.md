# TACD-288: Base AnimatorController 및 placeholder 클립 에셋 제작

> 상위 이슈: TACD-287 [전장 - 유닛] 애니메이션 프레임워크 제작

## 요구사항

OverrideController의 원본 템플릿이 되는 Base AnimatorController를 제작한다.
ActionStateType enum의 각 상태에 대응하는 State와 placeholder 클립을 포함한다.

## 배경지식

- 설계 문서: `Claude/work/TACD-287/애니메이션 프레임워크 설계.md`
- Base AnimatorController는 프리팹에 직접 붙지 않고, OverrideController의 기반 템플릿으로만 사용된다
- 각 State의 placeholder 클립은 OverrideController Inspector에서 교체 목록을 표시하기 위해 필요하다
- Transition은 없음 — 코드에서 `animator.Play()`로 직접 전환

## 완료 정의

- Base AnimatorController에 ActionStateType의 모든 상태(Idle, Move, Attack, Downed, Freeze, Waiting)가 State로 존재한다
- 각 State에 placeholder 클립이 할당되어 있다
- Idle이 기본 상태(Default State)로 설정되어 있다

## 기존 코드 참조

- `Assets/Scenes/Battle/Feature/Unit/ActionState/ActionStateType.cs` — 상태 enum 정의 (Idle, Move, Attack, Downed, Freeze, Waiting)
