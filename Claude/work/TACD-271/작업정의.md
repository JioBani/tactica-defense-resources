# TACD-271: [시너지 > 효과] 유닛 성급 변화시 효과 갱신

> 상위 이슈: TACD-251 [전장 - 시너지] - 시너지 계산 및 효과 적용

## 요구사항

유닛의 성급(승급·강화)이 변경되면, 해당 유닛에 적용된 시너지 효과가 새로운 기본 스탯을 기준으로 갱신되어야 한다.

현재는 `UnitStatSheet.UpgradeStar()` / `Reinforce()` 호출 시 `Init()` → `ClearModifiers()`가 실행되어 시너지 SSE의 스탯 수정자가 소실되지만, 시너지 시스템이 이를 감지하지 못해 효과가 재적용되지 않는다.

## 배경지식

- **성급 변화 시점**: 동일 유닛 3개 합성(승급) 또는 드래그 앤 드롭 강화 합성 시 `DefenderFusionManager`가 `OnDefenderFusedEventDto`를 발행한다.
- **ClearModifiers 문제**: `UnitStatSheet.Init()`은 모든 스탯의 수정자를 초기화(`ClearModifiers`)한 뒤 새 기본값을 설정한다. 이로 인해 시너지 SSE가 추가한 수정자가 사라진다.
- **기존 이벤트**: `UnitStatSheet.OnGradeChanged`가 성급/강화 변경 시 발생하고, `OnDefenderFusedEventDto`가 합성 완료 시 GlobalEventBus로 발행된다.

## 완료 정의

- 전장에 배치된 유닛이 승급(기본 합성)되면, 활성 시너지 효과가 새 기본 스탯 기준으로 정상 적용된다.
- 전장에 배치된 유닛이 강화 합성되면, 활성 시너지 효과가 새 기본 스탯 기준으로 정상 적용된다.
- 대기실에서 승급/강화된 유닛이 이후 전장에 배치되면 시너지 효과가 정상 적용된다.
- 기존 시너지 동작(배치/제거/티어 변경)에 영향 없다.

## 기존 코드 참조

- `UnitStatSheet.UpgradeStar()` / `Reinforce()`: `Init()` 호출 → `ClearModifiers()` → `SetBaseValue()` → `OnGradeChanged` 발행
- `SynergyController`: GlobalEventBus로 배치 변경/판매 이벤트를 구독하지만, 합성/성급 변경 이벤트는 구독하지 않음
- `OnDefenderFusedEventDto`: 합성 완료 시 발행. `Survivor`(승급 대상)와 `NewStar` 포함
- `UnitStatSheet.OnGradeChanged`: 성급/강화 변경 시 발생. `GradeChangedInfo(Star, Reinforcement)` 전달
