# TACD-261: StatusEffect 프레임워크 제작

> 상위 이슈: TACD-123 [전장 - 유닛] 상태이상

## 배경

유닛에 영향을 주는 모든 효과(시너지, CC, 버프/디버프, 스킬, 아이템 등)를 통합 관리하는 프레임워크가 없다. 현재 시너지 프레임워크(SynergyController/SynergyEffect)가 효과 생명주기를 독자적으로 관리하고 있으나, 이는 범용 상태효과 프레임워크가 담당해야 할 역할이다.

StatusEffect(상태효과) 프레임워크를 구축하여, 유닛에 영향을 주려면 반드시 SE를 부여하는 **단일 통로**를 거치게 한다.

### 용어

| 약어 | 정식 명칭 | 설명 |
|------|-----------|------|
| SE | StatusEffect | 유닛에 부여되는 효과 인스턴스 |
| SED | StatusEffect Definition | SE의 정의 (ScriptableObject) |
| SEController | StatusEffect Controller | 유닛에 부착, SE 컬렉션 관리 및 생명주기 실행 |
| HookProvider | IStatusEffectHookProvider | SEController에 붙는 플러그인. SE에 새로운 생명주기 훅을 제공 |

## 작업 내용

### SE 프레임워크 코어

- 유닛에 SE를 부여하면 Apply → Update(매 프레임) → Remove 생명주기가 실행된다
- 유닛당 여러 SE를 동시에 보유하고 관리할 수 있다
- SE의 정적 속성은 SED(ScriptableObject)로 정의하고, 동적 데이터는 부여 시 Context로 전달한다
- SEController는 범용 컨테이너로, 게임 특화 개념(시너지, 스킬 등)을 모른다

### HookProvider 시스템

- SEController에 HookProvider(순수 C# 플러그인)를 붙여 게임 이벤트를 SE에 전달한다
- SE는 반응할 트리거에 해당하는 인터페이스를 구현하여 특정 이벤트에 반응한다
- HookProvider는 관심 인터페이스를 캐싱하여 매 트리거마다 타입 검사를 하지 않는다

## 완료 정의

- 유닛에 SE를 부여하면 생명주기(Apply → Update → Remove)가 정상 실행된다
- SED + Context를 통해 SE를 생성·부여할 수 있다
- HookProvider를 등록하면 SE 추가/제거 알림과 인터페이스 캐싱이 동작한다
