# TACD-251: [전장 - 시너지] - 시너지 계산 및 효과 적용

## 요구사항

배치된 소환수 기반 시너지 카운트, 임계치 판정, 효과 적용/해제, 배치·제거 시 갱신 트리거를 구현한다.

- 전장(BattleArea)에 배치된 소환수에서 시너지별 카운트를 집계한다
- 동일 종류의 소환수는 여러 체 배치되어도 1회만 카운트한다
- 카운트를 기반으로 임계치를 판정하여 활성 티어를 결정한다
- 임계치 충족 시 해당 시너지를 보유한 소환수에게 효과를 적용한다
- 임계치 미달 시 효과를 해제한다. 티어 변경 시 효과를 교체한다
- 소환수 배치/제거 시 시너지를 재계산한다. 전투 중 다운 시에는 유지한다

## 배경지식

- **시너지**: 소환술사 효과와 소환수 특성의 총칭
- **소환술사 효과**: 소속 소환술사에 의해 고정. UnitDefinitionData.summonerEffect로 참조
- **소환수 특성**: 전장마다 랜덤 부여 (TACD-250 소관, 미구현). 카운트 구조는 지원하되 실제 값은 아직 없음
- **카운트 기준**: "종류"는 UnitDefinitionData 기준. 같은 소환수를 2체 배치해도 카운트는 1
- **임계치**: 소환술사 효과 2/4/6/8 (4단계), 소환수 특성 2/3/4 (3단계)
- **효과 적용 대상**: 해당 시너지를 가진 소환수에게만 적용
- **SynergyState**: 시너지 1개당 인스턴스 1개. 자신의 카운트, 티어, 효과를 관리하는 순수 C# 클래스
- **DTO 패턴**: SynergyState는 Defender에 직접 의존하지 않고 DTO를 받아 처리한다

## 완료 정의

- 시너지별 카운트가 정확히 산출된다 (종류 기준 중복 제거)
- 카운트 기반으로 올바른 활성 티어가 결정된다
- 임계치 충족 시 시너지 보유 소환수의 스탯이 효과에 따라 변경된다
- 임계치 미달 시 적용된 효과가 정확히 제거된다
- 티어 변경 시 효과가 올바르게 교체된다
- 소환수 배치/제거 시 시너지가 즉시 갱신된다
- 전투 중 소환수 다운 시 시너지 카운트가 변하지 않는다
- SynergyState는 순수 C#이며 new로 생성·테스트 가능하다

## 기존 코드 참조

- `SynergyDefinitionData`: 시너지 정의 (id, displayName, synergyType, tiers, GetActiveTier)
- `SynergyTier`: 임계치 + constants (SerializableDictionary<string, float>)
- `SynergyType`: SummonerEffect / SummonTrait
- `UnitDefinitionData.summonerEffect`: 소환술사 효과 참조 필드
- `UnitStat.AddModifier / RemoveModifiersBySource`: 수정자 추가/해제
- `StatModifier(Source, Type, Value)`: 수정자 (Source로 출처 추적)
- `DefenderManager.OnPlacementChange / OnDefenderChange`: 배치/생성·소멸 이벤트
- `FusionGroupDetector + FusionCandidate`: DTO 패턴 참고
