# TACD-253 — 전장 시작 시 소환수 풀에 역할군 보정된 특성이 랜덤 부여된다

> 이 문서는 사용자 요구사항을 시스템 컴포넌트 단위 책임으로 번역한 문서입니다.

## 개요

- Jira: TACD-253 (상위: TACD-250)
- 전장 시작 시 소환수 풀(32체) 전체에 SummonTrait 타입 특성을 균등 랜덤으로 1개씩 배정하고, 그 결과를 전장 중 런타임으로 보관하며 전장 종료 시 초기화한다.

| 컴포넌트 | 역할 |
|----------|------|
| 특성 풀 (SummonTrait 자산 집합) | 분배 대상 소환수 특성 목록 제공 |
| 특성 분배 서비스 | 전장 시작 시 32체에 균등 랜덤 배정 실행 |
| 소환수별 특성 저장소 | 배정 결과를 전장 중 런타임으로 보관, 전장 종료 시 초기화 |

## 현재 동작

- 전장 시작 시 소환수에게 특성이 배정되지 않는다.
- SummonTrait 타입의 SynergyDefinitionData 자산을 인식·로드하는 시스템 진입점이 없다.
- 소환수 정의 데이터(UnitLoadOutData)와 SummonTrait 자산을 런타임에 연결하는 구조가 없다.

## 시스템 요구사항

### 완료 정의 나열

- **DoD-S1.** SummonTrait 타입의 SynergyDefinitionData 자산 전체가 분배 대상 특성 풀로 시스템에 인식되어 접근 가능하다.
- **DoD-S2.** 전장 시작 시 소환술사 4명의 소환수 풀(각 8체, 총 32체) 각각에 특성 풀에서 1개의 특성이 균등한 확률로 무작위 배정된다. 동일 특성이 복수의 소환수에 배정될 수 있다.
- **DoD-S3.** 소환수별 배정된 특성 정보가 전장 중 런타임 데이터로 보관된다. 영속화(파일·DB 저장)되지 않는다.
- **DoD-S4.** 전장 종료 시 소환수별 특성 데이터가 초기화된다.

### 완료 정의를 기능별로 그룹화

**그룹 A. 전장 시작 특성 분배 실행**
- 포함 DoD: DoD-S1, DoD-S2
- 대응 사용자 요구사항 그룹: 그룹 A (DoD-U1, DoD-U3)
- 설명: 전장 시작 시 SummonTrait 자산을 인식하고 32체 전체에 균등 랜덤으로 배정하는 책임. 이 그룹이 충족되지 않으면 그룹 B 런타임 보관이 의미 없다.

**그룹 B. 분배 결과 런타임 보관 및 초기화**
- 포함 DoD: DoD-S3, DoD-S4
- 대응 사용자 요구사항 그룹: 그룹 A (DoD-U2)
- 설명: 배정 결과를 전장 중 접근 가능한 런타임 상태로 유지하고, 전장 종료 시 초기화하여 다음 전장에서 새 분배가 가능한 상태를 보장하는 책임.
