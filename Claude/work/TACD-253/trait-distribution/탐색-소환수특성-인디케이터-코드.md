# TACD-253 — 코드 탐색 결과 (코드 모드)

> 이 문서는 impl 멤버의 위임 요청(사용자 문의 대응)에 대한 코드 탐색 응답이며, 도메인 언어로 먼저 답한 뒤 필요한 경우 코드 단계까지 풀어낸 결과입니다.

## 요청 컨텍스트

- **요청자**: impl 멤버 (사용자 문의 대응 중)
- **요청 단계**: 사용자 문의 대응 (TACD-253 후속)
- **요청 내용**: "소환술사 효과 인디케이터는 뜨는데, 소환수 특성(SummonTrait) 인디케이터가 안 뜬다. 이유는?"
- **탐색 범위**:
  - `Assets/Scenes/Battle/Feature/Synergy/Scripts/` (SynergyManager, SynergyControllerFactory)
  - `Assets/Scenes/Battle/Feature/Synergy/UI/` 추정 (SynergyListPanel, SynergyIndicator, SynergyDetailPanel)
  - `Assets/Scenes/Battle/Feature/SummonTrait/` (SummonTraitStore, SummonTraitDistributor)
  - `Assets/Common/Data/Synergies/` (SynergyType, SynergyDefinitionData)
  - `Assets/Tests/Editor/` (참조 확인용)

## 응답

### 질문 1: SummonTrait 타입 SynergyDefinitionData를 인디케이터로 표시하는 통합 로직 존재 여부

**도메인 표현**: 통합 로직이 존재하지 않는다. `SynergyType.SummonTrait`는 enum에만 정의되어 있고, 이 값을 인디케이터 표시 흐름에 연결하는 코드는 코드베이스 어디에도 없다.

**코드 구조**:

- **정의 위치**: `Assets/Common/Data/Synergies/SynergyType.cs`
  ```csharp
  public enum SynergyType
  {
      SummonerEffect,
      SummonTrait,
  }
  ```

- **분기 처리 부재**: `SynergyType.SummonTrait` 또는 `SynergyType.SummonerEffect` 값을 비교·분기하는 코드가 Assets 하위 어디에도 존재하지 않음.

- **의도된 미구현 명시**: `SynergyManager.cs` (41~42줄 주석):
  ```
  /// 현재는 소환술사 효과 시너지에 한해 누적되며,
  /// 향후 소환수 특성 분배 도메인 로직이 추가되면 동일 인덱스에 자연 합산된다.
  ```

### 질문 2: SynergyManager.Start()가 SummonerEffect만 수집하는지

**도메인 표현**: 명확히 SummonerEffect만 수집한다. SummonTrait를 수집하거나 추적하는 코드는 SynergyManager에도, 다른 어떤 클래스에도 없다.

**코드 구조**: `Assets/Scenes/Battle/Feature/Synergy/Scripts/SynergyManager.cs`

- **BuildUnitSynergyMap()** (90~114줄):
  ```csharp
  foreach (SummonerLoadOutData summoner in summoners)
  {
      SynergyDefinitionData effect = summoner.Summoner.SummonerEffect;
      foreach (UnitLoadOutData unit in summoner.Summoner.SummonPool)
      {
          _unitSynergyMap[unit] = effect;
      }
  }
  ```
  → `SummonerEffect` 값만 수집. `SummonTrait` 처리 없음.

- **BuildSynergyMembersMap()** (123~156줄):
  ```csharp
  foreach (SummonerLoadOutData summoner in summoners)
  {
      SynergyDefinitionData effect = summoner.Summoner.SummonerEffect;
      // ... effect 키로 _synergyMembers 누적
  }
  ```

- **InitializeSynergyActivations()** (159~169줄):
  ```csharp
  var uniqueEffects = new HashSet<SynergyDefinitionData>();
  foreach (SummonerLoadOutData summoner in summoners)
  {
      uniqueEffects.Add(summoner.Summoner.SummonerEffect);
  }
  InitializeSynergyActivations(uniqueEffects);
  ```

**핵심 식별자**:
- 필드: `_synergyMembers : Dictionary<SynergyDefinitionData, List<UnitLoadOutData>>` — SummonerEffect만 들어 있음
- 필드: `_synergyActivations : Dictionary<SynergyDefinitionData, SynergyActivation>` — SummonerEffect만 들어 있음
- 속성: `SynergyActivations : IReadOnlyDictionary<...>` — UI 진입점

### 질문 3: 인디케이터 UI 코드의 SummonTrait 분기 여부

**도메인 표현**: UI 계층은 `SynergyManager.SynergyActivations`에서만 데이터를 가져오며, `SynergyType`을 검사하거나 분기하는 코드는 없다. 인디케이터·디테일 패널·컨트롤러 모두 시너지 종류를 구분하지 않는다.

**코드 구조**:

- **SynergyListPanel.cs** (23~51줄):
  ```csharp
  private void Start()
  {
      IReadOnlyDictionary<SynergyDefinitionData, SynergyActivation> activations =
          SynergyManager.Instance.SynergyActivations;
      
      int index = 0;
      foreach (SynergyActivation activation in activations.Values)
      {
          if (index < indicators.Length)
          {
              indicators[index].Bind(activation);
              indicators[index].gameObject.SetActive(activation.Count > 0);
          }
      }
  }
  ```
  → 데이터 소스가 `SynergyManager.SynergyActivations` 단일. SummonTrait 분기 없음.

- **SynergyIndicator.cs**: `SynergyType` 참조 없음. 바인딩된 `SynergyActivation`의 이름·아이콘·티어만 표시.

- **SynergyDetailPanel.cs**: `SynergyType` 참조 없음. 모든 시너지를 동일하게 처리.

- **SynergyControllerFactory.cs**: `SynergyId` enum 기반 switch 분기로 컨트롤러 생성. `SynergyType` 분기 없음.

### 질문 4: TACD-253 작업물(SummonTraitStore / SummonTraitDistributor)의 연결성

**도메인 표현**: 두 클래스는 구현되어 있으나 `SynergyManager`·`SynergyListPanel`·인디케이터 어디에도 참조되지 않는다. SummonTraitStore는 데이터를 저장하지만, 그 데이터를 읽어가는 프로덕션 코드(consumer)가 없다.

**코드 구조**:

- **파일 위치**:
  - `Assets/Scenes/Battle/Feature/SummonTrait/SummonTraitStore.cs`
  - `Assets/Scenes/Battle/Feature/SummonTrait/SummonTraitDistributor.cs`

- **SummonTraitStore 공개 API**:
  - `Initialize(Dictionary<UnitLoadOutData, SynergyDefinitionData> traitMap)`
  - `GetTrait(UnitLoadOutData unit)`
  - 속성 `All` (전체 조회)

- **SummonTraitStore 참조 그래프** (Assets 전체 grep):
  ```
  Assets/Scenes/Battle/Feature/SummonTrait/SummonTraitStore.cs            [정의]
  Assets/Scenes/Battle/Feature/SummonTrait/SummonTraitDistributor.cs      [Initialize 호출]
  Assets/Tests/Editor/SummonTraitStoreTests.cs                            [테스트]
  Assets/Tests/Editor/SummonTraitDistributionTests.cs                     [테스트]
  ```
  → **프로덕션 consumer는 SummonTraitDistributor 단 1곳뿐이며, 그것도 쓰는 쪽(write)이지 읽는 쪽(read)이 아님.**

- **SummonTraitDistributor 동작**:
  - `OnBattleStart()` 시점에 분배 결과를 계산해 `SummonTraitStore.Initialize(...)` 호출

- **읽는 측 부재**: `SynergyManager`, `SynergyListPanel`, `SynergyIndicator`, `SynergyDetailPanel`, `SynergyController*` 중 어느 것도 `SummonTraitStore.Instance` 또는 `GetTrait` / `All` 을 호출하지 않음.

### 질문 5: 인디케이터가 안 뜨는 근본 원인

**도메인 표현**: 원인은 **(c) 별도 통합 코드 부재**다. 분배 로직과 저장소는 구현되어 있고 UI도 준비되어 있지만, 저장소의 데이터를 SynergyManager로 끌어와 `SynergyActivations`에 합치는 통합 계층이 존재하지 않는다.

**코드 구조**:

| 계층 | 상태 | 근거 |
|------|------|------|
| 데이터 분배 | 구현됨 | `SummonTraitDistributor.OnBattleStart()` |
| 저장소 | 구현됨 | `SummonTraitStore.Initialize` / `GetTrait` / `All` |
| **통합 조율** | **부재** | `SynergyManager`가 `SummonTraitStore`를 참조하지 않음. `_synergyMembers`·`_synergyActivations`에 SummonTrait 데이터 합산 없음 |
| UI 렌더링 | 구현됨 | `SynergyListPanel`이 `SynergyManager.SynergyActivations` 읽음. 데이터만 들어오면 표시 가능 |

**판정 근거 코드**:
- `SynergyManager._synergyActivations` 딕셔너리는 `SummonerEffect`만 채워짐 → UI가 읽는 데이터에 SummonTrait 항목 자체가 없음
- `SummonTraitStore`에는 데이터가 있지만 `SynergyManager`가 모름
- `SynergyManager.cs` 주석이 해당 통합이 "향후 추가" 예정임을 명시

## 발견한 특이사항

- **SynergyType enum의 미사용**: `SynergyType.SummonerEffect` / `SummonTrait` 두 값 모두 Assets 하위 코드에서 분기·비교에 사용된 곳이 없음. 데이터 분류 표시 용도로만 존재.
- **SynergyManager 초기화 타이밍**: `Start()`에서 1회 수집. SummonTraitDistributor의 `OnBattleStart()` 이벤트 발생 시점보다 먼저 실행되었을 가능성 있음 (라이프사이클 순서상). 즉 SynergyManager가 SummonTraitStore를 참조하더라도 타이밍 결합 검토 필요.
- **SummonTraitStore가 싱글톤**: `SummonTraitStore.Instance` 형태로 접근 가능 (grep 결과 기준).
- **테스트 커버리지 있음**: `SummonTraitStoreTests`, `SummonTraitDistributionTests` 존재 — 분배·저장 로직 자체는 검증되어 있음.

## 탐색 한계

- **SynergyManager의 라이프사이클 순서**: `Start()`에서 SummonTraitDistributor의 초기화보다 먼저 실행되는지의 정확한 보장은 SceneSingleton 구현/Awake 순서에 따라 다를 수 있어 정적 탐색만으로 단정 불가.
- **인디케이터 슬롯 수**: `SynergyListPanel.indicators[]` 배열 크기가 SummonerEffect(최대 4개)에 맞춰져 있는지, SummonTrait(8개) 합산까지 수용 가능한지는 Inspector 설정 사항이라 코드 탐색으로 확인 불가.
- **SummonTrait 자산 등록 여부**: TACD-300에서 생성된 자산이 실제로 SummonPool 내 각 UnitLoadOutData에 어떤 방식으로 연결되어 있는지(또는 SummonTraitDistributor가 어떤 소스에서 자산을 로드하는지)는 본 탐색 범위 외.
