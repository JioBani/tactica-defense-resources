## 후보 식별 시 제외한 항목 (참고)

- **CreateFactoryInScene() 호출 반복**: 대다수 SynergyManager 통합 테스트가 반복적으로 호출하나, HandleDefenderChanged 계열(TC-401~405) 은 Factory 가 불필요. SetUp 일괄 셋업으로 옮기면 불필요 인스턴스가 생성된다는 점에서 인라인 유지가 더 명확.
- **`traitA/B/C` 인덱싱 명명**: 'A/B/C' 는 단순 카운팅이 아니라 unique trait 식별 의도가 강해 풀네임화하더라도 의미가 더해지지 않음. 가독성 유지를 위해 그대로 둠.
- **§T-2 static 함수 제한**: 본 파일의 정적 헬퍼들은 정적 상태(`SceneSingleton._instance` / `GlobalEventBus.Handlers`) 에 접근하는 테스트 인프라로, 이전 사이클들에서 합의된 패턴 유지.
- **§E-4 매개변수화**: TC-202 가 이미 `[TestCase]` 3 종으로 매개변수화되어 있음. TC-401~404 는 결정 테이블의 각 행이 의도가 다르고 셋업 분리가 명확하여 분리 유지가 가독성에 우월.
