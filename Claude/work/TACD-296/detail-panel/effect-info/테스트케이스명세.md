# TACD-296 — effect-info 테스트 케이스 명세

## 구현 단위 범위

- 구현 단위 식별자: `effect-info`
- 다루는 DoD 묶음 (SR 그룹 B — 시너지 효과 정보 표시):
  - DoD-S7: 이름·효과 설명·티어 임계치 목록·티어별 효과 수치를 정의 저장소에서 조회하여 표시.
  - DoD-S8: 효과 설명을 정의 저장소가 제공하는 원본 표현으로 표시 (플레이스홀더 치환 없음).
  - DoD-S9: 모든 티어를 임계치 순서대로 나열, 활성 티어를 다른 티어와 구분되는 표시 상태로 둠.
  - DoD-S10: 표시 상태 동안 활성 티어 변경 알림을 구독하여 강조 표시를 갱신.

## 테스트 모드 결정

- **EditMode** — 효과 정보 섹션은 일반 클래스(MonoBehaviour 아님) 이고 검증 대상은 외부 표면(Bind/Rebind/Unbind) 호출 후의 VisualElement 표시 상태(`text` / USS 클래스 / 자식 개수) + RxValue 알림 구독 결과. 시간 진행 / 물리 / 렌더 의존 없음.

## 방법론별 도출

### 1. 유스케이스 테스트

#### UC-1: 정적 정보 표시 (DoD-S7, DoD-S8)

- 정상 흐름:
  - **TC-701**: Bind(activation) → `_nameLabel.text == Definition.DisplayName`.
  - **TC-702**: Bind(activation) → `_descriptionLabel.text == Definition.Description`.
  - **TC-703**: Bind(activation) → `tier-list` 자식 개수 == `Definition.Tiers.Count`.
  - **TC-704**: 각 자식이 `tier-item` USS 클래스를 보유.
  - **TC-903**: tier-list 자식 순서가 `Definition.Tiers` 의 입력 순서(임계치 오름차순)와 동일.
- 예외 흐름:
  - **TC-1101**: Bind(null) → 라벨/티어 컨테이너 상태 불변, 예외 없음.

#### UC-2: 플레이스홀더 보존 (DoD-S8)

- 동치 분할 — description 의 형태:
  - **TC-801**: `"공격력 +@AttackPower@% / @Bonus*3@턴"` 형태 (단일·곱셈 플레이스홀더 혼합) → 라벨에 동일 문자열 그대로.
  - **TC-802**: `description == null` → 라벨 text 가 `""`.

#### UC-3: 활성 티어 강조 (DoD-S9, DoD-S10)

- 정상 흐름:
  - **TC-901**: 카운트 ≥ 1단계 임계치 → 1단계 항목에 `tier-item--active`, 2단계 항목에 없음.
  - **TC-1001**: Bind 후 카운트를 2단계 임계치 이상으로 변경 (`Recalculate`) → 2단계 항목에 `tier-item--active`, 1단계 항목에서 제거.
- 예외·경계 흐름:
  - **TC-902**: 카운트 < 모든 임계치 → 어떤 항목도 `tier-item--active` 보유하지 않음.
  - **TC-1002**: 활성 상태(1단계) → 카운트 0 으로 감소 → 모든 항목에서 강조 제거.

#### UC-4: 라이프사이클 정리 책임 (DoD-S10, 본 단위 자기 일관성)

- **TC-1003**: Bind 후 Unbind → 이후 `Recalculate` 로 ActiveTier 변경해도 강조 항목 없음 (구독 해제 검증).
- **TC-1102**: Bind(actA) → Rebind(actB) → actA 의 `Recalculate` 변경이 actB 의 tier-list 강조에 영향 없음 (구독 교체 검증).
- **TC-1103**: Bind → Unbind → name/description 라벨 `""`, tier-list 자식 0.

### 2. 동치 분할

#### 입력: description

- 관련 DoD: DoD-S8
- 유효 그룹:
  - 플레이스홀더 포함 문자열 — **TC-801**
  - 빈 문자열 (간접 — TC-802 의 null 케이스로 흡수)
- 무효 그룹:
  - null — **TC-802**

#### 입력: SynergyActivation.ActiveTier 값

- 관련 DoD: DoD-S9
- 유효 그룹 (대표값):
  - 1 단계 활성 (카운트 == 1단계 임계치) → **TC-901**
  - 비활성 (카운트 < 1단계 임계치) → **TC-902**

### 3. 결정 테이블

#### 결정 포인트: 활성 티어 ↔ 항목 강조 매핑 (DoD-S9)

- 조건:
  - C1: `_tierListContainer != null` (픽스처 정상)
  - C2: `ActiveTier.HasValue`
  - C3: `_tierItemsByStage.ContainsKey(ActiveTier.Tier)`
- 결정 테이블:

  | 조합 | C1 | C2 | C3 | 기대 결과 | TC ID |
  |---|---|---|---|---|---|
  | 1 | T | T | T | 해당 단계 항목에만 `tier-item--active` | TC-901 |
  | 2 | T | F | * | 어떤 항목도 강조 없음 | TC-902 / TC-1002 |
  | 3 | T | T | F | (정의 데이터 결손 — 본 단위 픽스처 외) | 미커버 |

#### 결정 포인트: Bind / Rebind / Unbind 의 구독 상태

- 조건: 어느 라이프사이클 호출 이후 ActiveTier 변경이 발생했는가
- 결정 테이블:

  | 조합 | 마지막 호출 | ActiveTier 변경 대상 | 기대 결과 | TC ID |
  |---|---|---|---|---|
  | 1 | Bind(act) | act 의 카운트 변경 | act 의 강조 갱신 | TC-1001 |
  | 2 | Rebind(actB) | actA(이전 시너지) 변경 | actB 강조 불변 | TC-1102 |
  | 3 | Unbind | act 변경 | 강조 갱신 없음 | TC-1003 |

## 미커버 / 보류 항목

- **활성 티어 단계가 정의의 Tiers 에 존재하지 않는 케이스 (예: tier=99 가 ActiveTier 로 설정됨)** — SynergyActivation 의 `FindActiveTier` 가 항상 Definition.Tiers 의 원소를 반환하므로 도메인적으로 발생 불가. 본 단위 픽스처로도 재현 비자연 — 미커버로 기록.
- **sectionRoot 가 null 또는 자식 노드 미발견** — 컴포넌트 외부 인접 책임 (UXML 자산 손상). impl 메모 §자율 판단 5 에 따라 null 가드로 조용히 통과만 검증 필요시 추후. 본 단위 DoD 범위 외.
