## 외부 표면 (요구사항 테스트 단계의 입력)

```csharp
// Scenes.Battle.Feature.Ui.SynergyInfo.SynergyDetailEffectSection
public SynergyDetailEffectSection(VisualElement sectionRoot);
public void Bind(SynergyActivation activation);
public void Rebind(SynergyActivation activation);
public void Unbind();
```

- 생성자에서 sectionRoot 의 자식 (`synergy-name`/`synergy-description`/`tier-list`) 을 쿼리하여 캐시. sectionRoot 가 null 이거나 자식이 없는 픽스처에서는 각 캐시가 null 로 남아 본체 메서드가 조용히 통과 (안전망).
- `Bind(activation)` 동작 (DD §그룹 B 비즈니스 로직 §1·§2·§3):
  - `Definition.DisplayName` → name 라벨
  - `Definition.Description` → description 라벨 (플레이스홀더 원본 그대로 — DoD-S8, 본 이슈에서 치환 없음)
  - `Definition.Tiers` 를 SynergyTier 컨벤션의 *임계치 오름차순* 순서 그대로 순회하여 항목 생성 → `tier-list` 컨테이너에 추가. 항목 = "{Tier}단계" / "{RequiredCount}명" / 각 Constants 항목 라벨
  - 현재 `ActiveTier.Value` 가 있으면 그 단계 항목에만 `tier-item--active` 클래스 (DoD-S9)
  - `ActiveTier.OnChange += handler` 구독 개시 (DoD-S10). handler 는 매번 `RenderActiveTier(newTier)` 호출하여 모든 항목에서 `--active` 제거 후 새 active 단계에만 다시 부여
- `Rebind(activation)` — `Unbind()` 후 `Bind(activation)` 단순 위임. 항목 잔존 / 시각 클래스 잔존 회피는 `Unbind` 의 `ClearTierItems()` 와 `UnsubscribeActiveTier()` 에서 처리.
- `Unbind()` — 구독 해제 + tier 항목 컨테이너 비움 + 항목 stage→VisualElement 맵 비움 + name/description 라벨 빈 문자열로 정리 + `_activation = null`.

## 자율 판단 (계획 외 결정)

1. **공통 유틸 `SerializableDictionary<TKey, TValue>` 에 `IEnumerable<KeyValuePair<TKey, TValue>>` 구현 추가** — DD §그룹 B 비즈니스 로직 §1 이 "`SynergyTier.Constants` 의 각 상수 표시" 를 결정했으나 `SerializableDictionary` 가 IEnumerable 미구현 + 내부 `_dict` 가 private 라 외부 순회 불가능했음. 다음 후보 검토:
   - (A) `SerializableDictionary` 에 `GetEnumerator` 추가 — *Dictionary 인터페이스 흉내내는 공통 유틸의 기본 표면 누락 보강*. 본 이슈 외 다른 영역에도 자연 이득. 채택.
   - (B) `SynergyTier` 에 `EnumerateConstants()` 추가 — `SerializableDictionary` 내부 접근 불가라 결국 (A) 의존. 미채택.
   - (C) 리플렉션으로 우회 — 부적절. 미채택.
   - (A) 가 *공통 유틸의 사소 보강* 이라 다른 영역에 회귀 위험 거의 없음 (행동 추가만, 기존 메서드 시그니처 변경 없음). 새 큰 설계 결정 아님 — DD 가 결정한 데이터 흐름의 *접근 표면 부재* 라는 작은 결손을 본 사이클 자율 영역으로 보강.
2. **티어 항목의 표시 형식** — `"{Tier}단계"` / `"{RequiredCount}명"` / 상수는 한 줄당 `"{Name} {Value}"` 평문. 더 보기 좋은 단위·접미사·% 변환은 UI 디자인 단계 (TACD-298 결정 §4: 프로토타입 수준).
3. **알림 구독 표면 선택** — DD §그룹 B 비즈니스 로직 §3 의 "`ActiveTier.OnChange + OnTierActivated/Changed/Deactivated 중 일반 표면`" 중 `RxValue<SynergyTier?>.OnChange` 단일 구독 채택. 이 표면이 모든 활성↔비활성·티어 변경 케이스를 하나로 흡수하므로 `OnTierActivated/Changed/Deactivated` 는 중복 알림이 되어 미사용.
4. **`Bind(null)` 처리** — 조용히 무시 (안전망). 호출 계약상 발생하지 않음.
5. **sectionRoot null 또는 자식 미발견** — 캐시 null 상태로 두고 본체 메서드가 null 가드로 조용히 통과. 픽스처 테스트가 sectionRoot 를 직접 구성하지 않는 경우의 컨트롤러 단위 테스트 호환성을 위함.

## 테스트 멤버에 전달하는 주의 사항

- 본 사이클의 요구사항 테스트 대상 DoD: **S7 / S8 / S9 / S10**.
  - **DoD-S7**: `Bind` 후 name/description/tier 항목이 표시되는가 (라벨 text 검증, tier-list 자식 개수가 `Definition.Tiers.Count` 와 일치)
  - **DoD-S8**: description 라벨의 text 가 `Definition.Description` 과 *정확히 일치* (플레이스홀더 치환 없음 — `@xxx@` 같은 문자열이 그대로 보존)
  - **DoD-S9**: 모든 티어 항목이 임계치 순서로 나열되며, 활성 티어가 있으면 그 단계 항목에만 `tier-item--active` 클래스가 붙고 다른 항목에는 없음
  - **DoD-S10**: `Bind` 후 `activation.ActiveTier.Value = newTier` 변경 시 강조 항목이 갱신됨 — `Recalculate(count)` 를 호출하여 ActiveTier 가 바뀌도록 유도 가능 (SynergyActivation.Recalculate(int) 가 ActiveTier 를 임계치에 따라 계산)
- `member-list` 의 stub (`SynergyDetailMemberList`) 는 본 사이클에서 변경하지 않음 — Bind/Rebind/Unbind no-op 그대로. 본 사이클의 `SynergyDetailEffectSection.Bind` 가 활성 티어 강조까지 처리하므로 멤버 리스트 미구현은 본 사이클의 테스트에 영향 없음.
- 픽스처 전략 권장: `SynergyDefinitionData` 의 `displayName`/`description`/`tiers` 등 private 필드를 리플렉션으로 주입 + `SynergyTier` struct 도 `tier`/`requiredCount`/`constants` private 필드 리플렉션 주입. `SerializableDictionary<string, float>` 는 이제 IEnumerable 구현 + 인덱서 (`this[TKey]`) 로 값 셋팅 가능.
