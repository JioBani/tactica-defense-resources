## 외부 표면 (요구사항 테스트 단계의 입력)

```csharp
// Scenes.Battle.Feature.Ui.SynergyInfo.SynergyDetailPanel : MonoBehaviour
public bool IsVisible { get; }       // 표시 상태 관측
public SynergyActivation Current { get; }  // 표시 대상 슬롯 (비표시면 null)
public void Show(SynergyActivation activation);  // 외부 진입점 (분기 자체 캡슐화)
public void Hide();                  // 외부 비표시 전이
```

`Show(activation)` 의 동작 분기 (DD §그룹 A 비즈니스 로직 §1 그대로):
- `activation == null` → 무시 (안전망, DoD 명세 외)
- `Current == null` → 비표시 → 표시 진입 (Enter)
- `Current == activation` → 토글 닫기 (Exit) — DoD-S4 "현재 표시 대상 시너지의 아이콘 재클릭"
- `Current != activation` → 표시 상태 유지 교체 (Replace, 비표시 거치지 않음) — DoD-S5

내부 닫기 트리거 (모두 `ExitVisible` 로 수렴):
- `close-button` 클릭 (UXML 의 닫기 버튼)
- 패널 루트 외부 영역 PointerDown — `_root.panel.visualTree` 에 `TrickleDown` 으로 등록한 캡처 단계 콜백이 `evt.target` 이 `_root` 또는 자손이 아닐 때 닫기로 해석.

자식 섹션 라이프사이클 전달:
- `EnterVisible` → 두 섹션 `Bind(activation)`
- `Replace` → 두 섹션 `Rebind(activation)` (단일 호출, 컨트롤러 측 분기 무)
- `ExitVisible` → 두 섹션 `Unbind()`

## 자율 판단 (계획 외 결정)

자율 판단으로 추가된 사소 결정 — DD 4축 결정 변경 아님:
1. **USS 클래스 명명** — `synergy-detail-panel` (BEM 형식의 블록명) + `synergy-detail-panel--hidden` (modifier). `__close` / `__effect-section` / `__member-section` 자식 클래스. DD 는 "USS 클래스로 활성 티어 / 활성 소환수 시각 구분 제공" 만 결정했고 구체 명명은 본 단계 자율 영역.
2. **VisualElement 이름 명명** — `synergy-detail-root` / `close-button` / `effect-section` / `member-section`. 코드의 `Q<T>(name)` 쿼리 키로 사용.
3. **`Show(null)` 처리** — 조용히 무시 (에러 로그 없음). DD 예외 처리 표에 "호출 계약상 발생하지 않음 / 호출자 부담 감소" 로 결정됨.
4. **컴포넌트 OnEnable 시 자식 섹션 인스턴스 생성** — `Awake` 가 아닌 `OnEnable` 선택. `UIDocument.rootVisualElement` 가 `Awake` 시점에 빌드되지 않을 수 있어 안전한 `OnEnable` 시점에 쿼리.

## 테스트 멤버에 전달하는 주의 사항

**중요 — stub 클래스에 대해**:
- `SynergyDetailEffectSection` / `SynergyDetailMemberList` 는 본 lifecycle 사이클에서 *시그니처 stub* 만 생성됨. Bind/Rebind/Unbind 본체가 no-op (빈 메서드) 이므로 *미구현으로 오인하지 말 것*. DD §그룹 B / C 의 본체 구현은 후속 사이클 (`effect-info` / `member-list`) 의 책임이며, 본 사이클의 요구사항 테스트는 *그룹 A (DoD-S1 ~ S6)* 즉 라이프사이클 / 진입점 통합 책임에 한정한다.
- 따라서 본 사이클의 요구사항 테스트가 검증할 것:
  - DoD-S1 / S2 — `SynergyListPanel.OnIndicatorClicked` → `SynergyInfoPanel.HandleIndicatorClicked` → `SynergyDetailPanel.Show(activation)` 라인이 깨지지 않고 표시 진입을 일으키는가
  - DoD-S3 — 시너지 종류 분기 코드 없음 (`Show` 내부에서 `SynergyType` 참조 없음)
  - DoD-S4 — 동일 시너지로 재진입 시 `IsVisible` 가 false 로 전이 (토글 닫기)
  - DoD-S5 — 다른 시너지로 진입 시 `IsVisible` 유지 + `Current` 가 새 시너지로 갱신 + 두 섹션의 `Rebind` 호출 발생 (비표시 거치지 않음)
  - DoD-S6 — `Current` 슬롯은 항상 단일 인스턴스
  - 닫기 입력 — 닫기 버튼 click / 바깥 영역 PointerDown 가 `ExitVisible` 로 수렴
- 두 섹션의 Bind/Rebind/Unbind 가 *호출되었는지* 만 검증 가능 (호출 후 무엇이 표시되는지는 후속 사이클 책임). 호출 발생 검증은 stub 에 카운터를 임시 두는 형태가 아니라 *공개 표면을 직접 호출하여 컨트롤러 상태(`Current`/`IsVisible`) 의 결과적 변화* 로 검증하는 것이 자연.
