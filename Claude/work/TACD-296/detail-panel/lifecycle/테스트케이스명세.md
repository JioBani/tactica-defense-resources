# TACD-296 — lifecycle 테스트 케이스 명세

## 구현 단위 범위

- 구현 단위 식별자: `lifecycle`
- 다루는 DoD 묶음 (SR 그룹 A — 상세 패널 라이프사이클 + 진입점 통합):
  - DoD-S1: 시너지 목록 패널이 클릭 입력을 발행 (식별 가능한 도메인 키 포함).
  - DoD-S2: 시너지 상세 패널이 클릭 입력을 수신하여 표시 대상 설정 + 표시 전이.
  - DoD-S3: 시너지 종류와 무관한 동일 진입·표시 책임.
  - DoD-S4: 다음 입력 중 하나에서 비표시 전이 — 재클릭 / 닫기 버튼 / 바깥 영역.
  - DoD-S5: 표시 상태에서 다른 시너지 클릭 시 비표시를 거치지 않고 교체.
  - DoD-S6: 한 번에 한 시너지만 표시 상태로 유지.

## 테스트 모드 결정

- **EditMode** — 전 케이스. 본 단위의 검증 대상은 `SynergyDetailPanel` 의 표시 대상 슬롯 전이 + 진입점 라우팅 책임으로, 시간 진행 / 물리 / 렌더 의존이 없다. UIDocument 의 visualTree 빌드는 PlayMode 영역이지만, *컨트롤러 자체의 상태 전이 책임* 은 OnEnable 이 만든 자식 표면 (`_root` / `_closeButton` / `_effectSection` / `_memberList`) 을 EditMode 에서 직접 구축하여 검증 가능 (impl 메모 §테스트 멤버에 전달하는 주의 사항 — "공개 표면을 직접 호출하여 컨트롤러 상태의 결과적 변화로 검증" 지침과 정합).

## 방법론별 도출

### 1. 유스케이스 테스트

#### UC-1: 클릭 입력 발행·수신·표시 진입 (DoD-S1, DoD-S2)

- 정상 흐름:
  - **TC-101**: `SynergyListPanel.OnIndicatorClicked` 이벤트가 `Action<SynergyActivation>` 형으로 공개 표면에 존재. (반사 조회로 검증.)
  - **TC-201**: 비표시 상태(`IsVisible=false`)에서 `Show(activation)` → `IsVisible=true`, `Current==activation`.
  - **TC-202**: `SynergyInfoPanel.HandleIndicatorClicked(activation)` (private 라우터, 반사로 호출) → 주입된 detailPanel 의 `IsVisible=true`, `Current==activation` (DoD-S1·S2 의 라우팅 통로 검증).
- 예외 흐름:
  - **TC-203**: `Show(null)` → `IsVisible=false` 유지 (조용히 무시; 로그/예외 없음).

#### UC-2: 비표시 전이 트리거 (DoD-S4)

- 정상 흐름:
  - **TC-401**: 표시 상태(`Show(actA)`) → 다시 `Show(actA)` → `IsVisible=false`, `Current==null` (재클릭 토글 닫기).
  - **TC-402**: 표시 상태에서 `_closeButton.clicked` invoke → `IsVisible=false`, `Current==null`.
  - **TC-403**: 표시 상태에서 `Hide()` → `IsVisible=false`.
- 예외 흐름:
  - **TC-404**: 비표시 상태에서 `Hide()` → 예외 없이 `IsVisible=false` 유지.

#### UC-3: 표시 대상 교체 (DoD-S5)

- 정상 흐름:
  - **TC-501**: 표시 상태(`Show(actA)`) → `Show(actB)` → `IsVisible=true` 유지, `Current==actB`.
- 대안 흐름:
  - **TC-502**: 동일 시퀀스 도중 임의 시점에 `IsVisible` 가 false 로 떨어지지 않음 (Show(actA) 직후·Show(actB) 직후 모두 true).

### 2. 동치 분할

#### 입력: Show 의 인자 `activation`

- 관련 DoD: DoD-S2, DoD-S6
- 유효 그룹:
  - 새 SynergyActivation 인스턴스 (현재 슬롯과 다름) — 대표값: actA → **TC-201**
  - 현재 슬롯과 동일 인스턴스 — 대표값: actA(2회) → **TC-401**
- 무효 그룹:
  - null — **TC-203**

#### 입력: 표시 진입 이력 (단일 슬롯 불변식)

- 관련 DoD: DoD-S6
- 유효 그룹:
  - 단일 Show 호출 후 — Current 단일 → **TC-601** (TC-201 와 동일 검증의 다른 관점)
  - 다회 Show 후 — Current 가 항상 가장 최근 호출의 단일 슬롯.

### 3. 결정 테이블

#### 결정 포인트: Show(activation) 호출 시 분기

- 관련 DoD: DoD-S2, DoD-S3, DoD-S4(재클릭), DoD-S5
- 조건:
  - C1: `activation == null`
  - C2: `Current == null` (비표시 상태)
  - C3: `Current == activation` (현재 슬롯과 동일 인스턴스)
- 결정 테이블:

  | 조합 | C1 | C2 | C3 | 기대 결과 | TC ID |
  |---|---|---|---|---|---|
  | 1 | T | * | * | 무시 (IsVisible/Current 불변) | TC-203 |
  | 2 | F | T | (불가) | Enter — IsVisible=true, Current=activation | TC-201 |
  | 3 | F | F | T | Exit (토글 닫기) — IsVisible=false | TC-401 |
  | 4 | F | F | F | Replace — IsVisible=true 유지, Current=activation | TC-501 / TC-502 |

- 불가능한 조합: C2=T && C3=T (비표시 상태에서는 Current==null 이므로 activation 과 동일 인스턴스 비교 불가).

#### 결정 포인트: 시너지 종류 무관 (DoD-S3)

- 관련 DoD: DoD-S3
- 조건: 호출되는 activation 의 정의 데이터의 임의 속성 (종류 / 티어 구성 등) — 본 컨트롤러는 조회·분기하지 않음.
- 결정 테이블:

  | 조합 | activation 정의의 속성 | 기대 결과 | TC ID |
  |---|---|---|---|
  | 1 | 정의 인스턴스 A (속성 구성 임의) | Show 동작 동일 — Enter | TC-301 (a) |
  | 2 | 정의 인스턴스 B (속성 구성 임의, A 와 다름) | Show 동작 동일 — Enter | TC-301 (b) |

  → 두 케이스 모두 동일 분기로 진입함을 검증 → 종류 분기 코드 없음.

## 미커버 / 보류 항목

- **외부 영역 PointerDown 닫기 트리거 (DoD-S4 의 한 항목)** — 사유: UI Toolkit 의 `PointerDownEvent` 디스패치는 패널(Panel) 인스턴스가 필요한 PlayMode 영역. 본 EditMode 명세에서는 비표시 전이의 *두 다른 경로* (재클릭 / 닫기 버튼) 와 외부 진입점 `Hide()` 로 *비표시 전이 책임* 자체는 검증되며, 외부 영역 클릭 경로의 추가 확인은 PlayMode 영역으로 위임. 본 lifecycle 단위 범위 내에서는 추적성 보존 목적의 미커버 항목으로 기록.
