# TACD-246: 정비 페이즈 카메라 전환 (아군 ↔ 침략자 프리뷰)

> 상위 이슈: TACD-236 [전장 - 침략자] - 웨이브 프리뷰

---

## 작업 내용

정비 페이즈에서 소환수 배치 카메라(X=0)와 침략자 프리뷰 카메라(X=3)를 전환할 수 있는 기능을 구현한다.

### 요구사항

1. 정비 페이즈에서 전환 버튼을 눌러 아군 배치 ↔ 침략자 프리뷰 카메라를 전환
2. 프리뷰 카메라 전환 시 침략자 영역을 표시 (실제 침략자 엔티티 활용은 별도 이슈 TACD-247 범위)
3. 전투 시작 시 프리뷰 상태여도 자연스럽게 전투 카메라(X=1.5, ortho=4.0)로 이동
4. 기존 페이즈별 카메라 이동과 충돌하지 않아야 함
5. 전투 페이즈에서는 카메라 전환 버튼이 동작하지 않아야 한다

### 카메라 위치

| 뷰 | X 위치 |
|----|--------|
| 소환수(아군) 측 | 0 |
| 침략자 측 | 3 |

---

## 기존 코드 분석

### CameraControlManager (`Assets/Scenes/Battle/Feature/CameraControl/CameraControlManager.cs`)
- `IStateListener<PhaseType>` 구현, RoundManager에 등록
- 전투 진입 시: X=1.5, ortho=4.0 (SetCombatMode)
- 전투 종료 시: X=0, ortho=3.3 (SetMaintenanceMode)
- `ShowAggressorSide()` / `ShowDefenderSide()` — Y축 이동 (현재 사용 중인 뷰 전환)

### SwitchViewManager (`Assets/Scenes/Battle/Feature/Ui/SwitchViewManager.cs`)
- 전환 버튼 토글 (`SwitchView()`) → CameraControlManager의 Y축 이동 호출
- 라운드 정보 패널 표시/숨김
- 정비 페이즈 진입 시 라운드 정보 업데이트

### 변경 방향

- **CameraControlManager**: 기존 Y축 뷰 전환(`ShowAggressorSide`/`ShowDefenderSide`)을 X축 기반(X=0 ↔ X=3)으로 변경
- **SwitchViewManager**: 전투 시작 시 프리뷰 상태 해제 (전투 카메라 전환은 기존 SetCombatMode가 처리)
- 기존 페이즈 전환 카메라(SetCombatMode/SetMaintenanceMode)와 충돌 방지 필요
