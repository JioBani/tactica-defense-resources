# TACD-276: 소환술사 정의 데이터 에셋 구조 설계

> 상위 이슈: TACD-273 [전장 - 소환술사] - 데이터 스키마 제작

## 요구사항

소환술사의 **인게임(전장) 데이터**를 설계한다. 소환술사 전체 데이터(이름, 성격, 설정, 호감도, 스킨 등)가 아니라, 전장에서 소환술사를 식별하고 동작시키는 데 필요한 최소한의 데이터만 구성한다.

구체적으로 두 층위:
1. **소환술사 정의** — 최소한의 식별 데이터 (이름, 시너지 연결, 소환수 풀 등)
2. **인게임 소환술사 데이터** — 전장에서 소환술사가 동작하기 위한 데이터 (전투 스탯, 배치 정보 등)

## 배경지식

### 소환술사의 전장 역할

- 출전 편성 시 4명 선택 → 각 소환술사의 소환수 8체가 소환 터미널 풀에 등장 (총 32체)
- 침략자가 라인 끝까지 도달하면 소환술사와 직접 교전
- 소환술사도 공격하며, 피격 시 HP 감소
- 한 명이라도 HP가 0이 되면 **전장 패배**
- 소환술사의 육성 상태(레벨, 승급)에 따라 소환술사 효과(시너지)가 강화됨

### 기존 소환수(Defender)와의 차이

| 구분 | 소환수 (Defender) | 소환술사 |
|------|------------------|---------|
| 스탯 진행 | 성급(별) 기반 (1성/2성/3성) | 레벨 + 승급 기반 |
| 전장 배치 | 그리드 칸에 드래그 배치 | 라인 끝에 출전 편성 순서대로 고정 |
| 시너지 | 소환술사 효과 + 소환수 특성 | 소환술사 효과만 (고유) |

## 완료 정의

- 소환술사 정의 데이터(식별, 시너지 연결, 소환수 풀)가 확정된다
- 인게임 소환술사 데이터(전투 스탯 등)가 확정된다
- 테스트용 소환술사 데이터 에셋이 생성된다

## 기존 코드 참조

- `UnitDefinitionData`: 유닛 정체성 (id, displayName, icon, illustration, summonerEffect)
- `UnitStatsByLevelData`: 성급 기반 스탯 진행 (StarStatRecord로 별 단위 값 관리)
- `UnitLoadOutData`: 정의 + 스킬 + 스탯을 조합하는 복합 에셋
- `SynergyDefinitionData`: 시너지 정의 (SynergyType으로 소환술사 효과/소환수 특성 구분)

## 결정 사항

1. **전장 배치**: 라인 끝에 출전 편성 순서대로 고정 배치
2. **스탯 범위**: 소환수와 동일한 14개 스탯 전부 사용
3. **스탯 체계**: 기존 `UnitStatsByLevelData` 재사용 (StarStatRecord 인덱스를 승급 단계로 해석)
