# 컨텍스트 수집

## 1. 요구사항 수집

### Jira 티켓 (TACD-276)

- **작업 범위**: 소환술사의 인게임(전장) 데이터 에셋 구조 설계
- **완료 기준**: 소환술사 정의 데이터 ScriptableObject 구조가 확정되고 에셋이 생성됨

### 설계 범위

소환술사 전체 데이터(이름, 성격, 설정, 호감도, 스킨 등)가 아니라, **전장에서 필요한 최소한의 데이터**만 구성한다.

두 층위:
1. **소환술사 정의** — 최소 식별 데이터 (이름, 시너지 연결, 소환수 풀 등)
2. **인게임 소환술사 데이터** — 전투 스탯, 배치 정보 등

### 기획서 핵심 규칙

**출전 편성** (00 기획서 2.1.3)
- 전장 입장 전 보유 소환술사 중 4명 선택
- 각 소환술사마다 8체의 소환수가 소환 터미널 풀에 등장 (4 × 8 = 32체)

**전장 내 소환술사 역할** (00 기획서 3.1.2)
- 라인 끝에 출전 편성 순서대로 고정 배치
- 침략자가 라인 끝까지 도달하면 소환술사와 직접 교전
- 소환술사도 공격하며, 피격 시 HP 감소
- 한 명이라도 HP가 0이 되면 전장 패배

**소환술사 효과(시너지)** (07 기획서 5.2)
- 각 소환술사는 고유한 소환술사 효과를 보유
- 소환술사의 소환수는 해당 효과를 자동 보유
- 육성 상태(레벨, 승급)에 따라 효과가 강화됨

**스탯 체계** (07 기획서 1.1~1.2)
- 소환수와 동일한 14개 스탯 전부 사용
- 기존 `UnitStatsByLevelData` 재사용 (StarStatRecord 인덱스를 승급 단계로 해석)

### 사용자 결정 사항

1. **전장 배치**: 라인 끝에 출전 편성 순서대로 고정 배치
2. **스탯 범위**: 소환수와 동일한 14개 스탯 전부 사용
3. **스탯 체계**: 기존 `UnitStatsByLevelData` 재사용

## 2. 기존 코드 분석

### 관련 데이터 구조

**UnitDefinitionData** — 유닛 정체성
- id (int), displayName (string), description (string)
- icon (Sprite), illustration (Sprite)
- summonerEffect (SynergyDefinitionData) — 소환술사 효과 1개
- cost (int)
- Synergies 프로퍼티로 시너지 목록 반환

**UnitStatsByLevelData** — 성급 기반 스탯 진행
- 14개 스탯 각각 StarStatRecord로 관리
- StarStatRecord: baseValuesByStar (List<float>) + additionalPerExtraStar (float)
- GetValue(int star)로 특정 성급의 스탯값 조회

**UnitLoadOutData** — 인게임 복합 에셋
- unit (UnitDefinitionData) + skill (SkillDefinitionData) + stats (UnitStatsByLevelData)
- costByStar / refundByStar — 성급별 비용/환원 마나

**SynergyDefinitionData** — 시너지 정의
- id (SynergyId enum), displayName, description, icon
- synergyType (SummonerEffect / SummonTrait)
- tiers (List<SynergyTier>) — 활성화 임계치 + 상수

### 기존 패턴

- ScriptableObject 기반 데이터 정의
- 컴포지트 구조: LoadOut = Definition + Skill + Stats
- enum 기반 타입 식별 (SynergyId, UnitStatKind 등)

## 3. 외부 리서치

### 유사 게임의 인게임 캐릭터 데이터 패턴

| 게임 | 정의 데이터 (컬렉션) | 인게임 데이터 (전투) |
|------|---------------------|---------------------|
| TFT | Champion 정의 (이름, 특성, 코스트) | 스탯 템플릿 (성급별) + 스킬 참조 |
| 블루아카이브 | 학생 프로필 (이름, 학교, 무기타입, 스킨) | 전투 스탯 (레벨·승급 반영) + 스킬셋 + 장비 보정 |
| 명일방주 | 오퍼레이터 정의 (직군, 레어리티, 배치코스트) | 정예화 단계별 스탯 테이블 + 스킬 레벨 |

### 인사이트

1. **정의는 얇게, 인게임은 두껍게** — 정의는 식별만, 인게임에 전투 필요 데이터를 모두 담음
2. **스탯 템플릿 재사용** — 스탯 구조는 공통, 값만 캐릭터마다 다름
3. **컴포지트 패턴** — 인게임 데이터 = 정의 참조 + 스탯 참조 + 스킬 참조

현재 프로젝트의 `UnitLoadOutData = UnitDefinitionData + UnitStatsByLevelData + SkillDefinitionData` 패턴이 이미 이 방식과 일치한다.
