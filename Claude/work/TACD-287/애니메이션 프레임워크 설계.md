# TACD-287: 애니메이션 프레임워크 설계

## 개요

ActionStateController의 상태 변경에 연동하여 유닛별 셀 애니메이션을 자동 재생하는 프레임워크.
Unity AnimatorController의 상태 전이를 ActionStateController와 연결하고, OverrideController로 유닛별 클립을 교체한다.

---

## 핵심 설계 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 애니메이션 시스템 | Unity Animator 사용 | 셀 애니메이션 드래그 생성 등 기본 기능 활용 |
| 상태 관리 | ActionStateController가 담당 | Animator 상태 머신과 중복 방지 |
| 유닛별 구분 | AnimatorOverrideController | Base Controller 공유, 클립만 교체 |
| 성급별 비주얼 | 성급별 OverrideController | 성급마다 다른 스프라이트 세트 |
| 상태-애니메이션 매핑 | enum 이름 기반 (`animator.Play(state.ToString())`) | 새 상태 추가 시 코드 수정 없음 |

---

## 구성 요소

### 1. Base AnimatorController (에셋, 1개)

OverrideController의 원본 템플릿. 프리팹에 직접 붙지 않고 프로젝트 에셋으로만 존재한다.

- **State 목록**: ActionStateType enum과 1:1 대응 (Idle, Move, Attack, Downed, Freeze, Waiting)
- **각 State에 1프레임짜리 placeholder 클립 할당** (OverrideController의 교체 목록 표시용)
- **Transition 없음** — 코드에서 `animator.Play()`로 직접 전환

### 2. AnimatorOverrideController (에셋, 유닛별·성급별)

Base Controller 기반으로 생성. placeholder 클립을 실제 셀 애니메이션 클립으로 교체한다.

```
예시: 궁수 1성 OverrideController
├── 기반: Base AnimatorController
├── placeholder_idle   → archer_star1_idle.anim
├── placeholder_move   → archer_star1_move.anim
├── placeholder_attack → archer_star1_attack.anim
├── placeholder_downed → archer_star1_downed.anim
├── placeholder_freeze → archer_star1_freeze.anim
└── placeholder_waiting → archer_star1_waiting.anim
```

### 3. UnitDefinitionData 필드 추가

기존 UnitDefinitionData에 성급별 OverrideController 참조를 추가한다.

```csharp
[Header("애니메이션")]
[Tooltip("성급별 AnimatorOverrideController. 인덱스 = 성급 (0은 미사용)")]
[SerializeField] private AnimatorOverrideController[] animatorByStars;
```

### 4. UnitAnimator (런타임 컴포넌트, 신규)

Unit 프리팹에 추가되는 MonoBehaviour. IStateListener<ActionStateType>을 구현하여 상태 변경 시 애니메이션을 전환한다.

**책임:**
- 소환 시 데이터에서 OverrideController를 가져와 Animator에 할당
- ActionStateController 상태 변경 시 `animator.Play(state.ToString())`으로 클립 전환

**의존:**
- Animator 컴포넌트 (같은 GameObject, Controller 필드는 비워둠)
- ActionStateController (같은 GameObject)

## 데이터 흐름

```
┌─ 에셋 구조 ──────────────────────────────────────┐
│                                                   │
│  Base AnimatorController (1개)                    │
│  └── State: Idle, Move, Attack, ... + placeholder │
│            │                                      │
│            ▼                                      │
│  OverrideController (유닛별·성급별)                │
│  └── placeholder → 실제 .anim 클립으로 교체       │
│            │                                      │
│            ▼                                      │
│  UnitDefinitionData (SO)                          │
│  └── animatorByStars[1] = 궁수_1성.overrideController │
│      animatorByStars[2] = 궁수_2성.overrideController │
│      animatorByStars[3] = 궁수_3성.overrideController │
│                                                   │
└───────────────────────────────────────────────────┘

┌─ 런타임 흐름 ────────────────────────────────────┐
│                                                   │
│  유닛 소환                                        │
│  → UnitAnimator.Init(data, star)                  │
│  → animator.runtimeAnimatorController             │
│    = data.Unit.GetAnimatorByStars(star)            │
│  → Idle 상태 자동 재생                             │
│                                                   │
│  상태 변경 (예: Idle → Attack)                     │
│  → ActionStateController 상태 전이                │
│  → UnitAnimator.OnStateEnter(Attack)              │
│  → animator.Play("Attack")                        │
│  → OverrideController의 Attack 클립 재생           │
│                                                   │
└───────────────────────────────────────────────────┘
```

## 새 콘텐츠 추가 시 작업

### 새 유닛 추가

1. 스프라이트 여러 개 선택 → 하이어라키에 드래그 → `.anim` 클립 자동 생성
2. Base Controller 기반으로 OverrideController 생성 → placeholder를 실제 클립으로 교체
3. UnitDefinitionData SO의 `animatorByStars`에 성급별 OverrideController 할당
4. 코드 수정 없음

### 새 ActionState 추가 (예: Stun)

1. `ActionStateType` enum에 `Stun` 추가
2. Base AnimatorController에 `Stun` State 추가 + placeholder 클립
3. 각 유닛 OverrideController에서 Stun placeholder를 실제 클립으로 교체
4. 코드 수정 없음 — `animator.Play("Stun")`이 자동으로 동작

---

## Unit 프리팹 변경사항

- **Animator 컴포넌트 추가** (Controller 필드는 비워둠, 런타임에 할당)
- **UnitAnimator 컴포넌트 추가** (Animator, ActionStateController 참조)
- 기존 `Unit.cs`의 `//TEMP` 스프라이트 할당 코드 제거
