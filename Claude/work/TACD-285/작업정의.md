# TACD-285: 소환술사 테스트 데이터 제작

> 상위 이슈: TACD-273 [전장 - 소환술사] - 데이터 스키마 제작

## 요구사항

TACD-276에서 설계한 소환술사 데이터 스키마(SummonerDefinitionData, SummonerUnitDefinitionData, SummonerLoadOutData)에 테스트용 데이터를 채운다. 이후 프리팹 제작(TACD-277)에서 즉시 활용 가능한 상태로 만든다.

## 배경지식

- 소환술사는 소환수를 소유하는 상위 개념. 각 소환술사는 시너지 1개와 소환수 풀(최대 8체)을 가진다.
- 데이터 구조: SummonerDefinitionData(정체성/시너지/소환수풀) → SummonerLoadOutData(인게임 복합) → SummonerUnitDefinitionData(유닛 타입 판별)
- 기존 테스트 에셋 3개가 TACD-276에서 빈 껍데기로 생성됨
- 기존 소환수 로드아웃: 01아무무, 02알리스타, 03애니, 04징크스, 05아리, 공뇽 (총 6종)
- 기존 시너지: 난동꾼, 비전 마법사, 전쟁기계, 총잡이, 프렐요드

## 완료 정의

- 소환술사 정의 데이터에 이름, 설명, 시너지, 소환수 풀이 설정되어 있다
- 소환술사 유닛 정의에 표시 정보(이름, cost)가 설정되어 있다
- 소환술사 전용 UnitStatsByLevelData 에셋이 존재하고 스탯이 설정되어 있다
- 소환술사 로드아웃에 유닛 정의, 스탯, 소환술사 정의가 연결되어 있다

## 미결사항

1. **소환술사 수**: 테스트 소환술사 1명으로 충분한가, 복수(2~3명)를 만들어야 하는가?
2. **소환수 풀 구성**: 현재 소환수가 6종뿐이다. 8체에 못 미치는데 가용한 만큼만 배정해도 되는가?
3. **시너지 배정**: 기존 시너지(난동꾼, 비전 마법사, 전쟁기계, 총잡이, 프렐요드) 중 어떤 것을 소환술사에 배정할 것인가?
4. **스탯 수치 기준**: 소환술사 스탯을 기존 소환수(공뇽/수호자)와 동일 수준으로 설정할 것인가, 별도 기준이 있는가?
