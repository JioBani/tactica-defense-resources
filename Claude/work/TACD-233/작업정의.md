# TACD-233: 배치 상한에 따라 스캔 시 소환수가 2성/3성으로 등장

## 작업 내용

스캔(리롤) 시 현재 배치 상한 레벨에 따라 소환수가 1성이 아닌 2성 또는 3성으로 등장하는 확률 메커니즘을 구현한다.

## 완료 정의

- 배치 상한 레벨에 따라 스캔 결과에 2성/3성 소환수가 확률적으로 등장한다
- 확률 테이블이 ScriptableObject 데이터로 분리되어 수치 조정이 가능하다
- 확률 테이블의 구조만 제작하고, 실제 확률 수치는 임시값으로 설정한다

## 기획서 참조

- `07 게임 시스템 기획서` > 3.2 소환(구매):
  > "배치 상한 증가에 따라서 소환수가 2성 또는 3성으로 등장하는 확률이 생긴다"

## 기존 코드 분석

### 현재 구조

1. **MarketManager.cs**: 소환터미널 전체 관리. `Level` (RxValue<int>)로 배치 상한 레벨을 추적
2. **MarketUnitRoller.cs**: 스캔 시 유닛 선택 담당. 현재 66번째 줄에 TODO 존재:
   ```csharp
   // TODO: 배치 상한 레벨에 따른 2성/3성 등장 확률 적용
   int star = 1;  // 항상 1성
   ```
3. **PlacementConfig.cs**: 배치 상한 레벨업 테이블 (ScriptableObject). 레벨 1~10, 배치 상한 3~12
4. **MarketDefenderSlot.cs**: `Star` 속성을 이미 지원함

### 변경 대상

- `MarketUnitRoller.cs` - 성급 확률 로직 추가
- `PlacementConfig.cs` 또는 새 ScriptableObject - 레벨별 성급 확률 테이블 데이터
- `MarketManager.cs` - Roller에 현재 레벨 전달
