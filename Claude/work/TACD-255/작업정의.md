# TACD-255: 소환수가 생성되면 소속 소환술사의 효과를 자동으로 보유한다

> 상위 이슈: TACD-254 [전장 - 시너지] - 소환술사 효과 부여

## 요구사항

소환수가 생성될 때 소속 소환술사의 시너지 효과를 자동으로 보유한다.
런타임 시너지 데이터는 Defender 클래스가 직접 관리한다.

## 배경지식

- 각 소환술사는 `SummonerDefinitionData.summonerEffect`로 고유 시너지를 가짐
- 각 소환술사의 `summonPool`에 8체의 소환수(UnitLoadOutData)가 포함
- 소환수(Defender)가 마켓에서 스폰될 때 소속 소환술사를 알아야 효과를 부여 가능
- 시너지 시스템은 `Defender.HasSynergy()`로 시너지 보유 여부를 조회한다

## 완료 정의

- [x] 소환수가 생성되면 소속 소환술사의 효과를 자동으로 보유한다

## 구현 결과

- SynergyManager가 `Start()`에서 SummonerManager의 편성 데이터를 읽어 역방향 맵(`UnitLoadOutData → SynergyDefinitionData`)을 자체 구축
- Defender 스폰 이벤트(`OnDefenderChangedEventDto`) 구독 → 역방향 맵에서 시너지를 조회하여 `Defender.AddSynergy()` 호출
- `HasSynergy`를 Defender로 이동하여 시너지 보유 판단의 책임을 Defender가 담당
- null 데이터 시 `InvalidOperationException` throw로 설정 오류 조기 감지
