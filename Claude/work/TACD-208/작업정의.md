# TACD-208: 3성 소환수에 동일 종류 2성 소환수를 추가하여 강화할 수 있다

## 작업 내용

3성 소환수에 동일 종류 2성 소환수 1개를 투입하면 **강화 합성**이 발동된다.

- 3성 + 동일 종류 2성 → 3성+1강
- 3성+1강 + 동일 종류 2성 → 3성+2강
- 반복 가능 (N강 + 2성 → N+1강)
- 강화 합성은 **정비 페이즈에서만** 발생한다
- 강화 합성도 자동 합성이며, 연쇄 처리된다

## 완료 정의

- [ ] 3성 + 동일 종류 2성 투입 시 강화 합성이 발동된다
- [ ] 강화 단계(Reinforcement)가 올바르게 증가한다
- [ ] 반복 강화가 정상 동작한다 (1강→2강→3강...)
- [ ] 정비 페이즈에서만 강화 합성이 동작한다
- [ ] 기존 기본 합성(1성×3→2성, 2성×3→3성)과 충돌 없이 공존한다

- 스탯 반영: `additionalPerExtraStar` 필드가 이미 초과 성급 스탯 증가를 지원

## 기존 코드 참조

- `UnitStatSheet`: Star 속성 보유, UpgradeStar()로 성급 증가, Init()에서 스탯 로드
- `StarStatRecord.GetValue(star)`: star > preset count 시 additionalPerExtraStar로 자동 스케일
- `FusionGroupDetector`: 동일 (UnitDefinitionId, Star) 3개 그룹 탐지
- `DefenderFusionManager`: 기본 합성 오케스트레이션, 정비 페이즈 체크, 연쇄 합성
- `FusionCandidate`: (UnitDefinitionId, Star, Index) 구조체

## 설계 방향

### Star vs Reinforcement 분리
- `Star`: 기본 성급 (1, 2, 3). 기본 합성으로만 증가.
- `Reinforcement`: 강화 단계 (0, 1, 2, ...). 강화 합성으로만 증가.
- 스탯 계산 시 effective star = `Star + Reinforcement`로 `GetValue()` 호출
- `StarStatRecord.additionalPerExtraStar`가 강화 시 스탯 증분으로 자연스럽게 동작

### 기본 합성 3성 제한
- 기본 합성은 Star < 3인 그룹에서만 발동 (1성×3→2성, 2성×3→3성)
- 3성 이상에서는 기본 합성 대신 강화 합성이 담당

### 강화 합성 탐지
- EnhancementFusionDetector: 순수 C# 서비스 클래스
- 탐지 규칙: 동일 UnitDefinitionId, 타겟 Star ≥ 3, 재료 Star = 2
