# TACD-277: 소환술사 프리팹 제작

> 상위 이슈: TACD-273 [전장 - 소환술사] - 데이터 스키마 제작

## 요구사항

소환술사 전장 프리팹을 제작한다. 유닛 공통 컴포넌트(Attacker, Victim, HealthBar, ActionStateController 등)를 구성하여, 소환술사가 전장에서 유닛으로 동작 가능한 상태를 만든다.

## 배경지식

### 소환술사와 Defender의 차이

| 구분 | Defender | 소환술사 |
|------|----------|---------|
| 배치 방식 | 그리드에 드래그 배치 | 라인 끝에 편성 순서대로 고정 |
| 전용 컴포넌트 | `Defender` (드래그/배치 이벤트) | 없음 (신규 필요) |
| 스킬 | `SkillCaster` 보유 | 미정 |

### 현재 프리팹 상태

`Summoner Variant.prefab`이 이미 Unit.prefab의 Variant로 존재한다. 기본 `Unit` 컴포넌트만 붙어 있고, Defender처럼 전용 컴포넌트가 없다.

**현재 컴포넌트 구성 (루트):**
- Transform, SpriteRenderer, CircleCollider2D, EventTrigger
- Draggable2D, Poolable, Unit, ActionStateController
- Selectable2D, Animator, UnitAnimator

**자식 오브젝트:** Attacker, Victim, HealthBar, StatusEffectController (Defender와 동일)

### 소환술사 데이터 에셋 (TACD-276, TACD-285에서 완료)

- `SummonerDefinitionData` — 소환술사 정체성 (이름, 시너지, 소환수 풀)
- `SummonerLoadOutData` — `UnitLoadOutData`를 상속, 소환술사 정의 참조 추가
- `SummonerUnitDefinitionData` — `UnitDefinitionData`를 상속, `is` 키워드로 판별용
- 테스트 데이터 4종 생성 완료 (빌지워터/녹서스/슈리마/데마시아)

## 완료 정의

- 소환술사 프리팹이 전장에서 유닛으로 동작 가능한 상태
- 소환술사 전용 컴포넌트(`Summoner`)가 존재하며, `Defender`처럼 `Unit`을 상속
- 소환술사는 고정 배치이므로 드래그 관련 기능이 불필요

## 기존 코드 참조

- `Unit.cs` — 기본 유닛 MonoBehaviour. HealthBar, ActionStateController, StatSheet 관리
- `Defender.cs` — Unit 상속. 드래그 이벤트, 배치(Placement) 관리, IStateListener 구현
- `SummonerLoadOutData` — UnitLoadOutData 상속. 소환술사 정의 참조

## 결정 사항

1. **Summoner 컴포넌트**: `Defender`처럼 `Unit`을 상속하는 `Summoner` MonoBehaviour를 신규 생성한다
2. **Draggable2D 제거**: 소환술사는 고정 배치이므로 프리팹에서 Draggable2D를 제거한다
3. **SkillCaster 포함**: 소환술사도 스킬 사용을 허용한다. 단, 스킬 데이터가 없는 소환술사도 있으므로 `SkillCaster`에서 스킬 미설정 시 안전하게 비활성 처리한다
4. **프리팹 이름**: "Summoner Variant" → "Summoner"로 변경한다
