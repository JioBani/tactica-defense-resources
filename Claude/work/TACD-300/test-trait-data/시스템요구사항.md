# TACD-300 — 테스트용 소환수 특성 제작

> 이 문서는 사용자 요구사항을 시스템 컴포넌트 단위 책임으로 번역한 문서입니다.

## 개요

- Jira: TACD-300 (상위: TACD-250)
- 소환수 특성 부여 기능(TACD-250) 개발을 지원하기 위해, SummonTrait 타입의 SynergyDefinitionData 자산 8개와 이를 식별하기 위한 SynergyId enum 값 8개를 추가한다.

| 컴포넌트 | 역할 |
|----------|------|
| SynergyDefinitionData (ScriptableObject 자산) | 소환수 특성 하나의 정의 — 표시 정보·상태 효과 참조·임계치 목록 |
| SynergyId (enum) | 특성에 고유 식별 값 부여 |
| SynergyTier (중첩 데이터) | 임계치별 활성화 조건(requiredCount)과 효과 상수(constants) 정의 |

## 현재 동작

- `Assets/Common/Data/Synergies/`에 SummonTrait 타입의 SynergyDefinitionData 자산이 존재하지 않는다.
- SynergyId enum에 SummonTrait 전용 식별 값이 없다(현재 값: None=0, Bruiser=1, Arcanist=2, Freljord=3, Warmonger=4, Gunslinger=5).
- 소환수 특성 분배 런타임 코드(TACD-250)가 구현 중으로, SummonTrait 자산의 로드·분배 처리가 아직 완성되지 않은 상태다.

## 시스템 요구사항

### 완료 정의 나열

- **DoD-S1.** `Assets/Common/Data/Synergies/`에 SynergyDefinitionData 자산 8개가 존재한다.
- **DoD-S2.** 각 자산의 synergyType이 SummonTrait(=1)로 설정되어 있다.
- **DoD-S3.** SynergyId enum에 8개 테스트 특성에 대응하는 고유 값 8개가 추가되어 있다.
- **DoD-S4.** 각 자산의 id 필드가 DoD-S3에서 추가된 고유 SynergyId 값으로 설정되어 있다.
- **DoD-S5.** 각 자산의 displayName이 다른 자산과 중복되지 않는 값으로 설정되어 있다.
- **DoD-S6.** 각 자산의 tiers 목록에 requiredCount가 각각 2, 3, 4인 SynergyTier 항목이 3개 존재한다.
- **DoD-S7.** 각 SynergyTier의 constants 딕셔너리에 하나 이상의 키-값 쌍이 설정되어 있다.
- **DoD-S8.** 각 SynergyDefinitionData 자산에 대응하는 StatusEffectDefinitionData 자산 8개가 존재하며, 각 자산의 statusEffectDefinition 필드가 해당 StatusEffectDefinitionData를 참조하고 있다. 단, 각 StatusEffectDefinitionData 자산은 구조만 존재하고 효과 내용(수치·로직)은 비어 있는 상태다.
- **DoD-S9.** 각 자산의 id(SynergyId)와 displayName이 아래 확정 목록의 값으로 설정되어 있다: 강인함/TestFortitude, 맹습/TestStrike, 주술/TestArcane, 신속/TestSwift, 불굴/TestEndurance, 집중/TestFocus, 파쇄/TestBreach, 맹독/TestVenom.

### 완료 정의를 기능별로 그룹화

**그룹 A. 소환수 특성 자산 생성 및 식별**
- 포함 DoD: DoD-S1, DoD-S2, DoD-S3, DoD-S4, DoD-S5, DoD-S9
- 대응 사용자 요구사항 그룹: 그룹 A (데이터 존재 및 인식)
- 설명: SummonTrait 타입 SynergyDefinitionData 자산 8개가 확정된 고유 SynergyId 값·displayName과 함께 프로젝트에 존재하는 상태. SynergyId enum 확장 포함. DoD-S9는 UR OQ-3 합의 후 확정.

**그룹 B. 임계치별 효과 데이터 설정**
- 포함 DoD: DoD-S6, DoD-S7, DoD-S8
- 대응 사용자 요구사항 그룹: 그룹 C (임계치 단계별 효과 테스트 커버리지)
- 설명: 각 자산이 requiredCount 2·3·4에 대응하는 SynergyTier 3개와 상수 데이터를 보유하고, 효과 내용이 비어 있는 StatusEffectDefinitionData(빈 껍데기) 자산이 연결된 상태.
