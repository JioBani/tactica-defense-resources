# TACD-300 — 테스트용 소환수 특성 제작

> 이 문서는 사용자 시점의 요구사항을 도메인 언어로 정의한 문서입니다.

## 배경

- Jira: TACD-300 (상위: TACD-250 — 소환수 특성 부여)
- 소환수 특성 부여 기능(TACD-250) 개발 중, 실제 특성 콘텐츠 데이터가 존재하지 않으면 기능을 구동하거나 검증할 수 없다. 테스트용 소환수 특성 8개를 제작하여 특성 분배 로직·역할군 보정·임계치 효과 발동이 올바르게 동작하는지 확인할 수 있는 기반 데이터를 마련한다.

## 현재 동작

- 소환수 특성 데이터가 게임에 존재하지 않는다.
- 특성 부여 기능(TACD-250)이 분배할 특성을 인식할 수 없어, 기능 자체를 실행·검증할 수 없는 상태다.

## 사용자 요구사항

### 완료 정의 나열

- **DoD-U1.** 테스트용 소환수 특성 8개가 데이터로 존재하며, 특성 부여 기능이 이 8개를 정상적으로 인식하여 소환수 풀(32체)에 분배를 실행할 수 있다.
- ~~**DoD-U2.**~~ *(OQ-1 결정으로 이번 범위에서 제외 — 역할군 우선/범용 구분 없이 8개 모두 역할군 필드 미설정 상태로 제작)*
- **DoD-U3.** 각 테스트 특성은 시너지 임계치 2단계·3단계·4단계에 각각 대응하는 효과가 설정되어 있어, 배치된 특성 보유 소환수 수에 따라 단계별 효과 발동 여부를 확인할 수 있다.
- **DoD-U4.** 각 테스트 특성은 서로 구별 가능한 명칭을 가지며, 어떤 특성이 어느 소환수에 부여됐는지 식별할 수 있다.
- ~~**DoD-U5.**~~ *(OQ-1 결정으로 이번 범위에서 제외 — 역할군 우선 할당은 추후 별도 작업)*

### 완료 정의를 기능별로 그룹화

**그룹 A. 데이터 존재 및 인식**
- 포함 DoD: DoD-U1, DoD-U4
- 설명: 테스트 특성이 실제로 존재하고, 특성 부여 기능이 이를 로드·인식할 수 있는 상태. 이 그룹이 충족되지 않으면 나머지 검증 자체가 불가능하다.

~~**그룹 B. 역할군 보정 로직 테스트 커버리지**~~ *(OQ-1 결정으로 이번 범위에서 전체 제외 — DoD-U2, DoD-U5 모두 해당 없음)*

**그룹 C. 임계치 단계별 효과 테스트 커버리지**
- 포함 DoD: DoD-U3
- 설명: 각 특성이 3단계 임계치 효과를 모두 보유하여, 배치 수 변화에 따른 단계별 효과 발동·비발동을 검증 가능.
