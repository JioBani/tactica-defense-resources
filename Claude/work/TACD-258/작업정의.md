# TACD-258: 임계치 충족 시 시너지 보유 소환수에게 효과 적용, 미달 시 해제

> 상위 이슈: TACD-251 [전장 - 시너지] 시너지 계산 및 효과 적용

## 요구사항

- 시너지 임계치가 충족되면, 해당 시너지를 보유한 소환수에게 효과를 적용한다.
- 임계치 미달 시 적용된 효과를 제거한다.
- 티어가 변경(상향/하향)되면 이전 티어 효과를 제거하고 새 티어 효과를 적용한다.
- 시너지 효과는 스탯 변경에 한정되지 않는다. 스킬 횟수 변경, 행동 변경 등 다양한 효과가 가능하다.
- 시너지별로 고유한 효과 로직이 존재하며, `SynergyTier.constants`는 범용 상수 저장소로서 각 효과 구현이 자유롭게 해석한다.

## 배경지식

- **시너지 상수(constants)**: SynergyTier의 `constants` 딕셔너리에 `string → float` 형태로 상수 값이 저장되어 있다. 상수명이 스탯에 기계적으로 매핑되는 것이 아니라, 각 시너지 효과 구현이 필요한 상수를 이름으로 조회하여 자유롭게 사용한다.
- **수정자 시스템**: `StatModifier(source, type, value)`로 생성하여 `UnitStat.AddModifier()`로 적용한다. `RemoveModifiersBySource(source)`로 특정 출처의 수정자를 일괄 제거할 수 있다. 스탯 변경 효과를 구현하는 시너지에서 활용한다.
- **시너지 그룹핑**: SynergyManager가 전장 디펜더를 SummonerEffect 기준으로 그룹핑하고 있다. 효과 적용 시 이 그룹의 모든 디펜더에게 효과를 적용/제거해야 한다.

## 완료 정의

- 시너지별 효과 구현 인터페이스가 존재하고, 티어 활성화/비활성화/변경 시 적절히 호출된다.
- 임계치 충족 시 시너지 보유 소환수에게 효과가 적용된다.
- 임계치 미달 시 적용된 효과가 정확히 해제된다.
- 티어 변경(상향/하향) 시 이전 티어 효과가 해제되고 새 티어 효과가 적용된다.
- 동일 시너지의 효과만 해제되고, 다른 출처의 효과는 영향받지 않는다.

## 미결사항

모두 해소됨.

- ~~시너지 종류 구분~~ → SynergyManager/SynergyState는 시너지 종류(SummonerEffect/SummonTrait)를 구분하지 않는다. 디펜더가 보유한 모든 시너지에 대해 동일하게 동작한다. 현재 `GroupBySynergy()`가 SummonerEffect만 참조하는 부분을 시너지 종류에 무관하게 수정한다.
- ~~테스트 데이터 상수~~ → 기존 데이터에 constants 값이 이미 설정되어 있음 (화염 시너지: damageBonus, 테스트 시너지: damage/attackSpeed/range).
