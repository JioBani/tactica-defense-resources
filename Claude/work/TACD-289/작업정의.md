# TACD-289: UnitDefinitionData에 성급별 AnimatorOverrideController 필드 추가

> 상위 이슈: TACD-287 [전장 - 유닛] 애니메이션 프레임워크 제작

## 요구사항

UnitDefinitionData에 성급별 AnimatorOverrideController를 지정할 수 있는 필드를 추가한다.
각 유닛의 SO에서 1성/2성/3성별로 다른 OverrideController를 할당할 수 있어야 한다.

## 배경지식

- 설계 문서: `Claude/work/TACD-287/애니메이션 프레임워크 설계.md`
- UnitDefinitionData는 유닛의 정체성 데이터 SO (아이콘, 일러스트, 시너지 등)
- AnimatorOverrideController는 Base Controller(UnitBase.controller)의 클립을 유닛별로 교체하는 에셋
- 소환 시 UnitAnimator가 이 필드에서 해당 성급의 OverrideController를 가져와 Animator에 할당

## 완료 정의

- UnitDefinitionData에 성급별 AnimatorOverrideController 배열 필드가 존재한다
- 해당 성급의 OverrideController를 반환하는 접근자 메서드가 존재한다

## 기존 코드 참조

- `Assets/Common/Data/Units/UnitDefinitions/UnitDefinitionData.cs` — 필드 추가 대상

## 작업 체크리스트

- [x] UnitDefinitionData에 성급별 AnimatorOverrideController 필드 및 접근자 추가
