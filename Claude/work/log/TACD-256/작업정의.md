# TACD-256: [시너지 > 계산] - 전장 배치 소환수 기반 시너지 카운트 집계

> 상위 이슈: TACD-251 [전장 - 시너지] - 시너지 계산 및 효과 적용

## 요구사항

전장(BattleArea)에 배치된 소환수들이 보유한 시너지(소환술사 효과, 소환수 특성)별로 유닛 수를 집계한다.

- 동일 소환수(같은 종류)가 여러 체 배치되어 있어도 1회만 카운트한다
- 소환술사 효과와 소환수 특성을 각각 별도로 집계한다
- 집계 결과는 "시너지별 현재 카운트" 형태로 외부에서 조회할 수 있어야 한다

## 배경지식

- **시너지**: 소환수가 가진 속성(소환술사 효과 / 소환수 특성)으로, 같은 시너지를 보유한 유닛이 일정 수 이상 배치되면 효과가 발동된다
- **소환술사 효과(SummonerEffect)**: 유닛 정의(UnitDefinitionData)에 고정 할당. 소속 소환술사에 의해 결정됨
- **소환수 특성(SummonTrait)**: 전장마다 랜덤 부여됨 (TACD-250에서 구현 예정)
- **카운트 기준**: "종류" 단위. 같은 UnitDefinitionData를 가진 유닛들은 하나로 카운트
- 현재 UnitDefinitionData에는 `SummonerEffect` 필드만 존재. SummonTrait는 TACD-250에서 런타임 할당 필드로 추가될 예정

## 완료 정의

- 전장에 배치된 소환수들의 시너지별 카운트가 정확히 집계된다
- 동일 종류 소환수를 여러 체 배치해도 해당 시너지의 카운트가 1만 증가한다
- 소환술사 효과와 소환수 특성이 각각 별도로 카운트된다
- 외부(다른 시스템)에서 시너지별 현재 카운트를 조회할 수 있다
