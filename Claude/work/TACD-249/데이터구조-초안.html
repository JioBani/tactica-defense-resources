TACD-249 시너지 데이터 구조 초안

소환술사 효과 · 소환수 특성 · 임계치별 효과를 정의하는 기반 데이터

ERD (Entity-Relationship Diagram)

SynergyDefinitionData
ScriptableObject
id PK int
displayName string
description string
icon Sprite
synergyType SynergyType
tiers SynergyTier[]
SynergyTier
Serializable struct
requiredCount int
statEffects SynergyStatEffect[]
SynergyStatEffect
Serializable struct
statKind UnitStatKind
modifierType StatModifierType
value float
SynergyType
enum
SummonerEffect 임계치 2/4/6/8
CreatureTrait 임계치 2/3/4
UnitDefinitionData
기존 ScriptableObject
id PK int
displayName string
description string
organization string
cost int
icon Sprite
illustration Sprite
summonerEffect FK NEW SynergyDefinitionData
관계
SynergyDefinitionData 1 ──── * SynergyTier (tiers 배열로 포함)
SynergyTier 1 ──── * SynergyStatEffect (statEffects 배열로 포함)
SynergyDefinitionData 1 ──── * UnitDefinitionData (summonerEffect로 참조)
SynergyStatEffect ──ref── UnitStatKind (기존 enum 참조)
SynergyStatEffect ──ref── StatModifierType (기존 enum 참조)

설계 결정 사항

결정근거
SynergyTier.requiredCount를 데이터로 정의 소환술사 효과(2/4/6/8)와 소환수 특성(2/3/4)의 임계치가 다름. 하드코딩 대신 데이터로 유연하게 설정
SynergyStatEffect를 별도 직렬화 구조체로 분리 기존 StatModifier는 런타임 객체(Source 포함). 에디터에서 설정할 데이터용 구조체가 별도로 필요
소환수 특성은 UnitDefinitionData에 미포함 전장마다 랜덤 부여되므로 정적 데이터가 아닌 런타임 매핑으로 처리
상위 단계가 하위 단계를 포함하지 않고 독립 정의 각 단계의 효과가 완전히 교체되는 설계. 런타임에서 "달성한 최고 단계"만 적용하면 됨
행동/메카닉 변경 효과는 현재 미포함 스탯 수정자 기반만 먼저 구현. SynergyTier에 추가 필드로 확장 가능
열린 질문: 기존 organization 필드(string)를 summonerEffect로 대체할 것인지, 별도 필드로 추가할 것인지?
