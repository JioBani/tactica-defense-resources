# 시너지 데이터 필요 분석

> 동작 트리(TACD-110 댓글) + 기획서 + 일반적 게임 패턴을 기반으로 정리

---

## 1. 구현에서 사용해야 할 데이터

> 코드가 시너지를 계산·적용·해제하기 위해 필요한 데이터

### TACD-249: 시너지 데이터 구조
| 데이터 | 용도 | 현재 설계 포함 여부 |
|--------|------|:---:|
| 시너지 고유 ID | 팩토리 매핑, 시너지 구분 | O |
| 시너지 종류 (소환술사 효과 / 소환수 특성) | 임계치 체계 분기, 부여 방식 분기 | O |
| 티어별 임계치 (requiredCount) | 카운트 → 활성 티어 판정 | O |
| 티어별 상수 (constants: name + value) | 구현 클래스가 읽어서 효과 적용 | O |

### TACD-250: 소환수 특성 부여
| 데이터 | 용도 | 현재 설계 포함 여부 |
|--------|------|:---:|
| 소환수의 역할군 (탱커, 딜러 등) | 특성 부여 확률 보정 | **X** (UnitDefinitionData에 없음) |
| 특성별 우선 역할군 | 어떤 역할군에 높은 확률로 부여할지 | **X** |
| 특성이 범용인지 역할군 우선인지 | 부여 알고리즘 분기 | **X** |

### TACD-251: 시너지 계산 및 효과 적용
| 데이터 | 용도 | 현재 설계 포함 여부 |
|--------|------|:---:|
| 유닛 → 소환술사 효과 매핑 | 어떤 유닛이 어떤 소환술사 효과를 가지는지 | O (UnitDefinitionData.summonerEffect) |
| 유닛 → 소환수 특성 매핑 | 전장별 랜덤 부여 결과 | 런타임 (설계 범위 외, TACD-250) |

---

## 2. 유저에게 표현되어야 할 데이터

> UI에서 플레이어에게 보여주기 위해 필요한 데이터

### 시너지 목록/진행도 UI (TACD-252)
| 데이터 | 용도 | 현재 설계 포함 여부 |
|--------|------|:---:|
| 시너지 이름 (displayName) | 시너지 명칭 표시 | O |
| 시너지 아이콘 (icon) | 시너지 목록에서 시각적 식별 | O |
| 시너지 설명 (description + 플레이스홀더) | 현재 티어의 효과를 수치와 함께 표시 | O |
| 현재 카운트 / 임계치 진행도 | "2/4" 같은 진행 상태 | 런타임 계산 (데이터 구조 불필요) |
| 활성 여부 / 현재 달성 티어 | 어떤 단계까지 활성화됐는지 | 런타임 계산 |
| 전체 티어 목록 (requiredCount 리스트) | "2 / 4 / 6 / 8" 단계 표시 | O (tiers 배열에서 추출) |

### 소환수별 정보 UI
| 데이터 | 용도 | 현재 설계 포함 여부 |
|--------|------|:---:|
| 소환수가 보유한 소환술사 효과 | 유닛 상세 UI에서 표시 | O (UnitDefinitionData.summonerEffect) |
| 소환수에 부여된 소환수 특성 | 유닛 상세 UI에서 표시 | 런타임 매핑 |

## 3. 일반적인 게임에서 추가로 필요할 수 있는 데이터

> 기획서/댓글에 명시되지 않았지만, TFT 등 유사 게임에서 흔히 사용되는 데이터

| 데이터 | 용도 | 참고 |
|--------|------|------|
| **티어별 시각 등급** (Bronze/Silver/Gold 등) | UI에서 티어 색상/테두리 구분. TFT는 `style_name: kBronze/kSilver/kGold/kChromatic` 사용 | TFT 필수 요소 |
| **소환수 역할군 (Role)** | 특성 부여 확률 보정 + UI에서 역할 표시. 현재 `organization` 필드가 string으로 비어있음 | TACD-250에서 반드시 필요 |
| **innate constants (기본 상수)** | 티어와 무관하게 항상 적용되는 기본 수치. TFT는 `innate_trait_sets`로 분리 | TFT에서 사용 |
| **시너지 정렬 순서 (sortOrder)** | UI에서 시너지 목록을 일관된 순서로 표시 | 시너지 수가 늘면 필요 |
| **시너지 활성 최소 표시 수** | 최소 N체가 있어야 UI에 시너지 항목이 나타남 (0체인 시너지 숨기기) | 보통 1 고정이지만 설정 가능하면 유연 |
| **티어별 설명 오버라이드** | 티어마다 설명이 완전히 달라지는 경우 (플레이스홀더 수치 교체로 부족할 때) | 복잡한 시너지에서 필요할 수 있음 |
| **소환수 특성 부여 가중치** | 특성별로 부여 확률 자체가 다를 수 있음 (희귀 특성 등) | 밸런스 조절용 |

## 현재 설계(v2) 누락 사항 요약

| 누락 항목 | 영향 받는 Task | 필수/선택 |
|-----------|:---:|:---:|
| 역할군(Role) 데이터 | TACD-250 | **필수** (부여 확률 보정에 필요) |
| 특성별 우선 역할군 / 범용 여부 | TACD-250 | **필수** |
| 티어별 시각 등급 (style) | TACD-252 | 선택 (없으면 UI에서 하드코딩) |
| innate constants (기본 상수) | TACD-251 | 선택 (현재 기획에 해당 케이스 없음) |
| 시너지 정렬 순서 | TACD-252 | 선택 |
