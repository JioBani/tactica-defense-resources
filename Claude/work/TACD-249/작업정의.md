# TACD-249: 시너지 데이터 구조

## 작업 내용

소환술사 효과, 소환수 특성, 임계치·단계별 효과 설정 등 시너지 시스템 전체가 의존하는 기반 데이터 구조를 설계·제작한다.

## 완료 정의

- 시너지 종류(소환술사 효과 / 소환수 특성)를 구분하는 enum이 존재한다
- 개별 시너지를 정의하는 ScriptableObject(SynergyDefinitionData)가 존재한다
  - 시너지 이름, 설명, 아이콘
  - 시너지 종류(소환술사 효과 / 소환수 특성)
  - 임계치별 효과 목록 (소환술사 효과: 2/4/6/8, 소환수 특성: 2/3/4)
- 임계치별 효과(SynergyTier)는 스탯 수정자 목록을 포함한다
  - 어떤 스탯에, 어떤 타입(Flat/Percent)으로, 얼마만큼 변경하는지
- UnitDefinitionData에 소환술사 효과와 소환수 특성 필드가 추가된다
  - 소환술사 효과: 에디터에서 고정 지정 (SynergyDefinitionData 참조)
  - 소환수 특성: 런타임에 랜덤 부여되므로 데이터 구조에는 포함하지 않음 (별도 런타임 매핑)
- 유닛 테스트로 데이터 구조의 정합성을 검증한다

## 관련 기획서

- `Document/기획서/시너지 기획 메모.md`
  - 소환술사 효과 임계치: 2 / 4 / 6 / 8 (4단계)
  - 소환수 특성 임계치: 2 / 3 / 4 (3단계)
  - 시너지 효과는 해당 시너지를 가진 소환수에게만 적용
  - 효과는 스탯 변경에 한정되지 않음 (행동/메카닉 변경도 가능)
