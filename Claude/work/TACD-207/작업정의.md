# TACD-207: 동일 소환수 3개를 모으면 자동으로 상위 성급으로 합성된다

## 작업 내용

동일 소환수(같은 UnitDefinitionData + 같은 성급) 3개가 존재하면 자동으로 상위 성급 소환수로 합성되는 시스템을 구현한다.

### 핵심 요구사항

1. **합성 조건**: 동일 소환수(같은 종류 + 같은 성급) 3개가 대기석 또는 전장에 존재
2. **합성 시점**: 정비 페이즈에서만 발생 (전투 중 불가)
3. **합성 실행 방식**: 가장 왼쪽 위에 있는 소환수가 승급하고, 나머지 재료 2개는 제거된다
4. **연쇄 합성**: 합성 결과로 다시 3개가 충족되면 자동으로 연쇄 합성
5. **전투 중 불가 안내**: 전투 중 합성 조건 충족 시 버블 메시지로 "전투 중에는 합성이 불가능합니다" 표시

### 참고 기획서

- `Document/기획서/07 게임 시스템 기획서.md` > 2.2 소환수 합성(승급) 규칙
- 합성 가능 표시는 하지 않는다
- 합성 시 기존 버프 상태 유지 (전투 중 합성 불가이므로 HP/상태 해당 없음)

## 현재 구조 분석

### 소환수(Defender) 데이터 모델

```
Defender (MonoBehaviour, 씬 인스턴스)
├── Unit.UnitLoadOutData (ScriptableObject 참조)
│   ├── UnitDefinitionData (ID, DisplayName, Cost, Icon 등)
│   ├── SkillDefinitionData (스킬)
│   └── UnitStatsByLevelData (성급별 스탯 - StarStatRecord)
├── Unit.StatSheet (UnitStatSheet - 런타임 스탯)
├── Placement (WaitingArea / BattleArea)
└── Draggable2D (드래그&드롭)
```

### 핵심 문제: 성급(Star) 속성 부재

현재 개별 Defender 인스턴스에 **성급(star)** 속성이 없다.
- `UnitStatSheet.Init(data, star = 1)` — star 파라미터는 있지만 항상 기본값 1로 호출됨
- `UnitStatsByLevelData`는 성급별 스탯을 정의하고 있어 데이터 계층은 준비됨
- Defender/Unit 클래스에 star 필드를 추가해야 함

## 완료 정의

1. **Unit 클래스에 Star 속성이 추가**되고, 스탯 초기화에 반영된다
2. **동일 소환수 3개 보유 시 자동 합성**이 정비 페이즈에서 발동된다
3. **합성 시 가장 왼쪽 위 소환수가 승급**하고, 나머지 재료 2개가 제거된다
4. **연쇄 합성**이 정상 동작한다
5. **전투 페이즈에서는 합성이 발생하지 않는다**
6. **전투 중 합성 조건 충족 시** 버블 메시지로 불가 안내가 표시된다
7. 유닛 테스트가 핵심 로직을 검증한다
