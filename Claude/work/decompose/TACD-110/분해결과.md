# 구현 이슈 분해: TACD-110 - [전장] 시너지

> 현재 작업: Task Jira 등록 완료

## 1단계: 기획서 파악 ✅

**참조 기획서**: `Document/기획서/시너지 기획 메모.md`, `Document/기획서/07 게임 시스템 기획서.md` (5장 시너지)

**핵심 요약**: 시너지는 **소환술사 효과**(고정)와 **소환수 특성**(전장마다 랜덤 부여) 두 종류로 구성. 전장 배치된 서로 다른 종류의 소환수를 카운트하여 임계치(소환술사 효과 2/4/6/8, 소환수 특성 2/3/4) 충족 시 해당 시너지를 보유한 소환수에게 효과 적용. 배치/제거 시 재계산, 다운 시 카운트 유지.

## 2단계: 에픽 경계 확인 ✅

**대상 에픽**: TACD-110 - [전장] 시너지

**인접 에픽**:
- TACD-80 [전장] 유닛: 수정자(Modifier) 시스템 소속. 시너지는 수정자를 사용만 함
- TACD-81 [전장] 소환수: 배치/제거 이벤트 발생 소속. 시너지는 이벤트를 구독
- TACD-237 [전장] 소환술사: 소환술사 육성에 따른 시너지 강화 수치는 소환술사/스케일링 영역

**경계 판단**: 시너지 에픽은 시너지 고유 로직(카운트, 임계치, 효과 적용, 특성 부여, 정보 UI)만 포함. 수정자 시스템, 배치/제거 이벤트, 소환술사 육성은 각 인접 에픽 소속.

## 3단계: Task 도출 ✅

### 상위 동작 트리

```
시너지
├─ 시너지 데이터 구조 (기반)
│   ├─ 💻 소환술사 효과 정의 데이터 구조를 설계한다
│   ├─ 💻 소환수 특성 정의 데이터 구조를 설계한다
│   └─ 💻 임계치·단계별 효과 설정 구조를 설계한다
├─ 소환술사 효과 부여
│   └─ 💻 소환수는 소속 소환술사의 고유 효과를 자동으로 보유한다
├─ 소환수 특성 부여 (전장 시작 시)
│   ├─ 💻 소환수 풀(32체)에 8개 특성을 랜덤 분배한다
│   ├─ 💻 역할군에 따라 부여 확률을 보정한다
│   └─ ❓ 부여 확률의 구체적 수치 (스케일링 디자인 영역)
├─ 시너지 계산 (배치 변경 시)
│   ├─ 💻 전장 배치된 서로 다른 종류의 소환수를 카운트한다
│   ├─ 💻 동일 소환수는 중복 카운트하지 않는다
│   ├─ 💻 소환술사 효과 임계치(2/4/6/8)를 판정한다
│   ├─ 💻 소환수 특성 임계치(2/3/4)를 판정한다
│   └─ 💻 다운 시에는 재계산하지 않는다 (카운트 유지)
├─ 시너지 효과 적용/해제
│   ├─ 💻 임계치 충족 시 해당 시너지 보유 소환수에게 효과를 적용한다
│   ├─ 💻 임계치 해제 시 효과를 제거한다
│   ├─ 🧩 수정자 시스템으로 스탯을 변경한다 (유닛 에픽)
│   └─ ❓ 스탯 외 행동/메카닉 변경 효과의 적용 프레임워크
├─ 시너지 갱신 트리거
│   ├─ 💻 소환수 배치 시 시너지를 재계산한다
│   └─ 💻 소환수 제거 시 시너지를 재계산한다
└─ 시너지 정보 UI
    ├─ 💻 활성 시너지 목록과 진행도를 표시한다
    ├─ 💻 소환수별 특성 배치 정보를 표시한다
    └─ ❓ UI의 구체적 레이아웃/디자인 (기획서에 와이어프레임 없음)
```
