# TACD-250: [전장 - 시너지] - 소환수 특성 부여

> 상태: 완료

---

## 5단계: Sub-task 도출 ✅

### 5-1. 동작 트리 상세화

```
소환수 특성 부여
├─ 전장 시작 시 특성 분배를 실행한다
│   ├─ 💻 소환수 풀(32체)을 입력으로 받는다
│   ├─ 💻 8개 소환수 특성(SynergyDefinitionData 중 SummonTrait 타입)을 로드한다
│   └─ 💻 각 소환수에게 특성 1개를 배정한다
├─ 역할군에 따라 부여 확률을 보정한다
│   ├─ 💻 역할군 우선 특성: 해당 역할군 소환수에게 부여 확률이 높다
│   ├─ 💻 범용 특성: 역할군과 무관하게 균등 분배된다
│   └─ ❓ 구체적인 부여 확률 수치 (스케일링 디자인 영역)
├─ 분배 결과를 소환수에 저장한다
│   ├─ 💻 소환수별 부여된 특성을 런타임 데이터로 보관한다
│   └─ 💻 동일 소환수도 전장마다 다른 특성을 가질 수 있다 (비영속)
└─ 역할군 데이터 (TACD-211 소속)
    └─ 🧩 역할군 enum/필드는 유닛 에픽(TACD-80)의 TACD-211에서 제공
```

### 5-2. 보조 관점 체크

**데이터/속성:**
- 💻 소환수 특성 목록 (SynergyDefinitionData 중 SummonTrait 타입) — TACD-249에서 데이터 구조 완료
- 🧩 역할군(Role) 정의 — TACD-211 (유닛 에픽, 기획 필요 상태)
- ❓ 특성별 역할군 우선 매핑 — 어떤 특성이 어떤 역할군을 우선하는지 (콘텐츠 디자인)
- ❓ 부여 확률 수치 — 기획서에서 "레벨디자인 및 스케일링 문서"로 위임 (스케일링 디자인)

**UI/정보 표시:**
- 각 소환수에게 부여된 특성 → 시너지 정보 UI(TACD-252)에서 표시. 이 Task 범위 외
- 이 Task 자체에서 별도 UI 불필요

**기반 작업:**
- 🧩 역할군 데이터 구조 + UnitDefinitionData 역할군 필드 — TACD-211 소속
- 🧩 소환수 풀(32체) 구성 — 소환술사 편성 시스템에서 제공. 이 Task는 풀을 입력으로 받기만 함
