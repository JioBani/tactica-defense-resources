# T1: 소환술사 전장 배치 및 라이프사이클

> 상태: 완료

## 5단계: Sub-task 도출

### 5-1. 동작 트리 상세화

```
소환술사 전장 배치 및 라이프사이클
├── 💻 전장에 배치된다
│   ├── 💻 전장 시작 시 4명의 소환술사가 자동 생성된다
│   │   ├── 💻 출전 편성 데이터에서 선택된 4명의 소환술사 정보를 수신한다
│   │   │   └── ❓ 출전 편성 데이터 전달 구조 (SceneDataManager 확장? 별도 컨테이너?)
│   │   └── 💻 소환술사 정보(스탯, 외형 등)를 기반으로 소환술사 엔티티를 생성한다
│   ├── 💻 소환술사가 라인 끝 위치에 자동 배치된다
│   │   ├── 💻 4개 라인의 왼쪽 끝(방어선 측)에 각 1명씩 배치된다
│   │   └── ❓ 어떤 소환술사가 어떤 라인에 배치되는지 결정 규칙
│   │       (출전 편성 순서? 플레이어 지정? 고정?)
│   └── ❓ 시각적 표현 (소환술사가 전장에서 어떻게 보이는가?)
│       └── ❓ 소환수와 구별되는 외형/크기/이펙트 등
├── 💻 대기한다 (전투 전 상태)
│   ├── 💻 전투 시작 전(Maintenance 페이즈)에 라인 끝에서 대기 상태를 유지한다
│   └── 💻 전투 페이즈에서 침략자가 도달하지 않은 동안에도 대기 상태를 유지한다
│       └── 🧩 전투 행동(공격/피격)은 T2 소환술사 전투 소속
├── 💻 라운드 간 상태 관리
│   ├── 💻 라운드(전투 페이즈) 종료 시 소환술사의 HP가 회복된다
│   │   └── ❓ 전부 회복? 부분 회복? (소환수는 기획서에 "전부 회복"으로 명시됨.
│   │       소환술사에 대한 명시적 기재 없음)
│   ├── 💻 라운드 실패 후 재도전 시 소환술사 상태가 초기화된다
│   │   └── 💻 다운 상태에서 복구되어 대기 상태로 전환된다
│   └── 💻 전장 종료(승리/패배) 시 소환술사 엔티티가 정리된다
└── 💻 소환술사 데이터 참조
    ├── 💻 전장에서 소환술사의 육성 데이터(레벨/승급)를 참조한다
    │   └── 🧩 육성 데이터에 따른 시너지 효과 강화는 TACD-110 시너지 에픽 소속
    └── ❓ 소환술사별 고유 데이터 구조 (소환수와 다른 스탯 체계? 전용 스탯?)
```

### 5-2. 상호작용 분석 (상세)

#### 대상별 x 내용별 매트릭스

##### 1. 상위 흐름: 라운드 시스템 (RoundManager / PhaseType)

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | **전장 시작**(Start): 소환술사 4명 생성/배치 촉발 | (없음 - 배치 자체는 외부에 이벤트 불필요) |
| 트리거 | **Maintenance 진입**: 소환술사를 Waiting 상태로 전환, HP 회복 | (없음) |
| 트리거 | **Combat 진입**: 소환술사를 전투 대기 상태(Idle)로 전환 | (없음) |
| 트리거 | **End/RoundLose 진입**: 소환술사를 Freeze 상태로 전환 | (없음) |
| 트리거 | **BattleWin/BattleLose 진입**: 전장 종료 시 소환술사 정리 | (없음) |
| 데이터 흐름 | 현재 Phase 정보 (어떤 페이즈인지) | (없음) |
| 효과 | Phase에 따라 소환술사의 ActionState가 변경됨 | (없음) |

##### 2. 인접 기능: 침략자 시스템 (Aggressor)

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | 침략자가 라인 끝(소환술사 위치)에 도달 시 → 교전 시작 촉발 | (없음 - 교전 자체는 T2 소속) |
| 데이터 흐름 | (없음) | 소환술사의 위치 정보 (침략자 이동 목표 지점) |
| 효과 | (없음 - 효과는 T2에서 처리) | (없음) |

> 참고: 현재 LifeCrystalAttacker가 DefenseLine 태그 충돌로 OnRoundLose를 발행하는 구조. 소환술사 도입 시 이 메커니즘이 "소환술사 엔티티와의 교전"으로 대체된다.

##### 3. 인접 기능: 소환수 시스템 (Defender)

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | (없음) | (없음) |
| 데이터 흐름 | (없음) | (없음) |
| 효과 | (없음) | (없음) |

> 소환술사와 소환수는 이 Task(배치/라이프사이클)에서 직접 상호작용하지 않는다. 연결 지점은 시너지(TACD-110)를 통해서만 존재한다.

##### 4. 공유 시스템: 유닛 프레임워크 (Unit, ActionStateController, UnitStatSheet 등)

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | (없음 - 프레임워크 자체는 트리거를 주지 않음) | (없음) |
| 데이터 흐름 | UnitStatSheet: HP, 스탯 관리 기반 제공 | (없음) |
| 데이터 흐름 | ActionStateController: 상태 전환 프레임워크 제공 | (없음) |
| 데이터 흐름 | HealthBar: HP 표시 기반 제공 | (없음) |
| 효과 | (없음) | (없음) |

> 소환술사가 유닛 프레임워크(Unit 클래스, ActionStateController, UnitStatSheet, HealthBar 등)의 공통 컴포넌트를 **활용**한다. 기존 Defender/Aggressor와 동일하게 유닛 공통 기반 위에서 동작하되, 소환술사는 "제3의 유닛 유형"으로 분류된다.

##### 5. 데이터: 출전 편성 데이터

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | (없음 - 전장 시작 시 데이터를 읽어올 뿐) | (없음) |
| 데이터 흐름 | 출전 편성에서 선택된 소환술사 4명의 정보 (ID, 스탯, 외형, 육성 데이터) | (없음) |
| 효과 | (없음) | (없음) |

> 현재 SceneDataManager는 selectedBattlefield만 관리. 출전 편성 데이터(소환술사 4명)를 전달하는 구조가 아직 없다 (TACD-129 출전편성 에픽 소속이나, 데이터 수신 측은 이 에픽).

##### 6. 데이터: 소환술사 육성 데이터

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | (없음) | (없음) |
| 데이터 흐름 | 소환술사 레벨, 승급 정보 → 전투 스탯 결정에 사용 | (없음) |
| 효과 | (없음) | (없음) |

> 육성 시스템(TACD-126)이 제공하는 데이터를 전장 진입 시 참조하여 소환술사 스탯을 초기화한다. 런타임 중 육성 데이터가 변경되지는 않는다.

##### 7. 플레이어

| 내용 분류 | <- 받는 | -> 보내는 |
|-----------|--------|---------|
| 트리거 | (없음) | (없음) |
| 데이터 흐름 | (없음) | 소환술사의 HP, 상태 등 시각 정보 표시 |
| 효과 | (없음) | (없음) |

> 소환술사는 자동 전투이며, 배치/라이프사이클에서 플레이어 조작은 없다. 플레이어는 소환술사의 상태(HP 등)를 시각적으로 확인만 한다.
