# T2: 소환술사 전투

> 상태: 완료

---

## 5단계: Sub-task 도출

### 5-1. 동작 트리 상세화

```
소환술사 전투
├── 💻 교전한다
│   ├── 💻 교전 시작 (침략자 도달 감지)
│   │   ├── 🧩 침략자가 라인 끝에 도달한다 (침략자 에픽 소속)
│   │   └── 💻 소환술사가 교전 상태로 전환한다
│   │       └── ✅ ActionStateController 상태 전환 (Idle → Attack)
│   │           (유닛 공통 프레임워크에 전환 로직이 존재)
│   ├── 💻 공격한다
│   │   ├── 💻 공격 대상을 인식한다
│   │   │   └── ✅ Attacker.OnTriggerEnter2D → 사거리 내 적 감지 (유닛 공통)
│   │   ├── 💻 공격을 수행한다
│   │   │   ├── ✅ 공격 반복 루프 (DynamicRepeater 기반, 유닛 공통)
│   │   │   ├── ❓ 공격 방식 (물리/마법 구분)
│   │   │   │   └── 소환술사별로 물리/마법 공격 구분이 필요한지 기획 미정
│   │   │   ├── ❓ 공격 연출 (근접/원거리 구분)
│   │   │   │   └── 소환술사의 공격이 근접인지 원거리인지 기획 미정
│   │   │   └── ✅ 데미지 계산 (DamageCalculator, 유닛 공통)
│   │   └── 💻 공격 적중 처리
│   │       └── ✅ AttackContext → Victim.Hit (유닛 공통)
│   └── 💻 피격당한다
│       ├── 💻 피격 감지
│       │   └── ✅ Victim.Hit 호출 (유닛 공통)
│       ├── 💻 HP 감소
│       │   └── ✅ UnitStatSheet.SetCurrentHealth (유닛 공통)
│       ├── 💻 HP 바 갱신
│       │   └── ✅ HealthBar.Display (유닛 공통, Health.OnChange 구독)
│       └── ❓ 피격 연출 (피격 VFX/SFX)
│           └── 기획서에 명시 없음
├── 💻 다운된다
│   ├── 💻 HP 0 → 다운 상태 전환
│   │   └── ✅ ActionStateTransitionService: health <= 0 → Downed (유닛 공통)
│   ├── 💻 다운 시 비주얼 처리
│   │   └── ❓ 소환술사 다운 시 시각 연출 (기획서에 명시 없음)
│   └── 💻 전투 패배 트리거 발송
│       ├── 💻 소환술사 다운을 감지한다
│       │   └── 소환술사의 ActionState가 Downed로 전환됨을 감지
│       └── 💻 라운드 시스템에 패배를 알린다
│           └── GlobalEventBus로 패배 이벤트 발행
│               (현재는 OnRoundLoseEventDto를 LifeCrystalAttacker가 발행함)
└── 💻 교전 종료
    ├── 💻 전투 페이즈 종료 시 교전 중단
    │   └── ✅ PhaseType 전환 → ActionState Freeze (유닛 공통)
    └── 💻 라운드 간 HP 회복
        └── ❓ 소환술사의 라운드 간 HP 회복 규칙
            └── 소환수는 라운드 종료 시 HP 전부 회복 (기획서 2.1.1)
                소환술사도 동일하게 전부 회복하는지 기획 미정
```

### 5-2. 상호작용 분석 (상세)

#### 대상 1: 침략자 (인접 기능)

| 내용 분류 | 방향 | 상세 |
|-----------|------|------|
| 트리거 | ← 받는 | 침략자가 라인 끝(소환술사 위치)에 도달하면 소환술사와의 교전이 시작된다 |
| 트리거 | → 보내는 | (없음 - 침략자에게 별도 이벤트를 보내지 않음, 공격은 Victim.Hit으로 직접 전달) |
| 데이터 흐름 | ← 받는 | 침략자의 Victim 컴포넌트(공격 대상), 침략자의 스탯(방어력 등 데미지 계산에 필요) |
| 데이터 흐름 | → 보내는 | 소환술사의 Victim 컴포넌트(침략자가 공격할 대상) |
| 효과 | ← 받는 | 침략자의 공격으로 소환술사 HP 감소 |
| 효과 | → 보내는 | 소환술사의 공격으로 침략자 HP 감소 |

**현재 구현과의 관계**: 현재 LifeCrystalAttacker가 침략자의 라인 끝 도달을 감지하여 즉시 OnRoundLoseEventDto를 발행하고 침략자를 DeSpawn한다. 소환술사 전투 도입 시, 라인 끝 도달 → 즉시 패배가 아니라 → 소환술사와 교전 → 소환술사 다운 시 패배로 흐름이 변경된다.

#### 대상 2: 라운드 시스템 (상위 흐름)

| 내용 분류 | 방향 | 상세 |
|-----------|------|------|
| 트리거 | ← 받는 | 전투 페이즈 시작 (PhaseType.Combat) → 소환술사 전투 가능 상태로 전환 |
| 트리거 | ← 받는 | 전투 페이즈 종료 (PhaseType.End/RoundLose 등) → 소환술사 교전 중단 |
| 트리거 | → 보내는 | 소환술사 다운 감지 → 전투 패배 이벤트 발송 → RoundManager가 RoundLose/BattleLose 전환 |
| 데이터 흐름 | ← 받는 | 현재 페이즈 상태 (IStateListener\<PhaseType\> 통해 수신) |
| 데이터 흐름 | → 보내는 | (없음) |
| 효과 | ← 받는 | 라운드 종료 시 소환술사 상태 리셋(Waiting 전환, HP 회복 등) |
| 효과 | → 보내는 | 패배 이벤트 발행으로 라운드 시스템의 페이즈 전환을 촉발 |

#### 대상 3: 유닛 공통 프레임워크 (공유 시스템)

| 내용 분류 | 방향 | 상세 |
|-----------|------|------|
| 트리거 | ← 받는 | (없음 - 프레임워크는 트리거를 보내지 않음) |
| 트리거 | → 보내는 | (없음) |
| 데이터 흐름 | ← 받는 | UnitStatSheet(스탯), ActionStateType(현재 상태), Attacker(공격 로직), Victim(피격 로직) |
| 데이터 흐름 | → 보내는 | (없음 - 프레임워크를 사용할 뿐 데이터를 제공하지 않음) |
| 효과 | ← 받는 | ActionStateController 상태 전환, DamageCalculator 데미지 계산, HealthBar HP 표시 |
| 효과 | → 보내는 | (없음) |

**비고**: 소환술사는 유닛이므로 Unit, Attacker, Victim, ActionStateController, HealthBar, UnitStatSheet 등 기존 유닛 공통 컴포넌트를 그대로 사용한다. 소환수/침략자와 동일한 공통 프레임워크 위에서 동작한다.

#### 대상 4: 소환술사 데이터 / 육성 데이터 (데이터)

| 내용 분류 | 방향 | 상세 |
|-----------|------|------|
| 트리거 | ← 받는 | (없음) |
| 트리거 | → 보내는 | (없음) |
| 데이터 흐름 | ← 받는 | 소환술사별 전투 스탯 (물리 공격력, 마법 공격력, 공격 속도, 사거리, 최대 체력, 방어력 등). 육성 상태(레벨/승급)에 따른 스탯 반영 |
| 데이터 흐름 | → 보내는 | (없음) |
| 효과 | ← 받는 | 스탯 데이터가 전투 행동(데미지, 공격 속도, 생존력)을 결정 |
| 효과 | → 보내는 | (없음) |

### 5-3. 보조 관점 체크

#### 데이터/속성

| 항목 | 필요 데이터 | 출처/현황 |
|------|------------|----------|
| 소환술사 전투 스탯 | 물리 공격력, 마법 공격력, 공격 속도, 사거리, 최대 체력, 물리 방어력, 마법 방어력, 치명타 확률/배율, 피해 증가/감소 | ❓ 소환술사 전용 ScriptableObject(UnitStatsByLevelData 형식) 필요. 구체적 수치는 스케일링 디자인에서 결정 |
| 소환술사 공격 타입 | 물리 공격 / 마법 공격 구분 | ❓ 기획서에 소환술사의 공격 타입이 명시되지 않음. 소환술사별로 다를 수 있는지도 미정 |
| 소환술사 공격 형태 | 근접(Melee) / 원거리(Ranged) 구분 | ❓ 기획서에 명시 없음. 근접이면 직접 Hit, 원거리면 Projectile 필요 |
| 소환술사 라운드 간 HP 회복 규칙 | 라운드 종료 시 HP 전부 회복 여부 | ❓ 소환수는 전부 회복(기획서 2.1.1)이나 소환술사에 대한 명시 없음 |
| 소환술사 다운 판정 기준 | HP 0 → 다운 | ✅ 기획서 명시: "HP가 0이 되면 전장 패배" |
| 전투 패배 조건 | 소환술사 중 한 명이라도 다운 | ✅ 기획서 명시 (07 기획서 line 348) |

#### UI/정보 표시

| 표시 가능 정보 | 플레이어에게 표시 필요성 | 현황 |
|---------------|----------------------|------|
| 소환술사 HP 바 | 필요 - 소환술사가 피격당하면 HP가 줄어드는 것을 확인해야 전략적 판단(배치 재조정 등) 가능 | ✅ HealthBar 컴포넌트가 유닛 공통으로 존재 |
| 소환술사 다운 상태 표시 | 필요 - 다운되면 전투 패배이므로 시각적으로 명확히 표시 필요 | ❓ 다운 시 비주얼 연출 기획 없음 |
| 소환술사 공격 연출 | 필요 - 소환술사가 "공격을 하며"라고 기획서에 명시되어 있으므로 공격 행동이 보여야 함 | ❓ 공격 애니메이션/이펙트 기획 없음 |
| 소환술사 피격 연출 | 권장 - 피격 사실을 인지할 수 있어야 위기감 전달 | ❓ 피격 VFX/SFX 기획 없음 |

#### 기반 작업

| 기반 작업 | 설명 | 상태 |
|----------|------|------|
| T1 소환술사 전장 배치 | 소환술사가 전장에 존재해야 전투 가능. T1이 선행 | 미착수 |
| 침략자 라인 끝 도달 → 소환술사 교전 전환 | 현재 LifeCrystalAttacker가 라인 끝 도달 시 즉시 RoundLose를 발행함. 소환술사 교전 도입 시 이 흐름을 변경해야 함: 라인 끝 도달 → 소환술사와 교전 시작(침략자가 소환술사를 공격 대상으로 인식) | 기존 LifeCrystal 시스템 수정 필요 |
| 소환술사 전투 스탯 데이터 정의 | 소환술사별 전투 능력치 ScriptableObject 에셋 생성 | ❓ 콘텐츠 디자인 + 스케일링 디자인 영역 |
| 소환술사 Fraction 정의 | 소환술사는 Defender(아군)와 같은 Fraction이어야 침략자와 교전 가능. 기존 Fraction enum에 포함 여부 확인 필요 | ✅ 기존 Fraction에 Defender 값 존재 (소환수와 동일 Fraction 사용 가능) |
