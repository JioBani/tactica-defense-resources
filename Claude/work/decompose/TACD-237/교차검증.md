# 교차 검증: TACD-237 - [전장] 소환술사

## 분석 기반

- **기획서 참조**: `00 기획서.md` (line 79-80, 126-142, 230-232), `07 게임 시스템 기획서.md` (line 348, 183-190, 395-420)
- **에픽 경계**: 소환술사가 전장 내에서 존재하고 행동하는 것 전체
- **제외 범위**: 시너지(TACD-110), 육성(TACD-126), 출전 편성(TACD-129), 패배 판정(TACD-111), 유닛 공통 기반(TACD-80), 소환수(TACD-81), 침략자(TACD-83)

## 분해 과정

### 축 1: 동작 트리

소환술사의 라이프사이클을 따라 말단 동작을 나열한다.

**생성 단계**
- 전장 진입 시 소환술사가 전장에 생성/배치된다
  - 출전 편성에서 선택된 소환술사 4명이 전장에 등장해야 한다
  - 소환술사가 배치되는 위치는 각 라인의 방어선(왼쪽 끝)이다

**대기(정비 페이즈)**
- 소환술사는 정비 페이즈 동안 전장에 존재하며 대기한다
- 라운드 간 HP 회복 (소환수처럼 라운드 종료 시 상태 복원이 필요한가?)

**전투 페이즈 — 통상 상태**
- 소환술사는 전투 중 기본적으로 대기 상태 (침략자가 도달하지 않은 동안)

**전투 페이즈 — 교전 상태**
- 침략자가 라인 끝까지 도착하면 소환술사와 직접 교전이 발생한다
  - 소환술사가 침략자를 공격한다 (기획서: "소환술사도 공격을 하며")
  - 소환술사가 침략자로부터 피격당한다 (기획서: "피격에 따라 HP 감소")

**다운(소멸)**
- 소환술사 HP가 0이 되면 다운된다
- 한 명이라도 다운되면 전투 패배 (→ 패배 판정 트리거를 발행하는 것까지가 이 에픽의 범위, 판정 로직 자체는 TACD-111)

### 축 2: 상호작용

**인접 기능: 침략자(TACD-83)**
- 침략자 → 소환술사: 침략자가 방어선에 도달했을 때 소환술사를 공격 대상으로 인식하는 연결 지점
- 현재 구현: 침략자가 DefenseLine 태그에 닿으면 바로 OnRoundLoseEventDto를 발행하고 디스폰됨
- 이 에픽에서 필요한 변경: 침략자가 방어선 도달 시 "바로 패배" 대신 "소환술사와 교전" 시작

**인접 기능: 소환수(TACD-81)**
- 소환수와 소환술사는 같은 진영이지만 역할이 다름
- 소환술사는 소환수처럼 드래그/배치하지 않고, 라인 끝에 고정 배치됨
- 직접적 상호작용은 없음 (소환술사가 소환수에게 시너지를 부여하는 것은 TACD-110 범위)

**상위 흐름: 라운드/페이즈(TACD-111)**
- 전투 시작 시: 소환술사가 교전 가능 상태로 전환
- 전투 종료 시: 소환술사 상태 초기화 (HP 회복 등)
- 라운드 패배 트리거: 현재 OnRoundLoseEventDto를 침략자 방어선 도달 시 발행 → 소환술사 다운 시 발행으로 변경해야 함

**공유 시스템: 유닛 프레임워크(TACD-80)**
- 소환술사가 유닛 공통 기능(스탯, HP, 공격, 피격)을 사용하는지 여부
- 소환술사는 유닛의 일종이지만, 소환수/침략자와 동일한 Unit 컴포넌트를 그대로 쓸지, 별도 구조를 가질지 결정이 필요

**데이터: 출전 편성(TACD-129)**
- 출전 편성에서 선택된 소환술사 목록을 전장에서 참조하여 소환술사를 생성해야 함
- 이 에픽에서는 "전달받은 소환술사 목록을 기반으로 소환술사를 전장에 배치하는 것"까지가 범위

**플레이어**
- 플레이어가 소환술사를 직접 조작하지는 않음 (드래그/배치 불가, 자동 배치)
- 소환술사 HP 상태를 플레이어가 인지할 수 있어야 함

### 축 3: 보조 관점

**데이터/속성**
- 소환술사의 전장 내 스탯 (HP, 공격력 등) — 어디서 오는가?
  - 육성 상태(레벨/승급)에 따른 스탯은 TACD-126 범위이지만, "스탯을 읽어서 전장에 반영하는 구조"는 이 에픽에서 필요
- 소환술사 정의 데이터 — 소환술사를 식별/생성하기 위한 데이터 구조

**UI/정보 표시**
- 소환술사 HP바 표시
- 소환술사 위치의 시각적 표현 (전장 라인 끝에 소환술사 스프라이트/아이콘 표시)
- 소환술사 다운 시 시각적 피드백

**기반 작업**
- 소환술사 개념 자체가 전장에 존재하지 않으므로, 소환술사를 전장 엔티티로 표현하는 기반 구조가 필요
- 현재 방어선 도달 = 즉시 패배 구조를 소환술사 교전 구조로 전환하는 기반 변경
