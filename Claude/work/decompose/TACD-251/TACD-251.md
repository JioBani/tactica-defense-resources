# TACD-251: [전장 - 시너지] - 시너지 계산 및 효과 적용

> 상태: 완료

---

## 5단계: Sub-task 도출

### 5-1. 동작 트리 상세화

```
시너지 계산 및 효과 적용
├─ 시너지 카운트
│   ├─ 전장(BattleArea) 배치 소환수에서 시너지별 카운트를 집계한다
│   ├─ 동일 소환수는 중복 카운트하지 않는다 (종류 기준)
│   └─ 소환술사 효과와 소환수 특성을 각각 별도로 카운트한다
├─ 임계치 판정
│   ├─ 소환술사 효과 임계치(2/4/6/8)를 판정한다
│   ├─ 소환수 특성 임계치(2/3/4)를 판정한다
│   └─ 카운트에 해당하는 최고 활성 티어를 결정한다
├─ 시너지 효과 적용
│   ├─ 임계치 충족 시 해당 시너지를 보유한 소환수에게 효과를 적용한다
│   ├─ 수정자(Modifier) 시스템을 통해 스탯을 변경한다 (유닛 에픽)
│   └─ (기획 누락) 스탯 외 행동/메카닉 변경 효과의 적용 프레임워크
├─ 시너지 효과 해제
│   ├─ 임계치 미달 시 해당 시너지의 효과를 제거한다
│   └─ 티어 변경(하향) 시 이전 효과를 제거하고 새 티어 효과를 적용한다
└─ 시너지 갱신 트리거
    ├─ 소환수 배치(WaitingArea→BattleArea) 시 시너지를 재계산한다
    ├─ 소환수 제거(BattleArea→WaitingArea 또는 환원) 시 시너지를 재계산한다
    └─ 전투 중 소환수 다운 시에는 재계산하지 않는다 (카운트 유지)
```

### 5-2. 보조 관점 체크

**데이터/속성:**
- 시너지별 현재 카운트 값 (런타임 상태)
- 시너지별 현재 활성 티어 (런타임 상태)
- 효과 적용 대상 소환수 목록 (시너지 보유 기준 필터링)
- (기존 구현) SynergyDefinitionData.GetActiveTier() (TACD-249)
- (기존 구현) SynergyTier.constants (TACD-249)

**UI/정보 표시:**
- 시너지 정보 UI (TACD-252 소속) — 이 Task에서는 UI가 참조할 런타임 상태만 제공

**기반 작업:**
- (기존 구현) 시너지 데이터 구조 (TACD-249, 완료)
- (인접 Task) 소환수 특성 부여 (TACD-250)
- (인접 Task) 소환술사 효과 부여 (TACD-254)
- (기존 구현) DefenderManager.OnPlacementChange 이벤트
- (기존 구현) StatModifier / UnitStat 수정자 시스템
