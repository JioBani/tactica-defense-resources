## 5단계: Sub-task 도출 ✅

### 5-1. 동작 트리 상세화

```
소환수 라이프사이클
│
├─ 생성된다 (인스턴스 생성)
│   ├─ 🧩 소환 터미널에서 구매 → 프리팹 인스턴스화 (TACD-82)
│   └─ 💻 소환수 인스턴스 초기화 (성급/스탯/상태 세팅, 관리 목록 등록)
│
├─ 전투한다
│   └─ ✅ 유닛 공통 전투 라이프사이클 (TACD-121 완료)
│
├─ 다운된다 → 복귀한다
│   ├─ ✅ 다운 판정 (HP ≤ 0 → 비활성화) (TACD-121 완료)
│   └─ 💻 라운드 종료 시 HP 전부 회복 상태로 복귀
│
├─ 라운드를 넘긴다 (라운드 간 상태 관리)
│   ├─ 💻 HP 전부 회복
│   ├─ 💻 스킬 쿨다운 초기화
│   └─ ❓ 상태이상 처리 (종류별 해제 규칙 — 기획 미정)
│
└─ 제거된다
    └─ 🧩 환원(판매) 시 인스턴스 제거 (TACD-82)
```

### 5-2. 보조 관점 체크

**데이터/속성:**
- 💻 소환수 인스턴스 데이터 (고유 ID, 소환수 종류, 현재 성급, 강화 단계, 위치, 상태)

**UI/정보 표시:**
- 라운드 종료 시 일괄 회복이므로 별도 복귀 UI 불필요

**기반 작업:**
- ✅ 행동 상태 머신 프레임워크 (StateBase) — 이미 구현
- ✅ 유닛 공통 라이프사이클 — TACD-121 완료
- ✅ 성급별 스탯 적용 — TACD-209 완료
