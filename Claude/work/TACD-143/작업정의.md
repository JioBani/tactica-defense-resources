# TACD-143: 정비 페이즈에서만 상점 이용 가능하도록 변경

## 작업 내용

- 준비(Ready) 페이즈가 되면 상점 이용 불가하도록 변경
- 정비(Maintenance) 페이즈 시작 시 상점 이용 가능하도록 변경
- 패널 열기/닫기 애니메이션도 동일하게 적용

## 완료 정의

- 정비 페이즈에서만 상점(소환터미널) UI가 열려 있고 조작 가능
- 준비 페이즈 진입 시 상점 패널이 자동으로 닫히며(슬라이드 애니메이션), 버튼 조작 불가
- 정비 페이즈 진입 시 상점 패널이 자동으로 열리며(슬라이드 애니메이션), 버튼 조작 가능
- 상점 비활성 상태에서 구매/리롤/레벨업 등 경제 조작이 차단됨

## 현재 구조 분석

### 관련 파일
- `Assets/Scenes/Battle/Feature/Market/Scripts/MarketManager.cs` — 경제 로직, IStateListener<PhaseType> 구현
- `Assets/Scenes/Battle/Feature/Ui/Market/MarketUiManager.cs` — 터미널 UI, 패널 슬라이드 애니메이션
- `Assets/Scenes/Battle/Feature/Ui/Market/DefenderSlot.cs` — 슬롯 구매 UI
- `Assets/Scenes/Battle/Feature/Round/RoundManager.cs` — 페이즈 상태머신
- `Assets/Scenes/Battle/Feature/Round/Phase/PhaseType.cs` — 페이즈 열거형

### 현재 동작
- MarketManager는 IStateListener<PhaseType>를 구현하지만, Maintenance 진입 시 수입/리롤만 처리
- 다른 페이즈에서 상점 조작을 차단하는 로직이 없음
- MarketUiManager는 수동 토글(OnClickToggle)만 지원, 페이즈 연동 없음
- 드래그 중에만 market GameObject를 숨기는 로직이 있음 (MarketManager.OnDefenderDrag)
