# TACD-263: 유닛 전장 출입 시 시너지 효과 부여/해제

> 상위 이슈: TACD-251 [전장 - 시너지] 시너지 계산 및 효과 적용

## 요구사항

시너지가 활성 상태일 때, 유닛의 전장 출입에 따라 시너지 효과가 올바르게 부여·해제된다.

1. **진입**: 시너지가 이미 활성화된 상태에서 새 유닛이 전장에 배치되면, 해당 유닛에 활성 시너지 효과가 부여된다.
2. **퇴장**: 유닛이 대기석으로 이동하거나 판매되면, 해당 유닛의 시너지 효과가 해제된다.

## 배경지식

### 현재 시너지 효과 적용 흐름

Defender 배치 변경 → `SynergyManager.Recalculate()` → 시너지 카운트·티어 재계산 → 티어 **변경** 시에만 `SynergyController.OnActivated/OnDeactivated` 호출.

- `OnActivated(defenders)`: 전달받은 시너지 보유 Defender 전원에게 SSE를 Apply한다.
- `OnDeactivated()`: `_appliedDefenders`를 초기화한다. SSE는 `ActiveTier` RxValue 구독으로 자체 제거된다.

### 현재 코드의 갭

| 시나리오 | 현재 동작 | 기대 동작 |
|---------|---------|---------|
| 시너지 활성 상태에서 새 유닛 진입 (티어 변화 없음) | SSE 미부여 | 해당 유닛에 SSE 부여 |
| 유닛 퇴장, 시너지 유지 (티어 변화 없음) | SSE 유지 (대기석에서도 효과 지속) | 해당 유닛의 SSE 해제 |

핵심 원인: `Recalculate()`에서 `result.Changed == false`이면 아무 처리도 하지 않기 때문.

## 완료 정의

- 시너지 활성 상태에서 해당 시너지를 가진 새 유닛을 전장에 배치하면, 해당 유닛에 시너지 효과가 부여된다
- 시너지 활성 상태에서 유닛을 대기석으로 이동하면, 해당 유닛의 시너지 효과가 해제된다
- 시너지 활성 상태에서 유닛을 판매하면, 해당 유닛의 시너지 효과가 해제된다
- 기존 동작(티어 활성화/비활성화/업그레이드)은 변경 없이 유지된다

## 기존 코드 참조

- `SynergyManager.Recalculate()` — 티어 변경 시에만 Controller에 위임. 티어 불변 시 신규 유닛/퇴장 유닛 처리 없음
- `SynergyController.OnActivated()` / `ApplySSEToDefenders()` — `_appliedDefenders` HashSet으로 중복 방지
- `SynergyController.OnDeactivated()` — `_appliedDefenders` 초기화만 수행
- `SynergyStatusEffect.OnActiveTierChanged(null)` → `RequestRemove()` — SSE 자체 제거 메커니즘
