# TACD-293: 라운드 종료시 애니메이션이 적용되지 않는 문제 해결

> 상위 이슈: TACD-287 [전장 - 유닛] 애니메이션 프레임워크 제작

## 요구사항

라운드 종료 시 유닛이 Freeze 상태로 전환되면 스프라이트가 사라지는 문제를 해결한다.
실제 애니메이션 클립이 없는 상태(Freeze, Move, Waiting)로 전환될 때, 빈 placeholder 클립이 재생되면서 SpriteRenderer의 스프라이트가 null로 초기화되는 것이 원인.

## 배경지식

- 현재 스프라이트 시트는 3행(Idle, Attack, Downed)만 제공. Move, Freeze, Waiting 클립은 없음
- OverrideController에서 override되지 않은 상태는 빈 placeholder 클립이 재생됨
- `animator.Play("Freeze")` 호출 시 placeholder_freeze에 스프라이트 키프레임이 없어서 SpriteRenderer.sprite가 null이 됨

## 완료 정의

- 실제 클립이 없는 상태로 전환 시, 이전 상태의 마지막 스프라이트가 유지된다
- Freeze, Move, Waiting 등 미구현 상태에서 유닛이 사라지지 않는다

## 기존 코드 참조

- `Assets/Common/Scripts/StateBase/StateAnimator.cs:96-101` — OnStateEnter에서 PlayState 호출
- `Assets/Common/Scripts/StateBase/StateAnimator.cs:77-94` — CacheClipLengths에서 placeholder 포함 캐싱
