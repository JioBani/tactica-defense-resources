# 작업지시서

> 이 문서는 Claude Code 세션이 개발 작업 시 따라야 할 루틴이다.
> 참고 문서: `git.md`, `develop.md`, `CLAUDE.md`

---

## 작업 파일 구조

브랜치(일감)별로 작업 문서를 분리하여 관리한다.

```
Claude/work/
└── TACD-{번호}/
    └── 작업정의.md       # 작업 내용, 완료 정의, 작업 단위 체크리스트
```

---

## 작업 루틴

### 1. 작업 내용 확인

1. 사용자가 지정한 Jira 일감의 작업 내용과 완료 목표 확인
2. 해당 일감의 브랜치 생성 및 전환 (`TACD-{번호}`)
3. 하위 작업 확인:
   - **하위 작업이 있는 경우**: 하위 작업 목록을 조회하고, 각 하위 작업에 대해 2~6단계를 반복 수행한다
   - **하위 작업이 없는 경우**: 사용자에게 해당 Task 단위로 바로 작업할지 확인 후 진행한다
4. 작업 디렉토리 생성 (`Claude/work/TACD-{번호}/`, 하위 작업 시 하위 번호 사용)

### 2. 작업정의 작성

1. Jira 일감의 작업 내용과 완료 목표 확인
2. 기획서(`Document/기획서/`)에서 관련 요구사항 수집
3. 기존 코드를 분석하여 현재 구조 파악 (`CLAUDE.md` 참고)
4. `Claude/work/TACD-{번호}/작업정의.md` 작성
   - 작업해야 할 내용 정리
   - 완료 정의 (무엇이 되면 이 일감이 끝난 것인지)
   - 관련 기획서 항목 및 기존 코드 참조
5. 사용자 확인 후 다음 단계로 진행
6. **대시보드 갱신** (백그라운드)

### 3. 작업 단위 분리

1. 작업정의를 바탕으로 할 일을 **작업 단위**로 나눈다
   - 해당 일감에서 만들거나 수정할 대상에 한정
   - 코드 수준의 상세 구현 계획이 아닌, **무엇을 해야 하는지** 단위로 분리한다
2. `Claude/work/TACD-{번호}/작업정의.md`에 체크리스트를 추가 작성한다
   - `[ ]` 미완료 / `[~]` 진행 중 / `[x]` 완료
3. 사용자 확인 후 다음 단계로 진행
4. **대시보드 갱신** (백그라운드)

### 4. 구현

아래 사이클을 작업 단위마다 반복한다.

```
4.1 작업정의에서 다음 항목 선택 → [~] 표시
4.2 plan 모드로 해당 항목의 구현 계획 작성
4.3 plan 모드 승인
4.4 코드 구현
4.5 Unity 컴파일 확인 (refresh_unity + read_console)
4.6 유닛 테스트 작성
4.7 유닛 테스트 실행
4.8 실패 시 수정 → 4.7로 복귀
4.9 통과 시 작업정의에서 [x] 표시 → 대시보드 갱신 (백그라운드) → 다음 항목으로 이동
```

### 5. 완료

1. 변경사항을 사용자에게 보여주고 확인을 받는다 (git diff 등)
2. 사용자 승인 후 `git.md` 형식에 따라 커밋
3. Jira 일감 상태 변경
4. **대시보드 갱신** (백그라운드)

## 작업 규칙

- **섹션 단위(예: 2, 3, 4) 작업 완료 시마다 사용자에게 확인을 받는다.**
- 확인 없이 다음 섹션으로 넘어가지 않는다.
- **체크리스트 초기 상태**: 작업정의.md를 작성할 때 체크리스트는 반드시 `[ ]` (미완료) 상태로 초기화한다. 실제로 완료한 항목만 `[x]`로 변경한다.
- **plan 모드 범위**: plan 모드는 현재 진행 중인 **작업 단위 1개**만 대상으로 한다. 여러 작업 단위를 하나의 plan에 묶지 않는다.

### 대시보드 갱신 규칙

작업 파일(작업정의.md)이 변경되는 시점마다 대시보드를 갱신한다.

- **갱신 시점**: 단계 2 완료, 단계 3 완료, 단계 4.9 항목 완료 시, 단계 5 완료
- **실행 방식**: `Task` 도구를 `run_in_background: true`로 실행하여 메인 작업을 차단하지 않는다
- **서브에이전트 지시 내용**:
  - `.claude/commands/dashboard.md`의 **단일 갱신 모드** 절차를 수행한다
  - 대상 티켓: 현재 작업 중인 `TACD-{번호}`
  - 서브에이전트 타입: `general-purpose`
